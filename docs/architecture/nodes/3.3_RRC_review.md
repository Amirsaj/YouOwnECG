**Node:** 3.3
**Reviewer:** Red Review Council (RRC)

**Gate Decision:** GATE CONDITIONAL

## Issues Found

### REQUIRED-1: STAT suppression window applies to same ECG only — cross-session state not specified

**Location:** Node 3.3, Section 7.2 (alert suppression window)

The spec states the 5-minute suppression window applies to the same ECG upload and that "cross-ECG suppression = 0." However, it does not specify where the suppression state is stored. If it is stored in React component state, the suppression resets on page reload — meaning a clinician who accidentally reloads the page after acknowledging a STEMI alert will be re-alerted immediately. In a noisy ER, this is disruptive.

More critically, if it is stored only client-side, the suppression state is not shared across tabs. A clinician who opens the same ECG in a second tab gets a second STAT alert for the same acknowledged finding.

**Required fix:** The suppression state for STAT alerts must be stored server-side (per ECG ID, per session token) with a TTL of 5 minutes. The client reads the suppression state on mount. Reload does not re-trigger an acknowledged STAT alert within the 5-minute window.

### REQUIRED-2: Pulse animation duration is 10 seconds — no specification of what happens at second 10

**Location:** Node 3.3, Section 2 (pre-attentive processing — motion)

The spec states: "After the first 10 seconds, the pulse animation stops (alert acknowledged visually)." This is a passive acknowledgment. The spec does not specify whether the full alert banner remains at the top of the viewport (without animation) or collapses to a smaller persistent indicator.

If the banner remains full-size without animation, it blocks 15–20% of the viewport for the duration of the session. If it disappears entirely, a clinician who looked away during the first 10 seconds might not see the STAT finding.

**Required fix:** After 10 seconds without explicit acknowledgment (button click): the banner shrinks to a persistent "STAT: POSSIBLE ANTERIOR STEMI" strip at the top of the viewport (24px height, Vermillion background, no animation). The full banner is restored on hover/focus. Explicit "ACKNOWLEDGE" button click dismisses to the "dismissed alerts" count.

## Not Issues

- Okabe-Ito palette choice is correct for colorblind accessibility.
- Redundant encoding (color + shape + text) on all urgency levels is correct.
- 44×44 px minimum touch target with 48 px preference for gloved hands is correctly specified.
- Traffic-light system with 5 levels (STAT/CRITICAL/ABNORMAL/NORMAL/INSUFFICIENT) is appropriate.
