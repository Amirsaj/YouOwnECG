## EPM Adjudication

### Issue 1 — Q-Wave Duration Formula Was Wrong (`R_idx - QRSon_idx`)
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

Section 6.2 corrected:
- `Q_duration_ms = (Q_idx - QRSon_idx) / 500 * 1000` — uses Q trough index (FPT column 4), not R peak
- Guard added: `Q_idx == -1` → `Q_duration_ms = None`
- `is_pathological_q()` updated to accept `None` and return `False`

The original formula computed Q-to-R interval (160–200 ms), not Q-wave width (10–40 ms). Every beat would have been flagged pathological. Issue resolved.

---

### Issue 2 — `detect_fragmented_qrs()` Used filtfilt on ~50-Sample Window
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

Section 6.2 function replaced:
- Removed `butter(2, 40/250) + filtfilt` approach — unstable on 35–60 sample segments
- Replaced with direct derivative sign-change counter: count turning points exceeding 0.05 mV, subtract 1 for primary R-peak, flag if ≥ 2 remaining
- Docstring explicitly notes why filtfilt must not be used here

Issue resolved.

---

### Issue 3 — ST Isoelectric Window Had No Bounds Check (Negative Index + WPW Corruption)
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 5.1 updated:
- `iso_start = max(0, QRSon_idx - 10)` — prevents negative index on first beat
- `isoelectric_reference_mv = None` when `iso_start == QRSon_idx`
- `isoelectric_reference_unreliable = True` when `PR_interval_ms < 60` (WPW / extreme short PR)
- All ST features for flagged beats set to None; agents must not use them

Issue resolved.

---

### Issue 4 — AFib Left `heart_rate_atrial_bpm` and `av_ratio` Undefined
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 1.1 and 1.3 updated:
- `heart_rate_atrial_bpm`: explicit guard for `len(valid_ppeaks) < 2` → `None`
- `av_ratio`: explicit guard for `heart_rate_atrial_bpm is None or == 0` → `None`
- Section 11 schema updated: both fields documented as nullable

Issue resolved.

---

### Issue 5 — P Terminal Force V1 Unit Inconsistency (mV·ms vs mV·s, 1000× error)
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 6.1 updated:
- Formula now explicitly computes in **mV·s** (amplitude_mv × duration_s)
- Schema field renamed `p_terminal_force_v1_mv_s` throughout (Section 6.1 and Section 11)
- Unit note added warning against converting duration to ms before multiplication
- Threshold stated as `> 0.04 mV·s` with no alternative notation to avoid confusion

Issue resolved.
