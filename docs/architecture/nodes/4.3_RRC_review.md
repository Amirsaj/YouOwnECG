## Issues Found

### REQUIRED-1: ingest_all_books() has no error handling for per-batch embedding failures

**Location:** Node 4.3, Section 5

The batch embedding loop:

```python
for i in range(0, len(texts), batch_size):
    batch = texts[i : i + batch_size]
    response = await openai_client.embeddings.create(...)
    embeddings.extend([normalize_embedding(r.embedding) for r in response.data])
```

If any batch fails (API timeout, rate limit, network error), the exception propagates and the entire ingestion fails — discarding all work done on preceding batches. Re-running from scratch wastes time and API cost.

**Required fix:** Implement per-batch retry with exponential backoff, and checkpoint completed batches to disk:

```python
EMBEDDING_CHECKPOINT_PATH = "data/rag/embedding_checkpoint.jsonl"

async def embed_with_retry(client, texts: list[str], max_retries: int = 3) -> list[np.ndarray]:
    for attempt in range(max_retries):
        try:
            response = await client.embeddings.create(
                model="text-embedding-3-small", input=texts
            )
            return [normalize_embedding(r.embedding) for r in response.data]
        except Exception as e:
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)
```

Additionally, save completed chunk embeddings to a JSONL checkpoint after each batch. On re-run, skip chunks that are already checkpointed.

### REQUIRED-2: CLIP model loaded once but spec does not address memory management

**Location:** Node 4.3, Section 4

The CLIP model (`clip-vit-large-patch14`) is ~600 MB on MPS. The spec does not state when the CLIP model is loaded or unloaded. If it is loaded at server startup alongside the main ECG ML pipeline models, total MPS memory consumption may exceed the available MPS unified memory on a 16 GB Apple Silicon system.

**Required fix:** The CLIP model is loaded lazily — only when a morphology comparison query is issued (Node 3.4 Section 5). It is unloaded (del model; torch.mps.empty_cache()) after the figure embedding query completes. This is an offline/infrequent operation that does not need to be kept resident.
