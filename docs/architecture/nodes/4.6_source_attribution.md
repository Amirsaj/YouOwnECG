## 1. Attribution Requirement

Every LLM-generated clinical finding must have a traceable source. "Traceable" means:
- The specific textbook, chapter, section, and page that supports the claim
- The exact text that was retrieved and used
- The similarity score at retrieval time

This is required for:
1. **Cardiologist trust** (Node 3.4 Section 4 reasoning chain viewer)
2. **Audit trail** — if a finding is wrong, the investigation can determine whether the retrieval was flawed or the agent misinterpreted correct evidence
3. **Medical-legal** — "AI said X" is not sufficient; "AI said X, citing Chou's Ch. 5 p. 142" is auditable

---

## 2. Citation Data Structure

```python
@dataclass
class Citation:
    chunk_id: str
    book: str
    book_short: str
    chapter: str
    section: str
    page_number: int
    similarity_score: float
    retrieved_text: str         # the actual text returned from retrieval
    context_expanded: bool      # True if adjacent chunk was appended
```

Each `DiagnosticFinding` (Node 2.3 output schema) carries a `citations: list[Citation]` field populated by the IT/MR agents.

---

## 3. Agent Citation Protocol

The agent system prompt instructs the agent to cite chunks by their index in the RAG context block (`[1]`, `[2]`, etc.). The safety layer (Node 4.5) maps these indices back to chunk IDs:

```python
def parse_agent_citations(agent_text: str, retrieved_chunks: list[dict]) -> list[Citation]:
    """
    Extracts [N] citation markers from agent text and maps to retrieved chunks.
    """
    citations = []
    cited_indices = set(int(m) - 1 for m in re.findall(r'\[(\d+)\]', agent_text))

    for idx in cited_indices:
        if 0 <= idx < len(retrieved_chunks):
            chunk = retrieved_chunks[idx]
            citations.append(Citation(
                chunk_id=chunk["chunk_id"],
                book=chunk["book"],
                book_short=chunk["book_short"],
                chapter=chunk["chapter"],
                section=chunk["section"],
                page_number=chunk["page_number"],
                similarity_score=chunk["similarity_score"],
                retrieved_text=chunk["text"],
                context_expanded=chunk.get("context_expanded", False),
            ))

    return citations
```

If an agent claims a finding but cites no chunks (0 `[N]` markers), the safety layer (Node 4.5) flags it as CITATION_UNVERIFIED and the finding carries `citations: []`.

## 5. No-Citation Fallback

When no relevant chunks are retrieved (all similarity scores below `SIMILARITY_THRESHOLD = 0.35`):

```python
NO_CITATION_TEXT = (
    "No textbook evidence retrieved for this finding. "
    "The analysis is based on signal features alone. "
    "Clinical correlation with standard ECG criteria is required."
)
```

This text is shown in place of a citation. The finding is NOT suppressed — signal-only findings are valid (e.g., heart rate is a signal measurement, not a textbook claim).
