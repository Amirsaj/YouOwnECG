## Issues Found

### REQUIRED-1: filter_metadata_by_book() uses index.reconstruct() — not available on all FAISS index types

**Location:** Node 4.4, Section 5

The metadata filter implementation uses `index.reconstruct(i)` to retrieve individual vectors from the FAISS index for building a subset index. `index.reconstruct()` is only available on flat indexes (`IndexFlat*`). If the index type is changed in the future (e.g., to `IndexIVFFlat` for a larger corpus), `reconstruct()` will raise a `RuntimeError`. The spec establishes this only for the current flat index but does not guard against future index type changes.

Additionally, reconstructing all vectors individually in a loop (one `reconstruct()` call per filtered chunk) is O(n) per filter operation. For 3,130 vectors this is acceptable, but it is fragile.

**Required fix:** Document the `index.reconstruct()` dependency explicitly:

```python
def filter_metadata_by_book(...) -> tuple[faiss.IndexFlatIP, list[dict]]:
    """
    REQUIRES IndexFlatIP. Do not call on IVF or HNSW indexes.
    Reconstruct is not available on non-flat indexes.
    """
    assert isinstance(index, faiss.IndexFlatIP), \
        "filter_metadata_by_book requires IndexFlatIP"
    # ...
```

An alternative that avoids reconstruct: store the raw embedding matrix in memory alongside the index as a NumPy array. Subset filtering then uses array indexing rather than FAISS reconstruct.

### REQUIRED-2: SIMILARITY_THRESHOLD = 0.35 is arbitrary — no empirical basis stated

**Location:** Node 4.4, Section 3

The threshold `0.35` for minimum cosine similarity is presented without justification. For clinical text, cosine similarity of 0.35 between a clinical query and a textbook chunk is relatively low — queries about rare conditions (e.g., epsilon wave, Brugada Type 2) may retrieve weakly-related ST elevation passages at 0.35 similarity while the truly relevant ARVC section scores 0.42.

If the threshold is too low, low-relevance chunks pollute the agent context and may introduce confounding information. If too high, rare condition queries return 0 results.

**Required fix:** The threshold must be validated against the 10 test queries defined in VT-4.4-01. After running those queries, set the threshold to the lowest similarity score seen in a TRUE POSITIVE result across all 10 queries, minus 0.05 margin. Document the empirically derived value in the spec. Until validation is run, the spec must not present 0.35 as a final value but as a placeholder to be validated.
