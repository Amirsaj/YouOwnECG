### REQUIRED-1: Out-of-range citation indices logged and marked

**Fix applied to Node 4.6:**

```python
@dataclass
class InvalidCitation:
    index: int  # 1-based, as cited by the agent
    reason: str = "Citation index out of range"

def parse_agent_citations(
    agent_text: str,
    retrieved_chunks: list[dict],
    ecg_id: str,
    finding_type: str,
) -> list[Citation | InvalidCitation]:
    citations = []
    cited_indices = set(int(m) - 1 for m in re.findall(r'\[(\d+)\]', agent_text))

    for idx in sorted(cited_indices):
        if 0 <= idx < len(retrieved_chunks):
            chunk = retrieved_chunks[idx]
            citations.append(Citation(
                chunk_id=chunk["chunk_id"],
                book=chunk["book"],
                book_short=chunk["book_short"],
                chapter=chunk["chapter"],
                section=chunk["section"],
                page_number=chunk["page_number"],
                similarity_score=chunk["similarity_score"],
                retrieved_text=chunk["text"],
                context_expanded=chunk.get("context_expanded", False),
            ))
        else:
            log_safety_event(
                event_type="INVALID_CITATION_INDEX",
                ecg_id=ecg_id,
                finding=finding_type,
                details=f"Agent cited [{idx + 1}] but only {len(retrieved_chunks)} chunks retrieved",
            )
            citations.append(InvalidCitation(index=idx + 1))

    return citations
```

The UI renders `InvalidCitation` as: `"[{N}] Citation not found — agent referenced a source that was not retrieved."`.

        )
