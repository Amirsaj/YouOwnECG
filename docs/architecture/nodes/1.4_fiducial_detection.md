## Executive Summary

Node 1.4 defines how the pipeline detects cardiac fiducial points — the precise sample positions of each wave's onset, peak, and offset in every beat of every lead. The output is an FPT (Fiducial Point Table): a `(n_beats, 13)` integer array per lead, with columns `[Pon, Ppeak, Poff, QRSon, Q, R, S, QRSoff, L, Ton, Tpeak, Toff, class]`.

Two constraints are absolute:

1. **ECGdeli is the primary detector.** It is already in the repository ([ecgdeli/](ecgdeli/)). The FD-FPRE must understand how it works from first principles — not just call it blindly — so that failure modes are predictable and the output can be validated.

2. **FPT indices are in the 500 Hz preprocessed signal sample grid.** All downstream interval and voltage computations depend on this. The sample grid must be consistent with `PreprocessedECGRecord.preprocessed_signal` (500 Hz).

## 1. First Principles: Cardiac Wave Genesis

### 1.1 P Wave — Atrial Depolarization

Atrial depolarization begins at the SA node (right atrium, near the SVC junction) and propagates:
- First: right atrium → early P wave (first 30–40 ms)
- Then: left atrium via Bachmann's bundle → late P wave (40–100 ms)

The normal P wave is therefore a smooth, low-amplitude wave reflecting the sequential activation of right then left atrium. Duration: 80–100 ms. Amplitude: 0.1–0.25 mV in lead II. Axis: 0°–75° (same direction as QRS axis for normal sinus rhythm).

**Lead II** has the largest P-wave amplitude in normal sinus rhythm because the atrial activation vector points directly toward the positive electrode of lead II (inferior-leftward). This is why Lead II is the standard rhythm strip lead.

**P-wave detection challenges:**
- Low amplitude — easily obscured by baseline wander or noise
- Absent in atrial fibrillation (replaced by f-waves at 350–600 bpm)
- Retrograde in junctional rhythms (inverted in II, upright in aVR)
- Buried in preceding T wave in tachycardia (PR shortening)
- Notched (bifid) in left atrial enlargement — two humps appear as one P wave in some detection strategies

### 1.2 QRS Complex — Ventricular Depolarization

The sequence of ventricular depolarization:
1. **Interventricular septum** (left-to-right, 0–30 ms): produces the initial q wave in lateral leads (I, aVL, V5, V6) and r wave in V1
2. **Left ventricular free wall** (endocardium to epicardium, 30–70 ms): dominant R wave in lateral leads; S wave in V1/V2
3. **Basal right ventricle and posterior LV** (60–100 ms): terminal forces, S wave in lateral leads

Normal QRS duration: 80–100 ms (≤120 ms). The QRS is the highest-amplitude, fastest-changing waveform in the ECG — its detection is the most reliable step in fiducial analysis.

**Fiducial points within QRS:**
- `QRSon`: onset of ventricular depolarization — earliest deflection from isoelectric line
- `Q`: first negative deflection after QRSon (if present; absent in many leads)
- `R`: first positive peak (largest in most leads)
- `S`: first negative deflection after R (if present)
- `QRSoff` (J-point): end of ventricular depolarization — transition from QRS to ST segment

**QRSoff (J-point)** is the most clinically critical fiducial for ischemia detection. It is where ST elevation is measured from. Detection error of ±10 ms at QRSoff translates directly to ST elevation measurement error.

### 1.3 T Wave — Ventricular Repolarization

Repolarization occurs epicardium-to-endocardium (opposite order to depolarization) because the epicardium has a shorter action potential duration. This means repolarization produces a vector in the same general direction as depolarization — hence the T wave is normally upright in the same leads where the QRS is upright.

- `Ton`: T-wave onset — difficult to detect precisely because the ST segment transitions gradually into the T wave
- `Tpeak`: peak of T wave
- `Toff`: T-wave offset — transition back to isoelectric baseline

**T-wave detection challenges:**
- Variable morphology (upright, inverted, biphasic, peaked, flattened)
- U wave may follow T wave and be mistaken for T-wave offset
- In prolonged QT, T-wave offset is delayed and may overlap with next P wave in tachycardia
- Peaked T waves in hyperkalemia or de Winter pattern may confuse algorithms designed for normal T-wave amplitude

### 1.4 The 13-Column FPT Schema

ECGdeli produces a `(n_beats, 13)` integer array per lead:

| Column | Index | Name | Description |
|--------|-------|------|-------------|
| 0 | Pon | P onset | Sample index of P-wave start |
| 1 | Ppeak | P peak | Sample index of P-wave peak |
| 2 | Poff | P offset | Sample index of P-wave end |
| 3 | QRSon | QRS onset | Sample index of QRS start |
| 4 | Q | Q peak | Sample index of Q-wave trough (or QRSon if no Q) |
| 5 | R | R peak | Sample index of R-wave peak |
| 6 | S | S peak | Sample index of S-wave trough (or QRSoff if no S) |
| 7 | QRSoff | J-point | Sample index of QRS end / J-point |
| 8 | L | L point | J+60ms reference point for ST measurement |
| 9 | Ton | T onset | Sample index of T-wave start |
| 10 | Tpeak | T peak | Sample index of T-wave peak |
| 11 | Toff | T offset | Sample index of T-wave end |
| 12 | class | Beat class | 0=normal, 1=PVC, 2=PAC, 3=aberrant, 4=unanalyzable |

**Sentinel value:** ECGdeli uses `−1` for fiducial points that could not be detected for a given beat (e.g., Pon = −1 in atrial fibrillation). Feature extraction must check for −1 before using any FPT index.

**`L` point (column 8):** The L point is QRSoff + 60 ms (at 500 Hz: QRSoff + 30 samples). This is the standard ST measurement point (J+60ms) per AHA guidelines. ECGdeli computes it directly. Feature extraction uses `L` for ST amplitude measurement, not `QRSoff`.

## 2. ECGdeli Algorithm — Deep Analysis

ECGdeli's `mastermind_delineate()` operates in four phases. Understanding each phase is prerequisite to predicting and handling its failure modes.

### 2.1 Phase 0: QRS Detection (prerequisite to mastermind)

Before `mastermind_delineate()` can run, R-peaks and initial QRS boundaries must be detected. ECGdeli uses `qrs_detection()` for this, which implements a wavelet-based QRS detector:

```python
from ecgdeli.qrs_detection import qrs_detection

# Returns: fpt with columns 3–7 (QRSon, Q, R, S, QRSoff) filled
# Columns 0–2 (P wave) and 9–11 (T wave) are filled by mastermind_delineate
fpt = qrs_detection(lead_signal, samplerate=500)
```

The QRS detector uses the Haar stationary wavelet transform (SWT) at scale 2¹ (16 ms at 500 Hz) to produce a signal that emphasizes rapid transitions (QRS upstrokes/downstrokes) while suppressing slow waves (P, T) and high-frequency noise.

**Why Haar SWT for QRS:**
- Haar wavelet is a step function — its inner product with a signal measures the local difference, equivalent to a derivative
- Scale 2¹ at 500 Hz captures transitions in the 20–80 Hz range — exactly the QRS bandwidth
- SWT (not DWT): the stationary (undecimated) transform preserves time alignment, so detected peaks are at the correct sample positions

### 2.2 Phase 1: Condition Detection

```python
condition = detect_condition(signal, fs, fpt)
```

ECGdeli classifies the recording's dominant rhythm condition before running fiducial detection. This matters because the detection strategy for P waves in atrial fibrillation differs from sinus rhythm. Conditions detected include:
- Normal sinus rhythm
- Atrial fibrillation (no organized P waves)
- High-rate tachycardia (P waves buried in T waves)
- Complete heart block (P waves and QRS complexes at independent rates)

The condition detection uses RR regularity (coefficient of variation of RR intervals) and P-wave presence/absence features from a preliminary scan.

**Clinical significance:** If condition detection fails (e.g., misclassifies AFib as sinus), the P-wave detection algorithm runs and produces spurious "P-wave" detections from f-waves. The FPT will show `Pon ≠ −1` for every beat in an AFib recording. Feature extraction must not interpret these as true P waves — it should check the condition metadata.

### 2.3 Phase 2: Catalog Extrema

```python
beat_features = catalog_extrema(signal, fs, fpt)
```

For each beat (defined by the QRS boundaries from Phase 0), ECGdeli locates all local extrema (maxima and minima) in the expected P-wave and T-wave regions. These extrema are candidates for fiducial point assignment.

The search windows are:
- P-wave search: [previous QRSoff + 50 ms, current QRSon − 20 ms]
- T-wave search: [QRSoff + 50 ms, next QRSon − 50 ms]

If the RR interval is short (tachycardia), these windows shrink and may overlap — a known failure mode at heart rates >150 bpm.

### 2.4 Phase 3: Build Consensus Template

```python
template = build_consensus_template(beat_features, condition, fs)
```

ECGdeli computes an ensemble average of all beats in the recording, then identifies the fiducial positions in the template. This is the key insight of the "mastermind" algorithm: instead of detecting fiducials in each noisy beat independently, it first builds a high-SNR template and detects fiducials in the template, then projects those positions back to individual beats with local refinement.

**Why template-based detection is more robust:**
- The template has SNR proportional to √(N_beats) above individual beat SNR
- Rare morphology variants (PVCs, ectopic beats) are averaged out of the template
- The template provides a global reference that constrains per-beat detection

**Consequence for PVCs and aberrant beats:** Because the template is built from the majority morphology, PVCs will have their fiducial positions estimated by projecting the normal template onto an abnormal beat. For PVC-dominant features (QRS duration, morphology), the FPT values for `beat_class = 1` beats are less reliable. Feature extraction should flag these.

### 2.5 Phase 4: Assign Fiducials

```python
fpt = assign_fiducials(signal, fs, fpt, beat_features, template, condition)
```

For each beat, ECGdeli:
1. Uses the template fiducial positions as a starting point
2. Applies local waveform matching to refine positions within ±30 ms of the template position
3. Sets `−1` for any fiducial that cannot be found in the expected window
4. Sets `beat_class` based on morphology deviation from template

## 3. Multi-Lead FPT Construction

ECGdeli operates per-lead. The pipeline must construct a consistent multi-lead FPT:

```python
def run_ecgdeli(preprocessed: "PreprocessedECGRecord",
                quality_report: "QualityReport") -> dict[str, np.ndarray]:
    """
    Runs ECGdeli on all available leads. Returns per-lead FPT dict.

    Only runs on leads in quality_report.available_for_rhythm
    (GOOD, ACCEPTABLE, MARGINAL quality). Skips POOR and UNUSABLE leads.

    Returns: {"I": fpt_I, "II": fpt_II, ...}
    Each fpt is (n_beats, 13) int array or None if the lead was skipped.
    """
    fpts = {}
    for lead_name in CANONICAL_LEADS:
        label = quality_report.lead_labels.get(lead_name, "UNUSABLE")
        if label in ("POOR", "UNUSABLE"):
            fpts[lead_name] = None
            continue

        lead_idx = CANONICAL_LEADS.index(lead_name)
        lead_signal = preprocessed.preprocessed_signal[:, lead_idx]

        # Restrict to safe analysis window (Node 1.2 edge margin)
        start = preprocessed.safe_analysis_start_sample
        end = preprocessed.safe_analysis_end_sample
        lead_signal_windowed = lead_signal[start:end]

        try:
            fpt = qrs_detection(lead_signal_windowed, samplerate=500)
            fpt = mastermind_delineate(lead_signal_windowed, 500, fpt,
                                        lead_name=lead_name)
            # Offset indices back to full-signal coordinates
            fpt_full = fpt.copy()
            for col in range(12):  # all except class column
                mask = fpt_full[:, col] != -1
                fpt_full[mask, col] += start
            fpts[lead_name] = fpt_full
        except Exception as e:
            # ECGdeli failure on a single lead is non-fatal
            # Log the error and treat lead as POOR
            fpts[lead_name] = None

    return fpts
```

### 3.1 Reference Lead Selection

For beat segmentation and RR interval computation, a single reference lead FPT is designated:

```python
REFERENCE_LEAD_PRIORITY = ["II", "V5", "V1", "I", "aVF"]

def select_reference_lead(fpts: dict[str, np.ndarray | None]) -> str:
    """
    Selects the reference lead for beat segmentation and RR intervals.
    Prefers II (standard rhythm lead); falls back in priority order.
    """
    for lead in REFERENCE_LEAD_PRIORITY:
        if fpts.get(lead) is not None:
            return lead
    raise ValueError("No usable lead FPT available — pipeline should have rejected at quality gate")
```

All beat timing (RR intervals, heart rate, rhythm analysis) is derived from the reference lead FPT. Other leads are aligned to reference lead beats using R-peak proximity matching.

### 3.2 Cross-Lead Beat Alignment

Beat `i` in lead II corresponds to the same cardiac cycle as beat `i` in V1 — but ECGdeli detects beats independently per lead, so the beat count may differ slightly if a beat is missed or spuriously added in one lead. Alignment:

```python
def align_beats_across_leads(fpts: dict[str, np.ndarray | None],
                              reference_lead: str,
                              max_rr_deviation_ms: float = 50.0) -> dict[str, np.ndarray]:
    """
    Aligns per-lead FPTs to the reference lead beat sequence.

    For each beat in the reference lead, finds the corresponding beat in
    each other lead by R-peak proximity (within max_rr_deviation_ms).

    If no corresponding beat is found: inserts a row of -1 (missing beat).
    If extra beats detected in a lead: discards beats not matching reference.

    Result: all FPTs have the same number of rows (one per reference beat).
    """
    ref_fpt = fpts[reference_lead]
    ref_r_peaks = ref_fpt[:, 5]  # column 5 = R peak
    max_dev_samples = int(max_rr_deviation_ms * 500 / 1000)

    aligned = {reference_lead: ref_fpt}
    for lead_name, fpt in fpts.items():
        if lead_name == reference_lead or fpt is None:
            aligned[lead_name] = fpt
            continue

        lead_r = fpt[:, 5]
        new_fpt = np.full((len(ref_r_peaks), 13), -1, dtype=int)
        new_fpt[:, 12] = 4  # default: unanalyzable until matched

        for ref_idx, ref_r in enumerate(ref_r_peaks):
            # Find nearest R-peak in this lead
            dists = np.abs(lead_r - ref_r)
            nearest = np.argmin(dists)
            if dists[nearest] <= max_dev_samples:
                new_fpt[ref_idx] = fpt[nearest]

        aligned[lead_name] = new_fpt

    return aligned
```

## 4. Beat Classification

ECGdeli sets `fpt[:, 12]` (beat class) per beat per lead. The pipeline uses a consensus class across leads:

```python
BEAT_CLASS_NAMES = {
    0: "NORMAL",
    1: "PVC",       # Premature Ventricular Contraction
    2: "PAC",       # Premature Atrial Contraction
    3: "ABERRANT",  # Aberrantly conducted (BBB morphology in one beat)
    4: "UNANALYZABLE",
}

def consensus_beat_class(fpts_aligned: dict[str, np.ndarray],
                          beat_idx: int) -> int:
    """
    Determines consensus beat class across leads.

    Majority vote: if >50% of available leads agree on a class, use it.
    Tie: prefer abnormal class (1 PVC > 2 PAC > 3 ABERRANT > 0 NORMAL).
    If all leads return 4 (unanalyzable): return 4.
    """
    classes = []
    for fpt in fpts_aligned.values():
        if fpt is not None and beat_idx < len(fpt):
            c = fpt[beat_idx, 12]
            if c != -1:
                classes.append(c)

    if not classes:
        return 4

    counts = {}
    for c in classes:
        counts[c] = counts.get(c, 0) + 1

    # Majority vote
    majority_class = max(counts, key=counts.get)
    if counts[majority_class] > len(classes) / 2:
        return majority_class

    # Tie-break: prefer abnormal
    for cls in [4, 1, 2, 3, 0]:
        if cls in counts:
            return cls
    return 4
```

## 4b. Condition-Based FPT Corrections

Before constructing `FiducialTable`, a mandatory post-processing step applies condition-specific corrections to the aligned FPTs. This is not optional — it prevents downstream agents from receiving contradictory fiducial evidence.

```python
def _apply_condition_corrections(
    fpts_aligned: dict[str, np.ndarray | None],
    afib_suspected: bool,
    rhythm_regularity_cv: float,
) -> dict[str, np.ndarray | None]:
    """
    Applies condition-based corrections to the aligned FPT dict.
    Must be called before FiducialTable construction.

    AFib correction:
    If afib_suspected is True, ECGdeli may have populated P-wave columns
    (Pon, Ppeak, Poff — columns 0, 1, 2) with f-wave peaks rather than
    true P waves. These are clinically meaningless and would mislead the
    RRC agent's P-wave analysis. Set all P-wave columns to -1 for all beats.

    High-irregularity warning:
    If rhythm_regularity_cv > 0.20 (highly irregular rhythm, may not be
    true AFib but rhythm is unreliable), mark Pon/Poff as unreliable by
    setting them to -1 while preserving Ppeak (which is more robustly
    detected). This applies to 2nd/3rd degree AV block and multifocal AT.
    """
    corrected = {}
    for lead_name, fpt in fpts_aligned.items():
        if fpt is None:
            corrected[lead_name] = None
            continue

        fpt_c = fpt.copy()

        if afib_suspected:
            # Zero out all P-wave columns — no organized atrial activity
            fpt_c[:, 0] = -1  # Pon
            fpt_c[:, 1] = -1  # Ppeak
            fpt_c[:, 2] = -1  # Poff

        elif rhythm_regularity_cv > 0.20:
            # Irregular rhythm: P-wave onset/offset less reliable
            fpt_c[:, 0] = -1  # Pon
            fpt_c[:, 2] = -1  # Poff
            # Ppeak preserved — most robustly detected even in irregular rhythms

        corrected[lead_name] = fpt_c

    return corrected
```

This function is called immediately after `align_beats_across_leads()` and before any `FiducialTable` field is populated.

## 5. FPT Output Structure

```python
@dataclass
class FiducialTable:
    """
    Output of Node 1.4. Contains per-lead FPTs and summary beat metadata.
    Consumed by Node 1.5 (Feature Extraction).
    """
    fpts: dict[str, np.ndarray | None]
    # Per-lead (n_beats, 13) int arrays; None for skipped leads.
    # All FPTs have the same number of rows (aligned to reference lead).
    # Indices are in FULL-SIGNAL coordinates (0 = sample 0 of preprocessed_signal).
    # Sentinel -1 = fiducial not detected for this beat.
    # n_beats = beats within the safe analysis window only (Node 1.2 edge margin).

    reference_lead: str                 # lead used for beat timing (e.g., "II")
    n_beats: int                        # beats within safe analysis window (used for all analysis)
    total_recording_beats: int          # beats detected in full recording incl. edge zones
                                        # (used for reporting only — not for analysis)
    beat_classes: np.ndarray            # shape (n_beats,) — consensus class per beat
    rr_intervals_ms: np.ndarray         # shape (n_beats-1,) — RR intervals in ms
                                        # computed from offset-corrected full-signal R-peak indices:
                                        # rr_ms[i] = (R[i+1] - R[i]) / 500 * 1000
    sampling_rate_hz: float             # always 500.0

    # Rhythm metadata (from reference lead, safe-window beats only)
    mean_heart_rate_bpm: float
    rhythm_regularity_cv: float         # coefficient of variation of RR intervals

    # Condition flags (set by ECGdeli condition detection + _apply_condition_corrections)
    afib_suspected: bool                # no organized P waves — P-wave cols set to -1
    high_rate_suspected: bool           # HR > 150 bpm — T/P overlap possible
```

## 6. Known Failure Modes and Mitigations

| Failure mode | Trigger | ECGdeli behavior | Mitigation |
|-------------|---------|-----------------|------------|
| AFib — no P waves | f-waves at 350–600 bpm | Ppeak/Pon/Poff set to random values OR -1 | Check `afib_suspected`; set all P-wave columns to -1 in post-processing |
| Tachycardia >150 bpm | Short RR intervals | T-wave window overlaps next P | Feature extraction uses narrowed T-wave window; flag QT as unreliable |
| PVC with wide QRS | QRS duration >120 ms | QRSoff detection less reliable | Use consensus class; flag PVC beats for reduced morphology confidence |
| Complete flatline beat | Electrode pop | `beat_class = 4` | Already excluded by quality gate |
| Short recording (<5 beats) | <10 second recording | Template has poor SNR | Reject at quality gate (MIN_CLEAN_BEATS = 3) |
| LBBB morphology | Wide, notched QRS | QRS detection accurate; P-wave harder | Condition detection flags; P-wave detection uses wider search window |
