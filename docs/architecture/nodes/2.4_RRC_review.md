## Issue 1 — STAT Override Rule Fires on `combined_confidence == "LOW"` but Not on `INSUFFICIENT_EVIDENCE`; STEMI in a Single Poor-Quality Lead Is Silently Suppressed

**Severity: REQUIRED**

**Location:** Section 5.3, STAT Override:

```
If stat_threshold_met AND combined_confidence == "LOW":
    → Report as "LOW" confidence STAT finding, not suppressed.
```

**The flaw:**

The rule fires when `combined_confidence == "LOW"` but is silent on `combined_confidence == "INSUFFICIENT_EVIDENCE"`. Consider this scenario:

- V2 SQI label: POOR (`lead_quality_modifier = 0.5`)
- V3 SQI label: UNUSABLE (`lead_quality_modifier = 0.0`)
- ST elevation at V2: 0.32 mV (above 0.2 mV threshold but noise makes measurement uncertain)
- Agent verbalized confidence: "LOW"

With the `lead_quality_cap = min(raw, lead_quality)` formula from Section 5.1:
- V3 findings: `lead_quality_cap = 0.0` → `score_to_level(0.0) = "INSUFFICIENT_EVIDENCE"`
- V2 findings: `lead_quality_cap = 0.5` × combined raw ≈ 0.32 → `score_to_level(0.32) = "LOW"` → STAT fires (correct)

But: if V2 is ALSO UNUSABLE (lead_quality = 0.0) and only V1 shows borderline elevation with POOR quality, the `lead_quality_cap` drives the score to INSUFFICIENT_EVIDENCE. The STAT override rule does not fire. An anterior STEMI equivalent in a noisy ECG is silently converted to "INSUFFICIENT_EVIDENCE" with no alert.

This violates the core STAT safety invariant: "STAT findings are NEVER suppressed due to low confidence."

**Required action:** Extend the STAT override to cover INSUFFICIENT_EVIDENCE:

```python
if stat_threshold_met:
    if combined_confidence in ("LOW", "INSUFFICIENT_EVIDENCE"):
        # STAT alert fires regardless of confidence level
        # The confidence level is shown to the clinician — it is not used to suppress
        final_level = combined_confidence  # preserve the original level for display
        stat_alert_fires = True
        alert_message = (
            f"POSSIBLE {finding_type.upper()} — {combined_confidence} CONFIDENCE "
            f"— SIGNAL QUALITY IMPAIRED — IMMEDIATE REVIEW REQUIRED"
        )
```

Add explicit language: "The STAT override fires on ALL confidence levels including INSUFFICIENT_EVIDENCE. Signal quality impairment is communicated in the alert message, not used as a suppression criterion."
