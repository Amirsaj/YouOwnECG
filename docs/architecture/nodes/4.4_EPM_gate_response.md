### REQUIRED-1: filter_metadata_by_book() — reconstruct() dependency documented; raw matrix alternative

**Fix applied to Node 4.4:**

The raw embedding matrix is stored alongside the FAISS index to avoid `reconstruct()` dependency:

```python
# In Node 4.3 ingest_all_books(), save the raw matrix:
np.save("data/rag/ecg_books_matrix.npy", np.stack(embeddings))

# In filter_metadata_by_book():
def filter_metadata_by_book(
    metadata: list[dict],
    embedding_matrix: np.ndarray,  # shape (N, 1536)
    allowed_books: list[str],
) -> tuple[faiss.IndexFlatIP, list[dict]]:
    """
    Does NOT use index.reconstruct() — uses raw embedding matrix instead.
    """
    subset_indices = [i for i, m in enumerate(metadata) if m["book_short"] in allowed_books]
    subset_metadata = [metadata[i] for i in subset_indices]
    subset_vectors = embedding_matrix[subset_indices].astype(np.float32)

    sub_index = faiss.IndexFlatIP(subset_vectors.shape[1])
    sub_index.add(subset_vectors)
    return sub_index, subset_metadata
```

The `embedding_matrix` is loaded at server startup alongside the FAISS index (~19 MB).
