## EPM Adjudication

### Issue 1 — `estimate_snr_db()` Circular Dependency with R-Peak Detection
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

`_bootstrap_r_peaks(signal_1d, fs)` added as an internal function to Node 1.3 — a simplified derivative-threshold detector (squared first-derivative → 150 ms integration → 60% threshold). Its output is used solely for SNR estimation bootstrapping.

`estimate_snr_db(signal_1d, fs)` signature changed: `r_peaks` parameter removed; R-peaks are now computed internally via `_bootstrap_r_peaks()`. Returns 0.0 (POOR quality) if fewer than 3 beats detected.

Explicitly documented:
- `_bootstrap_r_peaks` is NOT the clinical FPT
- Its output is NOT passed to ECGdeli
- ECGdeli performs its own full detection from scratch in Node 1.4
- Circular dependency eliminated: Node 1.3 is fully self-contained

Issue resolved.

---

### Issue 2 — `REQUIRED_LEADS["ischemia"]` Missing Inferior and Lateral Territory Leads
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

`REQUIRED_LEADS["ischemia"]` updated to all 11 non-aVR leads:
`["I", "II", "III", "aVL", "aVF", "V1", "V2", "V3", "V4", "V5", "V6"]`

`REQUIRED_LEADS["full_analysis"]` updated to include aVR as well (all 12).

Inline comment documents the rationale: inferior STEMI (II/III/aVF), lateral (I/aVL), anterior/posterior (V1–V6). Omitting any territory means a STEMI in that territory cannot be confirmed. Issue resolved.
