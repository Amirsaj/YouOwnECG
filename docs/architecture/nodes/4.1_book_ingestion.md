## 2. PDF Text Extraction

### 2.1 Extraction Tool: PyMuPDF (fitz)

PyMuPDF is chosen over pdfplumber and pypdf for three reasons:
- Native C library (fast: ~1 s per book)
- Returns text blocks with bounding-box coordinates (needed for layout reconstruction)
- Extracts images without re-encoding loss

```python
import fitz  # PyMuPDF

def extract_book(pdf_path: str) -> list[dict]:
    """
    Returns a list of page dicts, each containing:
      page_number, text_blocks, images
    text_blocks: list of {bbox, text, block_type}
    images: list of {bbox, xref, width, height}
    """
    doc = fitz.open(pdf_path)
    pages = []
    for page_num, page in enumerate(doc):
        blocks = page.get_text("dict", flags=fitz.TEXT_PRESERVE_LIGATURES)["blocks"]
        text_blocks = [b for b in blocks if b["type"] == 0]   # text blocks
        image_blocks = [b for b in blocks if b["type"] == 1]  # image blocks
        pages.append({
            "page_number": page_num + 1,
            "text_blocks": text_blocks,
            "images": [
                {
                    "bbox": b["bbox"],
                    "xref": b["image"][0] if b.get("image") else None,
                }
                for b in image_blocks
            ],
        })
    return pages
```

### 2.2 Multi-Column Layout Reconstruction

Chou's and Marriott's use two-column layouts. PyMuPDF returns blocks in PDF content-stream order, which may be left-column → right-column → next row (correct) or interleaved (incorrect). Column reconstruction:

```python
def sort_blocks_by_reading_order(text_blocks: list[dict], page_width: float) -> list[dict]:
    """
    Reconstructs reading order for two-column layouts.
    Heuristic: if blocks cluster into two x-coordinate groups (left < midpoint, right > midpoint),
    sort left column first (top-to-bottom), then right column.
    """
    midpoint = page_width / 2
    left_blocks = [b for b in text_blocks if b["bbox"][0] < midpoint * 0.9]
    right_blocks = [b for b in text_blocks if b["bbox"][0] >= midpoint * 0.9]

    if not right_blocks:
        return sorted(text_blocks, key=lambda b: b["bbox"][1])  # single column

    return (sorted(left_blocks, key=lambda b: b["bbox"][1]) +
            sorted(right_blocks, key=lambda b: b["bbox"][1]))
```

## 3. Book Structure Parsing

### 3.1 Hierarchy Detection

Chapter and section headers are identified by:
- Font size > body text (PyMuPDF spans include `size` field)
- Font weight (bold flag in span flags)
- All-caps text (chapter titles in Chou's)
- Position (header at top of page)

```python
def classify_block_type(block: dict, body_font_size: float) -> str:
    spans = [span for line in block.get("lines", []) for span in line.get("spans", [])]
    if not spans:
        return "UNKNOWN"
    max_size = max(s["size"] for s in spans)
    is_bold = any(s["flags"] & 16 for s in spans)  # flag bit 4 = bold
    text = " ".join(s["text"] for s in spans).strip()

    if max_size > body_font_size * 1.4:
        return "CHAPTER_HEADER"
    if max_size > body_font_size * 1.15 or is_bold:
        return "SECTION_HEADER"
    return "BODY"
```

## 4. Figure Extraction and Classification

### 4.1 Figure Extraction

```python
def extract_figures(doc: fitz.Document, pages: list[dict]) -> list[dict]:
    figures = []
    for page_data in pages:
        page = doc[page_data["page_number"] - 1]
        for img_info in page_data["images"]:
            if img_info["xref"] is None:
                continue
            xref = img_info["xref"]
            img_bytes = doc.extract_image(xref)["image"]
            figures.append({
                "page_number": page_data["page_number"],
                "bbox": img_info["bbox"],
                "image_bytes": img_bytes,
                "width": img_info.get("width"),
                "height": img_info.get("height"),
            })
    return figures
```

### 4.2 Figure Classification

Figures are classified into types using simple heuristics:

```python
def classify_figure(figure: dict, nearby_caption: str) -> str:
    """
    ECG_STRIP: narrow height (aspect ratio < 0.4), horizontal waveform pattern
    DIAGRAM: drawn illustration (vector-like, limited pixel variety)
    TABLE: text-dense with grid lines
    PHOTO: photograph (high pixel variance, continuous tones)
    """
    w, h = figure["width"], figure["height"]
    aspect = h / w if w > 0 else 1.0
    caption_lower = nearby_caption.lower()

    if any(kw in caption_lower for kw in ["ecg", "electrocardiogram", "lead", "rhythm"]):
        return "ECG_STRIP"
    if aspect < 0.4 and w > 400:
        return "ECG_STRIP"  # wide and short = likely ECG strip
    if "table" in caption_lower or "fig" not in caption_lower:
        return "TABLE"
    return "DIAGRAM"
```

### 4.3 Figure-to-Text Association

Each figure is associated with its nearest caption (text block immediately below the figure bounding box) and the paragraphs that reference it (by scanning for "Fig. X.Y" patterns):

```python
def associate_figure_to_text(
    figure: dict,
    page_text_blocks: list[dict],
) -> dict:
    """
    Finds the caption (text block immediately below the figure bbox)
    and the in-text references ("Fig. X.Y").

    For full-page figures (no text on the page), caption is set to the
    text from the FOLLOWING page that references this figure.
    """
    fig_bottom = figure["bbox"][3]  # y-coordinate of figure bottom
    # Find closest text block below the figure
    candidates = [b for b in page_text_blocks if b["bbox"][1] > fig_bottom]
    if not candidates:
        # Full-page figure — caption is on adjacent page
        return {"caption": None, "caption_page": "next_page", "in_text_refs": []}

    caption_block = min(candidates, key=lambda b: b["bbox"][1] - fig_bottom)
    caption_text = " ".join(
        span["text"]
        for line in caption_block.get("lines", [])
        for span in line.get("spans", [])
    )
    return {
        "caption": caption_text,
        "caption_page": figure["page_number"],
        "in_text_refs": [],  # populated by cross-reference scan
    }
```

**Full-page figure handling:** When a figure occupies the entire page (no adjacent text), the caption is sought on the next page. If neither adjacent page has a relevant caption, the figure is tagged `"caption": null` and included with a note `"isolated_figure": true`. These are NOT discarded — they may contain diagnostic ECG examples. They are embedded with their figure type and book/page metadata only.
