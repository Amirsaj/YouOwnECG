### REQUIRED-1: assemble_qa_context() context scope

**Fix applied to Node 3.1:**

`assemble_qa_context()` now assembles a bounded context object:

```python
def assemble_qa_context(
    ecg_store: ECGStore,
    selected_beat_idx: int | None,
    selected_lead: str | None,
) -> dict:
    ctx = {
        "beat_summary": ecg_store.beat_summary,          # ~600 tokens (Node 2.5)
        "diagnostic_findings": [
            {
                "finding_type": f.finding_type,
                "confidence": f.confidence,
                "clinical_summary": f.clinical_summary,
                "key_features": f.key_feature_values,    # selected values only
            }
            for f in ecg_store.diagnostic_result.findings
        ],
        "vision_conflicts": ecg_store.vision_result.signal_vision_conflicts,
    }
    if selected_beat_idx is not None:
        beat = ecg_store.beat_summary.beats[selected_beat_idx]
        ctx["selected_beat"] = beat
        ctx["selected_lead"] = selected_lead
    return ctx
    # Full FeatureObject is NOT included
```

### REQUIRED-2: Beat selection fallback for empty R-peak array

**Fix applied to Node 3.1:**

Beat selection now guards against empty R-peak arrays:

```typescript
function handleLeadClick(clickX: number, leadId: string) {
    const rPeaks = ecgStore.featureObject.r_peak_positions[leadId] ?? [];
    if (rPeaks.length === 0) {
        showTooltip("Beat selection unavailable — signal quality insufficient for this lead.");
        return;
    }
    const beatIdx = rPeaks.reduce((best, pos, idx) =>
        Math.abs(pos - clickX) < Math.abs(rPeaks[best] - clickX) ? idx : best, 0);
    setSelectedBeatIdx(beatIdx);
}
```
