# RRC Adversarial Review — Node 1.2 Signal Preprocessing (FD-FPRE)

**Reviewed by:** Research Review Committee (RRC)
**Document under review:** `docs/architecture/nodes/1.2_signal_preprocessing.md`
**Review date:** 2026-03-29
**Verdict:** GATE CONDITIONAL

---

## Preamble

Node 1.2 is the filter stage. Its outputs feed directly into ECGdeli (fiducial detection) and feature extraction. A filter that silently distorts ST segments, over-suppresses P waves, or introduces phase artifacts creates systematic errors in every downstream clinical measurement. The RRC reviews with corresponding care.

Three issues are raised. One is CRITICAL and two are REQUIRED.

---

## Issue 1 — `filtfilt` on Short or Edge-Padded Recordings Produces Transient Artifacts

**Severity: CRITICAL**

**Location:** Sections 3.1, 4.1, 5.2 — all three filter stages use `scipy.signal.filtfilt`.

**The flaw:**

`filtfilt` pads the signal at both ends before applying the forward-and-reverse filter to avoid edge transients. The default padding mode in `scipy.signal.filtfilt` is `padtype="odd"` which reflects the signal around the endpoints. For ECG recordings, this causes a problem:

If the recording starts or ends mid-beat (which is common — PTB-XL records are 10 seconds, but many ER ECGs are 6–8 seconds), the reflection padding creates an artificial mirrored ECG pattern at the edges. When the lowpass or highpass filter is applied, these mirrored beats generate transient oscillations that decay into the first and last ~0.5–2 seconds of the actual signal (depending on filter order and cutoff frequency).

For a 0.05 Hz highpass filter (Section 4.1):
- The filter's time constant is approximately 1/(2π × 0.05) ≈ 3.2 seconds
- Edge transients from reflection padding extend for ≈ 3× the time constant ≈ 10 seconds
- For a 10-second PTB-XL recording, **the entire recording could be within the transient zone**

This means the filtered signal at the beginning and end of short recordings may have baseline that is NOT the recording's true baseline — it is a filter artifact. If beats at the start or end of the recording are used for feature extraction, their ST segments may be systematically shifted by the filter edge artifact, producing false ST elevation or depression findings.

The document does not acknowledge this limitation or specify any mitigation (minimum recording duration requirement, edge-beat exclusion, or alternative zero-phase filter implementation for short recordings).

**Required action:** (1) Document the edge-transient risk for 0.05 Hz highpass filtfilt on short recordings. (2) Specify a minimum safe analysis window: exclude the first and last `max(3 / cutoff_hz, 1.0)` seconds of the preprocessed signal from feature extraction and fiducial analysis. This is a hard requirement, not a recommendation — the first and last beats of a short recording are not reliable. (3) Add this exclusion zone to `PreprocessedECGRecord` as a typed field: `safe_analysis_start_sample: int` and `safe_analysis_end_sample: int`.

## Issue 2 — Severe Wander Detection Heuristic Is Unreliable

**Severity: REQUIRED**

**Location:** Section 9, `preprocess_ecg()`:

```python
wander_amplitude = np.max(np.abs(lead_signal)) - np.percentile(np.abs(lead_signal), 10)
if wander_amplitude > 2.0:  # severe wander: use wavelet fallback
```

**The flaw:**

This heuristic attempts to measure baseline wander amplitude before the highpass filter is applied (since the decision is made at Step 2, after only the notch filter). The formula `max(|signal|) - percentile(|signal|, 10)` does not isolate the low-frequency wander component — it measures the total signal range minus a low-percentile baseline estimate. In practice:

- A signal with a large QRS complex (R-wave amplitude 2.5 mV, as is common in V5 for LVH) will always trigger this threshold even with zero baseline wander: `max ≈ 2.5 mV`, `10th percentile ≈ 0`, difference ≈ 2.5 mV > 2.0 mV. The wavelet fallback fires on every high-amplitude lead.
- Conversely, a signal with 2.5 mV of slow sinusoidal baseline wander but small QRS amplitude may not trigger if the wander shifts the entire signal upward — the 10th percentile follows the wander, and the difference stays below 2.0 mV.

The heuristic conflates QRS amplitude with wander amplitude. It will apply the wavelet fallback unnecessarily to high-amplitude leads (causing wavelet-related distortion where it is not needed) and may miss severe wander in small-amplitude leads (where it would be beneficial).

**Required action:** Replace the amplitude heuristic with a low-frequency band power estimate. Compute the power in the 0.05–0.5 Hz band (where wander lives) using the FFT of the lead signal; if this band power exceeds a threshold (to be determined empirically from PTB-XL noisy subset), use the wavelet fallback. This directly measures wander energy rather than total signal range.

## Issue 3 — Resampling Not Applied to `raw.signal`; Morphology Features Use a Different Sample Grid
**Severity: REQUIRED**

**Location:** Section 9, `preprocess_ecg()` and Section 10 downstream interface contract:

```python
# Section 9:
if fs != 500.0:
    signal = resample_to_500hz(signal, fs, 500.0)
    fs = 500.0
# ...
preprocessed[:, lead_idx] = lead_signal

# Section 10:
# 1.5 (Features — morphology): raw.signal (original mV, never modified)
```

**The flaw:**

If the source recording is at 250 Hz, `signal = resample_to_500hz(signal, ...)` produces a 500 Hz array. The `preprocessed_signal` is at 500 Hz. But `raw.signal` (the original `ECGRecord.signal`) remains at 250 Hz.

Node 1.5 (Feature Extraction) uses fiducial positions from ECGdeli output to index into the signal for morphology measurements (e.g., "T-wave amplitude = signal[Tpeak_sample_index]"). ECGdeli produces fiducial indices in the 500 Hz sample grid (because it was given the 500 Hz preprocessed signal). But morphology features are told to use `raw.signal` — which is at 250 Hz. The fiducial index `Tpeak = 1200` in a 500 Hz grid corresponds to sample 600 in the 250 Hz grid. If Node 1.5 uses `raw.signal[1200]` on a 250 Hz signal that has only 2500 samples, it may either index out of bounds or index the wrong time point.

The document says morphology must use raw signal (correct) but does not address the sample-grid mismatch when resampling has occurred.

**Required action:** When resampling is performed, `PreprocessedECGRecord` must also store a `raw_signal_500hz` field: the raw (unfiltered) signal resampled to 500 Hz. This is the correct signal for morphology measurements when the original was not at 500 Hz — it preserves the raw amplitude values (unfiltered) but aligns the sample grid with ECGdeli's fiducial indices. Clarify: Node 1.5 morphology uses `raw_signal_500hz` if resampling was performed, or `raw.signal` if source was already 500 Hz.

## Verdict

**GATE CONDITIONAL**

Node 1.2 is well-grounded. The filter design choices (Butterworth + filtfilt, 0.05 Hz cutoff, notch-first ordering) are correct and appropriately justified. The pipeline order rationale is thorough. The raw signal preservation protocol is the right design.

The CRITICAL issue (filtfilt edge transients) is a real clinical risk: for 10-second recordings with a 0.05 Hz highpass filter, beats at the start and end of the recording may have systematically wrong ST measurements. The gate opens when all three issues are resolved.
