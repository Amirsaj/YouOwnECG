## Resolution of RRC Issues

### REQUIRED-1: STAT suppression window stored server-side

**Fix applied to Node 3.3:**

The STAT alert suppression state is stored server-side in the FastAPI session store (Redis or in-memory TTL cache for MVP) keyed by `(ecg_id, session_token, finding_type)` with a 5-minute TTL:

```python
STAT_SUPPRESSION_TTL_SECONDS = 300  # 5 minutes

async def acknowledge_stat_alert(ecg_id: str, finding_type: str, session_token: str):
    key = f"stat_suppressed:{ecg_id}:{session_token}:{finding_type}"
    await cache.set(key, "1", ttl=STAT_SUPPRESSION_TTL_SECONDS)

async def is_stat_suppressed(ecg_id: str, finding_type: str, session_token: str) -> bool:
    key = f"stat_suppressed:{ecg_id}:{session_token}:{finding_type}"
    return await cache.get(key) is not None
```

The client queries suppression state on mount and after any tab focus event, preventing multi-tab re-alerting.

### REQUIRED-2: Banner state after 10-second animation

**Fix applied to Node 3.3:**

After 10 seconds without explicit acknowledgment, the STAT banner transitions to a persistent mini-strip:

```
State machine:
  FULL (0–10 s, animated) → MINI_STRIP (10 s+, no animation, 24px height) → DISMISSED (after "ACKNOWLEDGE" click)

Mini-strip content: "▐ STAT: POSSIBLE ANTERIOR STEMI  [EXPAND]"
- Vermillion background, white text, 14px
- "EXPAND" restores full banner on click
- Remains visible until explicit acknowledgment
```

The transition is implemented via CSS class swap triggered by a 10-second `setTimeout`.
