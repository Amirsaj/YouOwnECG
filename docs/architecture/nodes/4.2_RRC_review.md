## Issues Found

### REQUIRED-1: split_section_into_chunks() undersized tail merge can exceed MAX_CHUNK_TOKENS

**Location:** Node 4.2, Section 2.2

When the tail chunk is below `MIN_CHUNK_TOKENS = 150`, it is merged into the last chunk:

```python
chunks[-1]["text"] += "\n\n" + current_text.strip()
```

If the last chunk is already at `MAX_CHUNK_TOKENS = 800` and the tail is 149 tokens, the merged chunk will be 949 tokens — exceeding the maximum. This violates VT-4.2-01 (0 chunks above 800 tokens).

**Required fix:** Before merging, check that the merged size would not exceed `MAX_CHUNK_TOKENS`. If it would, keep the tail as a standalone chunk regardless of its size:

```python
if current_tokens < MIN_CHUNK_TOKENS and chunks:
    last_chunk_tokens = len(tokenizer.encode(chunks[-1]["text"]))
    if last_chunk_tokens + current_tokens <= MAX_CHUNK_TOKENS:
        chunks[-1]["text"] += "\n\n" + current_text.strip()
    else:
        chunks.append({**metadata, "text": current_text.strip()})
else:
    chunks.append({**metadata, "text": current_text.strip()})
```

### REQUIRED-2: chunk_id determinism requires stable section numbering — not guaranteed for parsed sections

**Location:** Node 4.2, Section 4 (metadata schema)

The `chunk_id` is constructed as `book_short + chapter_idx + section_idx + page + chunk_idx`. The `section_idx` is derived from parsing SECTION_HEADER blocks (Node 4.1). If the section header classifier (Node 4.1 Section 3.1) misclassifies a bold body text as a SECTION_HEADER, the section count increments incorrectly — causing all subsequent chunks in that chapter to have different `chunk_id` values on re-ingestion. This breaks deduplication.

**Required fix:** The `chunk_id` should use `section_title_hash` (first 8 chars of MD5 of the section title text) instead of `section_idx`. Section titles are stable across re-ingestions; section indices are not.

```python
import hashlib

def make_chunk_id(book_short, chapter_number, section_title, page_number, chunk_idx):
    section_hash = hashlib.md5(section_title.encode()).hexdigest()[:8]
    return f"{book_short}_ch{chapter_number:02d}_{section_hash}_p{page_number}_c{chunk_idx}"
```
