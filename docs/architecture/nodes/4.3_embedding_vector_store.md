## 1. Embedding Model Selection

### 1.1 Candidate Evaluation

| Model | Dimensions | Context (tokens) | Cost | MTEB Medical Avg |
|-------|-----------|-----------------|------|-----------------|
| text-embedding-3-small | 1536 | 8191 | $0.02/M tokens | 61.2 |
| text-embedding-3-large | 3072 | 8191 | $0.13/M tokens | 64.6 |
| BAAI/bge-large-en-v1.5 (local) | 1024 | 512 | $0 | 63.5 |
| NV-Embed-v2 (local) | 4096 | 32768 | $0 | 69.3 |

### 1.2 Decision: text-embedding-3-small with Matryoshka truncation

**Chosen:** `text-embedding-3-small` at full 1536 dimensions.

Rationale:
- ECG medical text corpus is small (~3,130 chunks) — embedding cost is one-time ~$0.04 total (trivial)
- 8,191 token context covers all chunks (max chunk = 800 tokens)
- API-based: no local GPU memory consumed (MPS stays available for the ML pipeline)
- MTEB medical score (61.2) is sufficient for clinical terminology retrieval; the domain is narrow (ECG textbooks only)

If cost scaling becomes a concern at re-ingestion, NV-Embed-v2 is the local fallback.

### 1.3 Embedding Normalization

All embeddings are L2-normalized before storage. Cosine similarity then equals dot product — enables efficient FAISS `IndexFlatIP` (inner product) instead of slower L2.

```python
import numpy as np

def normalize_embedding(embedding: list[float]) -> np.ndarray:
    v = np.array(embedding, dtype=np.float32)
    norm = np.linalg.norm(v)
    if norm == 0:
        return v
    return v / norm
```

## 2. Vector Store: FAISS

### 2.1 Index Type

`faiss.IndexFlatIP` (exact inner product search). With ~3,130 vectors:
- Exact search (no approximation) completes in < 1 ms for top-k=10 on Apple Silicon
- No index training required
- Deterministic results

Approximate indexes (IVF, HNSW) are not needed at this corpus size. They would introduce recall loss with no latency benefit.

### 2.2 Index Construction

```python
import faiss
import numpy as np

def build_faiss_index(embeddings: list[np.ndarray]) -> faiss.IndexFlatIP:
    """
    Builds exact inner-product FAISS index from L2-normalized embeddings.
    """
    dim = embeddings[0].shape[0]
    index = faiss.IndexFlatIP(dim)
    matrix = np.stack(embeddings).astype(np.float32)
    index.add(matrix)
    return index
```

### 2.3 Persistence

```python
FAISS_INDEX_PATH = "data/rag/ecg_books.faiss"

# Save
```

## 3. Query Embedding

Queries at retrieval time are embedded with the same model and normalization:

```python
import openai

async def embed_query(query: str, client: openai.AsyncOpenAI) -> np.ndarray:
    response = await client.embeddings.create(
        model="text-embedding-3-small",
        input=query,
    )
    return normalize_embedding(response.data[0].embedding)
```

Query is the agent's sub-question (e.g., "ST elevation criteria anterior STEMI"), not the full ECG feature object.

## 4. ECG Figure Image Embedding (CLIP)

ECG_STRIP figures are embedded as images for the morphology comparison feature (Node 3.4 Section 5). This is a secondary index — only queried when the cardiologist selects "Compare with textbook."

**Model:** `openai/clip-vit-large-patch14` (local, MPS)

```python
from transformers import CLIPProcessor, CLIPModel
import torch
from PIL import Image

CLIP_MODEL_NAME = "openai/clip-vit-large-patch14"

def embed_ecg_figure(image_path: str, model: CLIPModel, processor: CLIPProcessor) -> np.ndarray:
    image = Image.open(image_path).convert("RGB")
    inputs = processor(images=image, return_tensors="pt").to("mps")
    with torch.no_grad():
        features = model.get_image_features(**inputs)
    embedding = features.squeeze().cpu().numpy().astype(np.float32)
    return normalize_embedding(embedding)
```

A separate FAISS index is built for ECG figures only (~400–600 vectors). Queried with CLIP text embeddings of clinical descriptions (e.g., "anterior STEMI hyperacute T waves").

## 5. Ingestion Pipeline Integration

```python
async def ingest_all_books(
    books_dir: str,
    openai_client: openai.AsyncOpenAI,
    batch_size: int = 100,
) -> tuple[faiss.IndexFlatIP, list[dict]]:
    """
    Full ingestion pipeline: extract → chunk → embed → index.
    Returns FAISS index and metadata list.
    """
    all_chunks = []

    for pdf_path in glob.glob(f"{books_dir}/*.pdf"):
        pages = extract_book(pdf_path)
        # Node 4.1: classify, associate figures
        text_chunks = build_text_chunks(pages)   # Node 4.2
        figure_chunks = build_figure_chunks(pages, pdf_path)  # Node 4.2
        all_chunks.extend(text_chunks + figure_chunks)

    # Batch embed (rate-limit aware)
    texts = [c["text"] for c in all_chunks]
    embeddings = []
    for i in range(0, len(texts), batch_size):
        batch = texts[i : i + batch_size]
        response = await openai_client.embeddings.create(
            model="text-embedding-3-small",
            input=batch,
        )
        embeddings.extend([normalize_embedding(r.embedding) for r in response.data])

    index = build_faiss_index(embeddings)
    return index, all_chunks
```
