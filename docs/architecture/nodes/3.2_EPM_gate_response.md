## Resolution of RRC Issues

### REQUIRED-1: EXPLANATION_TEMPLATES None-safety

**Fix applied to Node 3.2:**

All numeric substitutions in EXPLANATION_TEMPLATES now use a guard helper:

```python
def _fmt(value, unit: str = "", precision: int = 2) -> str:
    if value is None:
        return f"N/A{' ' + unit if unit else ''} (measurement unavailable — signal quality insufficient)"
    return f"{value:.{precision}f}{' ' + unit if unit else ''}"

# Example usage in template:
f"ST elevation of {_fmt(st_elevation_mv, 'mV')} detected in lead {lead}"
```

This applies to ALL numeric substitutions in all disease dashboard templates.

### REQUIRED-2: VL2 unavailability fallback — three states

**Fix applied to Node 3.2 and Node 1.6:**

`VisionVerificationResult` (Node 1.6) is amended to carry an absence reason:

```python
@dataclass
class VisionVerificationResult:
    # ... existing fields ...
    available: bool = True
    unavailability_reason: str | None = None
    # Values: None (available) / "VL2_TIMEOUT" / "VL2_NOT_APPLICABLE" / "VL2_ERROR"
```

Disease dashboard visual confirmation component renders:

```typescript
const VL2Panel = ({ result }: { result: VisionVerificationResult }) => {
    if (!result.available) {
        const messages = {
            VL2_TIMEOUT: "Visual confirmation timed out (> 15 s). Signal features used only.",
            VL2_NOT_APPLICABLE: "Visual confirmation not applicable for this finding type.",
            VL2_ERROR: "Visual confirmation failed (error). Signal features used only.",
        };
        return <InfoPanel>{messages[result.unavailability_reason] ?? "Visual confirmation unavailable."}</InfoPanel>;
    }
    // ... normal rendering ...
};
```
