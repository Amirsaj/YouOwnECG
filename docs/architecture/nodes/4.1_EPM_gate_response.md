### REQUIRED-1: Data-driven column split detection

**Fix applied to Node 4.1:**

`sort_blocks_by_reading_order()` now uses KMeans clustering to detect the actual column split:

```python
from sklearn.cluster import KMeans
import numpy as np

def sort_blocks_by_reading_order(text_blocks: list[dict], page_width: float) -> list[dict]:
    if len(text_blocks) < 4:
        return sorted(text_blocks, key=lambda b: b["bbox"][1])

    left_edges = np.array([b["bbox"][0] for b in text_blocks]).reshape(-1, 1)
    km = KMeans(n_clusters=2, n_init=10, random_state=0).fit(left_edges)
    centers = sorted(km.cluster_centers_.flatten())
    gap = centers[1] - centers[0]

    if gap < 30:  # single column
        return sorted(text_blocks, key=lambda b: b["bbox"][1])

    split_x = (centers[0] + centers[1]) / 2
    left_blocks = [b for b in text_blocks if b["bbox"][0] < split_x]
    right_blocks = [b for b in text_blocks if b["bbox"][0] >= split_x]
    return (sorted(left_blocks, key=lambda b: b["bbox"][1]) +
            sorted(right_blocks, key=lambda b: b["bbox"][1]))
```

This handles asymmetric column layouts (Chou's narrow margin column) correctly.
