## Executive Summary

Node 2.5 solves the context window problem for multi-beat, multi-lead ECG data. The Node 1.5 feature object contains up to N beats × 12 leads × ~50 features/beat/lead = potentially 6,000+ data points per ECG. No LLM context window can receive all of this as raw data without saturation, and even if it could, the model would fail to attend to sparse anomalies buried in a wall of numbers.

This node specifies:
1. Which conditions require per-beat detail vs aggregate statistics
2. How to compress per-beat data into agent-digestible summaries without losing clinically critical information
3. How to structure lead-by-lead data for territory analysis
4. How to detect and surface clinically significant beat-to-beat trends

The output of this node is a **beat summary schema** — a compact, agent-ready representation of the beat-level feature object that replaces the raw beat array in agent prompts.

## 1. When Does Beat-by-Beat Detail Matter?

### 1.1 Conditions That Require Individual Beat Analysis

These conditions are INVISIBLE in aggregate statistics and can only be diagnosed by examining individual beats:

| Condition | Why beats matter | Minimum beats needed |
|-----------|-----------------|---------------------|
| Wenckebach (Mobitz I 2° AVB) | Progressive PR lengthening over 3–5 beats, then dropped beat | 4 consecutive beats around the dropped beat |
| Mobitz II 2° AVB | Fixed PR, sudden dropped beat | The beat before and after the dropped beat |
| 3° AV block (complete) | P-P intervals regular, R-R intervals regular but DIFFERENT rates | Min 3 P waves and 3 QRS complexes |
| Bigeminy / Trigeminy | PVC every 2nd or 3rd beat — pattern only visible over multiple beats | 6 consecutive beats |
| Electrical alternans | QRS amplitude alternating every other beat | 5+ beat pairs (10 beats min) |
| T-wave alternans | T amplitude alternating every other beat | 5+ beat pairs |
| Intermittent pre-excitation | Delta wave appears on some beats, absent on others (intermittent WPW) | Needs per-beat delta wave flag |
| Rate-dependent BBB | QRS widens at faster heart rate only | Need per-beat QRS duration + instantaneous rate |
| Evolving STEMI (dynamic ST) | ST elevation increasing beat-to-beat | ST trend across beats |
| Concealed bigeminy | Dropped beat after every PVC — P wave falls in T wave | Context around each PVC |

### 1.2 Conditions Where Aggregate Statistics Are Sufficient

These conditions produce a stable pattern across all beats — aggregate statistics capture them fully:

- Sinus rhythm, sinus tachycardia, sinus bradycardia
- LBBB, RBBB, LAFB, LPFB (morphology consistent across all beats)
- LVH, RVH (voltage criteria consistent)
- Stable STEMI (threshold met on most beats, minor beat-to-beat variation)
- AFib (diagnosed by rhythm irregularity metric, absent P waves — no per-beat detail needed beyond what aggregate provides)
- QTc prolongation (aggregate mean/max QTc is the diagnostic criterion)
- Pericarditis (diffuse concave ST pattern — stable across beats)
- P-wave morphology findings (LAE, RAE — stable across beats)

### 1.3 Decision Rule for Beat Presentation

```python
NEEDS_PER_BEAT_DETAIL: dict[str, list[str]] = {
    # RRC agent conditions needing per-beat
    "wenckebach":         ["pr_interval_ms", "beat_class"],
    "mobitz_ii":          ["pr_interval_ms", "beat_class"],
    "complete_avb":       ["pr_interval_ms", "rr_interval_ms", "beat_class"],
    "bigeminy":           ["beat_class", "rr_interval_ms"],
    "trigeminy":          ["beat_class", "rr_interval_ms"],
    "intermittent_wpw":   ["pr_interval_ms", "delta_wave_present"],

    # IT agent conditions

```

When a condition in `NEEDS_PER_BEAT_DETAIL` is suspected (based on aggregate flags in the feature object), the relevant per-beat fields are included in the agent prompt. When not suspected, only the beat summary is provided.

## 2. Beat Summary Schema

The beat summary replaces the raw `beats[N]` array in agent prompts. It is computed once after Node 1.5 and passed to all agents.

### 2.1 Beat Summary Structure

```python
@dataclass
class BeatSummary:
    n_beats: int
    n_beats_in_recording: int
    n_normal_beats: int          # beat_class == 0
    n_pvc: int                   # beat_class == 1
    n_pac: int                   # beat_class == 2
    n_aberrant: int              # beat_class == 3
    n_paced: int                 # beat_class == 4
    ectopic_fraction: float      # (pvc + pac + aberrant) / n_beats

    # Interval aggregate statistics (from aggregate_beat_features())
    pr_mean_ms: float | None
    pr_std_ms: float | None
    pr_trend_ms_per_beat: float | None     # positive = lengthening
    pr_sequence: list[float | None]        # ALL per-beat values (for Wenckebach detection)

    qrs_mean_ms: float
    qrs_std_ms: float

    qt_mean_ms: float | None
    qtc_bazett_mean_ms: float | None
    qtc_bazett_max_ms: float | None        # worst-case QTc

    # RR intervals
    rr_mean_ms: float
    rr_std_ms: float
    rr_sequence: list[float]              # ALL per-beat RR values (for AV block patterns)

    # ST aggregate per lead — compact
    st_by_lead: dict[str, STLeadSummary]  # one entry per lead

    # Beat-class sequence (compact)
    beat_class_sequence: list[int]        # e.g., [0,0,1,0,0,1,0,0,1] for bigeminy
    beat_class_pattern: str | None        # detected pattern: "BIGEMINY" / "TRIGEMINY" / "REGULAR" / "IRREGULAR" / None

    # Voltage per lead — compact
    voltage_by_lead: dict[str, VoltageSummary]  # one entry per lead

    # Trend flags (pre-computed, not raw data)
    pr_progressive_lengthening: bool      # True if trend > 2 ms/beat over >= 3 beats
    st_progressive_change: dict[str, bool]  # per lead: True if ST trend != 0 significantly
    alternans_suspected: dict[str, bool]  # per lead: even/odd beat difference > 0.1 mV

    # Selected beats for special cases
    exemplar_normal_beat_idx: int         # index of most representative normal beat
    most_abnormal_beat_idx: int | None    # index of the beat with most extreme findings
    dropped_beat_context: list[dict] | None  # populated if beat_class_sequence has gaps


@dataclass
class STLeadSummary:
    lead: str
    mean_mv: float
    max_mv: float
    min_mv: float
    std_mv: float
    dominant_morphology: str             # most common ST morphology across beats
    above_threshold_fraction: float      # fraction of beats exceeding STEMI threshold


@dataclass
class VoltageSummary:
    lead: str
    r_mean_mv: float
    s_mean_mv: float
    t_mean_mv: float
    p_mean_mv: float | None
    r_std_mv: float
    dominant_t_morphology: str           # most common T morphology across beats
```

### 2.2 Beat Summary Computation

```python
def compute_beat_summary(features: FeatureObject) -> BeatSummary:
    """
    Computes the compact BeatSummary from the full Node 1.5 feature object.
    Called once after Node 1.5, before any agent dispatch.
    """
    beats = features.beats

    # Beat class sequence and pattern detection
    beat_classes = [b.beat_class for b in beats]
    pattern = detect_beat_class_pattern(beat_classes)

    # PR sequence: include all values including None (dropped beats appear as gaps)
    pr_sequence = [b.pr_interval_ms for b in beats]

    # RR sequence: full sequence for AV block pattern detection
    rr_sequence = [b.rr_interval_ms for b in beats if b.rr_interval_ms is not None]

    # Pre-compute trend flags
    pr_trend = features.aggregate.pr_interval_trend_ms_per_beat
    pr_lengthening = (pr_trend is not None and pr_trend > 2.0
                      and _consecutive_increasing(pr_sequence, min_run=3))

    # ST progressive change per lead
    st_progressive = {
        lead: abs(features.aggregate.leads[lead].st_elevation_mean_mv) > 0.05
              and features.aggregate.leads[lead].st_trend_mv_per_beat is not None
              and abs(features.aggregate.leads[lead].st_trend_mv_per_beat) > 0.01
        for lead in ALL_LEADS
    }

    # Alternans detection (pre-computed in Node 1.5; surfaced here explicitly)
    alternans = {
        lead: features.aggregate.electrical_alternans_present
              or features.aggregate.t_wave_alternans_present
        for lead in ALL_LEADS
    }

    # Dropped beat context: beats with no PR interval in a normally-conducting rhythm
    dropped = None
    if not features.aggregate.afib_suspected:
        dropped = _extract_dropped_beat_context(beats, pr_sequence, rr_sequence)

    return BeatSummary(
        n_beats=features.total_beats_analyzed,
        n_beats_in_recording=features.total_beats_in_recording,
        n_normal_beats=sum(1 for c in beat_classes if c == 0),
        n_pvc=features.aggregate.pvc_count,
        n_pac=features.aggregate.pac_count,
        # ... all fields as above ...
        beat_class_sequence=beat_classes,
        beat_class_pattern=pattern,
        pr_sequence=pr_sequence,
        rr_sequence=rr_sequence,
        pr_progressive_lengthening=pr_lengthening,
        st_progressive_change=st_progressive,
        alternans_suspected=alternans,
        dropped_beat_context=dropped,
    )
```

## 3. Beat Class Pattern Detection

```python
def detect_beat_class_pattern(classes: list[int]) -> str | None:
    """
    Detects regular beat-class patterns from the sequence.
    Returns pattern name or None if no pattern detected.

    Uses sliding window scan (not stride-2 slicing) to detect patterns at any
    offset within the recording. Requires >= 4 consecutive bigeminal pairs to
    avoid false positives from isolated ectopy at the start/end of the strip.
    """
    if len(classes) < 4:
        return None
    if all(c == 0 for c in classes):
        return "ALL_NORMAL"
    if all(c == 4 for c in classes):
        return "ALL_PACED"

    # Bigeminy: [normal, ectopic] repeating for >= 4 consecutive pairs
    # Slide through all positions — do NOT use stride-2 slicing
    for pvc_class in (1, 2):
        best_run = 0
        run = 0
        for i in range(len(classes) - 1):
            if classes[i] == 0 and classes[i + 1] == pvc_class:
                run += 1
                best_run = max(best_run, run)
            else:
                run = 0
        if best_run >= 4:
            return "BIGEMINY_PVC" if pvc_class == 1 else "BIGEMINY_PAC"

    # Trigeminy: [normal, normal, ectopic] pattern appearing >= 2 times
    count = 0
    for i in range(len(classes) - 2):
        if classes[i] == 0 and classes[i + 1] == 0 and classes[i + 2] == 1:
            count += 1
    if count >= 2:
        return "TRIGEMINY_PVC"

    ectopic_fraction = sum(1 for c in classes if c != 0) / len(classes)
    if ectopic_fraction < 0.1:
        return "OCCASIONAL_ECTOPY"
    if ectopic_fraction > 0.3:
        return "FREQUENT_ECTOPY"

    return None
```

### 4.1 Lead Groups for Territory Analysis

LEAD_TERRITORIES = {

### 4.2 Reciprocal Change Detection

```python
RECIPROCAL_MAP = {
    "inferior":    ["high_lateral", "anterior"],   # inferior STEMI → anterior/lateral depression
    "anterior":    ["inferior"],                   # anterior STEMI → inferior reciprocal
    "high_lateral": ["inferior"],                  # lateral STEMI → inferior reciprocal
    "posterior":   ["anterior"],                   # posterior STEMI → anterior ST depression
}

def detect_reciprocal_changes(
    st_by_lead: dict[str, STLeadSummary],
    primary_territory: str,
    threshold_mv: float = 0.05,
) -> dict[str, bool]:
    """
    For a given primary territory of elevation, checks if the expected
    reciprocal territories show ST depression.
    Returns dict of territory → reciprocal_change_present.
    """
    expected_reciprocal_territories = RECIPROCAL_MAP.get(primary_territory, [])
    results = {}
    for territory in expected_reciprocal_territories:
        leads_in_territory = LEAD_TERRITORIES[territory]
        n_depressed = sum(
            1 for l in leads_in_territory
            if st_by_lead.get(l) and st_by_lead[l].mean_mv <= -threshold_mv
        )
        results[territory] = (n_depressed >= 1)
    return results
```

### 4.3 Single-Lead Anomaly Detection

Some findings are significant precisely BECAUSE they are isolated to one lead:

```python
def detect_single_lead_anomaly(
    leads_flagged: list[str],
    territory_map: dict[str, list[str]],
) -> dict:
    """
    Returns whether a set of flagged leads represents an isolated finding
    (one lead, surrounded by normal leads in the same territory).
    Isolated findings may indicate:
    - Lead misplacement (non-anatomical pattern)
    - V4R finding (RV STEMI)
    - Technical artifact in one lead
    """
    if len(leads_flagged) == 1:
        lead = leads_flagged[0]
        for territory, members in territory_map.items():
            if lead in members and len(members) >= 3:
                n_normal = sum(1 for l in members if l not in leads_flagged)
                if n_normal >= 2:
                    return {
                        "isolated": True,
                        "lead": lead,
                        "territory": territory,
                        "consider_artifact": True,
                        "consider_misplacement": True,
                    }
    return {"isolated": False}
```

## 5. Dropped Beat Context Extraction

For AV block pattern detection, the context around dropped beats is extracted as a structured window:

```python
def _extract_dropped_beat_context(
    beats: list[Beat],
    pr_sequence: list[float | None],
    rr_sequence: list[float],
) -> list[dict] | None:
    """
    A "dropped beat" is inferred when:
    - A beat has beat_class == 0 (normal QRS) but PR == None (no preceding P wave found)
    - OR an RR interval is ≥ 1.8 × mean RR (suggesting a missing QRS)

    Returns a list of context windows (up to 3 beats before and after the gap).
    """
    contexts = []
    mean_rr = np.mean(rr_sequence) if rr_sequence else 0

    for i, beat in enumerate(beats):
        # Detect long pause (possible dropped QRS)
        if beat.rr_interval_ms is not None and beat.rr_interval_ms >= 1.8 * mean_rr:
            start = max(0, i - 3)
            end = min(len(beats), i + 3)
            contexts.append({
                "drop_type": "LONG_PAUSE",
                "beat_index": i,
                "rr_at_pause_ms": beat.rr_interval_ms,
                "context_pr_sequence": pr_sequence[start:end],
                "context_rr_sequence": [b.rr_interval_ms for b in beats[start:end]],
                "context_beat_classes": [b.beat_class for b in beats[start:end]],
            })

    return contexts if contexts else None
```

### 6.1 What Each Agent Receives

### 6.2 Token Budget Management

At 500 Hz, a standard 10-second ECG produces 10 beats × 12 leads × ~50 features = ~6,000 data points. The BeatSummary compresses this to:

| Component | Approx tokens |
|-----------|-------------|
| BeatSummary scalar fields | ~80 tokens |
| beat_class_sequence (10 integers) | ~20 tokens |
| pr_sequence (10 floats) | ~30 tokens |
| rr_sequence (10 floats) | ~30 tokens |
| st_by_lead (12 leads × 5 fields) | ~180 tokens |
| voltage_by_lead (12 leads × 5 fields) | ~180 tokens |
| dropped_beat_context (if any, 3 beats) | ~80 tokens |
| **Total BeatSummary** | **~600 tokens** |

Raw beat array equivalent: 10 × 12 × 50 = 6,000 numbers ≈ **~10,000+ tokens**.

Compression ratio: ~17×, with no loss of clinically critical information (all beat-level features are either aggregated, pattern-detected, or included as detail when a specific condition is suspected).

### 6.3 Detail Beat Injection

When a condition requiring per-beat detail is suspected, the relevant fields are injected after the BeatSummary:

```python
def inject_detail_beats(
    features: FeatureObject,
    suspected_conditions: list[str],
) -> dict:
    """
    For each condition in NEEDS_PER_BEAT_DETAIL that is suspected,
    returns the relevant per-beat field sequences.
    """
    detail = {}
    for condition in suspected_conditions:
        needed_fields = NEEDS_PER_BEAT_DETAIL.get(condition, [])
        for field in needed_fields:
            if field not in detail:
                detail[field] = _extract_beat_field(features.beats, field)
    return detail
```

The suspected conditions list is derived from the aggregate feature flags BEFORE agent dispatch:
- `aggregate.afib_suspected = True` → RRC agent gets rhythm detail
- `aggregate.pr_interval_trend_ms_per_beat > 2.0` → RRC agent gets `pr_sequence` + `rr_sequence`
- `aggregate.st_progressive_change` in any lead → IT agent gets per-beat ST per-lead
- `aggregate.electrical_alternans_present = True` → MR agent gets per-beat R amplitude
- etc.
