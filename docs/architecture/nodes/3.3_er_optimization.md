## 1. Cognitive Load Theory for Medical Dashboards

**Cognitive load theory** (Sweller, 1988): working memory can process ~4 chunks of information simultaneously. Medical dashboards that present > 4 categories of information above the fold demand serial attention, increasing error rate under stress.

**ER clinical workflow (ECG reading context):**
- Available attention time for initial ECG review: 15–90 seconds for triage nurses; 2–5 minutes for physicians in non-STAT situations; < 10 seconds in STAT situations
- Primary question in STAT: "Is this immediately life-threatening?" — answered by a SINGLE visual element
- Secondary: "What is it?" — answered by the finding name
- Tertiary: "What do I do?" — answered by the action panel

**Information hierarchy for YouOwnECG:**

```
Tier 1 (0–3 sec, pre-attentive):  STAT/CRITICAL/NORMAL status badge
Tier 2 (3–15 sec, attentive):     Finding list with confidence badges
Tier 3 (15–60 sec, analytical):   ECG visualization with highlights
Tier 4 (> 60 sec, expert):        Deep analysis, reasoning chain
```

All Tier 1 information must be above the fold, visually dominant, and perceivable without reading text.

## 2. Pre-Attentive Processing

Visual features perceived in < 200 ms without focused attention (Healey & Enns, 2012):
- **Color** (hue and intensity)
- **Size** (larger = more important)
- **Motion** (blinking/pulsing draws attention immediately)
- **Orientation** (lines at unusual angles stand out)
- **Enclosure** (borders/boxes group items)

**Application to STAT alerts:**
STAT alerts use all four pre-attentive channels simultaneously:
- Color: red (#CC0000 on white)
- Size: STAT badge is the largest text element on the page (72px+ font)
- Motion: 1 Hz pulse animation (border opacity oscillates 0.7 → 1.0) for the first 10 seconds
- Enclosure: thick red border around the entire viewport

After the first 10 seconds, the pulse animation stops (alert acknowledged visually). Continuous animation causes adaptation and ceases to attract attention after ~30 seconds.

## 4. Traffic-Light System with Redundant Encoding

Critical medical information is NEVER communicated through color alone (8% of males have color vision deficiency).

| Level | Color | Symbol | Text label | Sound |
|-------|-------|--------|-----------|-------|
| STAT | Red #CC0000 | ⬛ filled square + "!" | "STAT" | Alert tone (opt-in) |
| CRITICAL | Orange #E06000 | ▲ triangle | "CRITICAL" | — |
| ABNORMAL | Yellow #C0A000 | ● circle | "ABNORMAL" | — |
| NORMAL | Green #007700 | ✓ checkmark | "NORMAL" | — |
| INSUFFICIENT | Gray #666666 | ? question | "INSUFFICIENT DATA" | — |

**Redundant encoding:** Every status communication uses color + shape + text. The symbols are chosen to be distinct even in grayscale.

## 5. Accessible Color Palette

**Okabe-Ito palette** (designed for deuteranopia + protanopia safety):

```
Black:       #000000  (ECG trace, text)
Orange:      #E69F00  (ST elevation, warnings)
Sky Blue:    #56B4E9  (ST depression)
Green:       #009E73  (normal findings)
Yellow:      #F0E442  (borderline)
Blue:        #0072B2  (information)
Vermillion:  #D55E00  (STAT alerts — distinguishable from green even in red-green blindness)
Pink:        #CC79A7  (T-wave inversion)
```

**Do NOT use:**
- Red + green together without symbol/shape backup
- Green for normal findings if red/green colorblindness is present (use blue-green #009E73 instead)
- Yellow text on white background (insufficient contrast)

**WCAG 2.1 AAA requirements:**
- Normal text: contrast ratio ≥ 7:1
- Large text (≥ 18pt): ≥ 4.5:1
- All text on all backgrounds meets these requirements in the specified palette

## 6. Touch and Motor Accessibility

- All interactive targets: minimum 44×44 px (WCAG 2.5.5 AAA). ER nurses use gloved hands — prefer 48×48 px
- No hover-only interactions (tooltips must be accessible via focus/tap)
- Keyboard navigation: Tab through all findings, Enter to open detail, Escape to close
- STAT acknowledge button: extra-large (80×80 px), requires intentional click/tap (not accidental)
- Screen reader: all dynamic content updates announced via ARIA live regions

## 7. STAT Alert Design

### 7.1 Alert Appearance

```
┌══════════════════════════════════════════════════════╗
║  ⚠ POSSIBLE ANTERIOR STEMI — HIGH CONFIDENCE        ║
║  ST elevation detected in V1–V4                      ║
║  CLINICAL CORRELATION REQUIRED · AI-GENERATED        ║
║  [ACKNOWLEDGE]                  [VIEW DETAILS]       ║
╚══════════════════════════════════════════════════════╝
```

Red border, white interior. Font size: 18px for heading, 14px for subtext.

### 7.2 Alert Suppression Window

After a STAT alert is acknowledged:
- The alert banner collapses but remains accessible as a "dismissed alerts" count
- Suppression window: 5 minutes — if the same condition is re-flagged within 5 minutes on the same ECG, the alert does NOT re-fire (prevents rapid-fire re-acknowledgment)
- If suppression window expires and the condition persists: re-fire with a different message: "RE-ALERT: [condition] still detected"
- Cross-ECG suppression: 0 — a new ECG upload always produces fresh alerts

### 7.3 Alert Fatigue Prevention

Only STAT-level findings trigger the full alert. CRITICAL and below use the standard badge system in the findings list — no modal, no animation.

- Number of STAT conditions defined (from Node 2.3): STEMI, Wellens, de Winter, VT/VF, complete AVB, hyperK sine wave, Brugada Type 1, long QT with TdP risk
- These 8 conditions plus any STAT-downgraded finding at > 0.80 confidence trigger alerts
