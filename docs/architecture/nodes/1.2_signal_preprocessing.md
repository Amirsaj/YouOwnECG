## Executive Summary

Node 1.2 defines the signal preprocessing pipeline: the sequence of DSP operations applied to the ingested ECG signal before fiducial detection and feature extraction.

Four constraints drive every decision:

1. **Raw signal must be preserved.** Preprocessing alters the signal — baseline wander removal shifts the isoelectric line, bandpass filtering affects ST morphology. Morphology features (Node 1.5) must use the raw signal. The preprocessing pipeline produces a processed copy; the `ECGRecord.signal` (raw mV) is never overwritten.

2. **Pipeline order is not arbitrary.** Applying a bandpass filter before baseline wander removal may cause the baseline-wander energy to fold into the passband. The correct order — and the physiological justification for it — is specified in Section 6.

3. **ECGdeli operates on the preprocessed signal.** Fiducial detection (QRS, P-wave, T-wave) is more accurate on a denoised signal. However, if preprocessing is too aggressive, it will distort the very features ECGdeli is trying to locate. The pipeline must clean without distorting.

4. **Monitoring-mode vs diagnostic-mode.** Hospital bedside monitors apply a 0.5–40 Hz bandpass (monitoring mode) that distorts ST segments. If the ingested recording is in monitoring mode, Node 1.3 (Quality Assessment) must flag it — ST-based diagnosis (STEMI) on a monitoring-mode ECG is unreliable. This node specifies diagnostic-mode preprocessing only.

## 1. First Principles: Noise Sources in the ECG

### 1.1 Powerline Interference (50/60 Hz)

**Physical origin:** The human body acts as an antenna in the electromagnetic field radiated by AC power lines. The coupling mechanism is predominantly capacitive — the body-to-power-line capacitance (≈1–10 pF) forms a current path that drives a common-mode voltage onto all recording electrodes. The instrumentation amplifier rejects the common-mode component (CMRR >100 dB ideally), but any electrode impedance imbalance converts common-mode to differential mode — and differential-mode 50/60 Hz noise cannot be rejected by the amplifier.

**Frequency characteristics:**
- Fundamental: 50 Hz (Europe, Asia) or 60 Hz (Americas)
- Harmonics: 100/120 Hz, 150/180 Hz, 200/240 Hz (decreasing amplitude)
- Bandwidth: essentially a line frequency — a near-perfect sinusoid, very narrow spectral peak

**Clinical significance of residual powerline noise:**
- Amplitude: typically 50–500 μV peak (0.05–0.5 mV)
- Comparison: 1 mm ST elevation = 0.1 mV → residual 60 Hz noise of 0.5 mV can completely obscure ST changes
- For P-wave detection: P-wave amplitude ≈ 0.1–0.25 mV → 60 Hz noise is SNR-limiting

**Removal requirement:** Attenuate 50/60 Hz (and harmonics up to at least 150/180 Hz) by ≥40 dB. Must not distort frequencies below 40 Hz.

### 1.2 Baseline Wander (0.05–0.5 Hz)

**Physical origin:** Three primary sources:

1. **Respiration** (0.2–0.4 Hz): thoracic motion changes electrode-skin contact impedance cyclically; also shifts the heart's position in the chest, altering the projection of the cardiac vector onto each lead. This produces a slow, rhythmic undulation of the ECG baseline synchronized with breathing.

2. **Electrode drift** (0.05–0.2 Hz): the electrochemical potential at the skin-electrode interface changes over time as the electrolyte gel dries or the silver chloride electrode equilibrates.

3. **Patient motion** (<0.5 Hz): large-scale body movements modulate the electrode positions.

**Frequency characteristics:**
- Dominant energy: 0.05–0.5 Hz (below the P-wave spectral content of ~0.5–10 Hz)
- Can have amplitude of several millivolts — far exceeding the ECG signal itself in extreme cases

**Clinical significance of uncorrected baseline wander:**
- Shifts the isoelectric line, making ST-segment measurements relative to the wrong zero reference
- Causes P-wave detection failures (wave appears above or below the detection threshold)
- Distorts QRS onset/offset detection

**Critical preservation requirement:** The PR segment is used as the isoelectric reference for ST measurement. Aggressive baseline removal (e.g., a highpass filter with fc = 0.5 Hz) can remove the slow respiration component but may also shift the PR segment relative to adjacent ST segments, creating **pseudo-ST elevation or depression**. Filter cutoff selection is a clinical safety issue, not just a DSP parameter choice.

### 1.3 Electromyographic (EMG) Artifacts (5–500 Hz)

**Physical origin:** Skeletal muscle electrical activity. Voluntary movement, tremor (Parkinson's, anxiety), or respiration-related chest wall muscle activity. Characteristically broadband, appearing as a "fuzz" overlaid on the ECG.

**Frequency characteristics:**
- Broadband: 5–500 Hz (overlaps the QRS spectrum 5–40 Hz)
- Cannot be fully removed without distorting QRS morphology
- Detection via variance analysis (local variance >> global variance)

**Handling strategy:** EMG artifacts are detected and **flagged**, not removed. Removal would require adaptive filtering with a reference EMG channel (not available in standard 12-lead recordings). Flagged EMG segments are excluded from beat-level feature computation at Node 1.3/1.5.

### 1.4 Motion Artifacts and Electrode Pop

**Motion artifacts:** High-amplitude transients from sudden electrode movement. Characteristic: sharp, large-amplitude spike lasting 10–100 ms, often exceeding the ADC full-scale range (clipping). Already detected at Node 1.1 (clipping pre-check).

**Electrode pop:** Brief disconnection/reconnection of an electrode produces a characteristic exponential-decay transient (RC charging of the electrode-skin interface). Duration: 100 ms–2 s. Amplitude: can reach several mV.

**Handling strategy:** Same as EMG — detected and flagged at Node 1.3. Not removed — removal would require interpolation that could create fictitious ECG morphology.

## 2. ECG Spectral Content — What Must Be Preserved

Understanding the spectral content of the ECG determines which frequency bands the preprocessing filters must preserve:

| Component | Frequency range | Clinical use |
|-----------|----------------|--------------|
| P wave | 0.5–10 Hz | PR interval, P morphology, atrial rhythms |
| QRS complex | 5–40 Hz | QRS duration, BBB, axis, R-wave amplitude |
| ST segment | DC–0.5 Hz | STEMI, ischemia — **most sensitive to filter distortion** |
| T wave | 0.5–7 Hz | QT interval, T morphology, Wellens, de Winter |
| U wave | 0.1–1 Hz | Hypokalemia, long QT |
| Baseline wander | 0.05–0.5 Hz | Noise — remove |
| Powerline noise | 50/60 Hz, harmonics | Noise — remove |
| EMG/motion | 40–500 Hz, broadband | Noise — flag |

**Key implication:** The ST segment extends down to near-DC (0 Hz). Any highpass filter with cutoff above ~0.05 Hz will attenuate the ST segment. The AHA recommends a highpass cutoff of **0.05 Hz** for diagnostic-mode ECG recording to preserve ST information. Filters with cutoff ≥ 0.5 Hz (common in monitoring mode) cause clinically significant ST distortion and must not be used in this pipeline.

## 3. Powerline Interference Removal

### 3.1 Notch Filter Design

A notch filter is a bandstop filter with a very narrow rejection band centered on the interference frequency. For ECG:

**IIR Notch filter (recommended):**

```python
from scipy.signal import iirnotch, filtfilt
import numpy as np

def apply_notch_filter(signal_1d: np.ndarray,
                       fs: float,
                       freqs_hz: list[float] | None = None,
                       Q: float = 30.0) -> np.ndarray:
    """
    Applies IIR notch filters at powerline fundamental + harmonics.

    Q factor: quality factor = f0 / bandwidth. Q=30 gives a 3dB bandwidth
    of f0/30 (2 Hz at 60 Hz). Higher Q = narrower notch = less signal
    distortion but less robust to frequency drift.

    Uses filtfilt (zero-phase) to avoid group delay distortion.
    filtfilt applies the filter forward and backward, doubling the
    effective filter order and achieving exactly zero phase shift.

    Default frequencies: 50 Hz + harmonics if 50 Hz detected,
    or 60 Hz + harmonics if 60 Hz detected (autodetected from spectrum).
    """
    if freqs_hz is None:
        freqs_hz = _detect_powerline_frequencies(signal_1d, fs)

    result = signal_1d.copy()
    for f0 in freqs_hz:
        if f0 >= fs / 2:
            continue  # above Nyquist — skip
        b, a = iirnotch(f0, Q, fs)
        result = filtfilt(b, a, result)
    return result


def _detect_powerline_frequencies(signal_1d: np.ndarray,
                                  fs: float) -> list[float]:
    """
    Detects dominant powerline frequency (50 vs 60 Hz) from signal spectrum.
    Returns list of fundamental + harmonics up to fs/2.
    """
    fft = np.abs(np.fft.rfft(signal_1d))
    freqs = np.fft.rfftfreq(len(signal_1d), d=1/fs)

    # Check power in narrow bands around 50 and 60 Hz
    def band_power(f_center, width=2.0):
        mask = (freqs >= f_center - width) & (freqs <= f_center + width)
        return np.sum(fft[mask] ** 2)

    p50 = band_power(50.0)
    p60 = band_power(60.0)
    fundamental = 50.0 if p50 > p60 else 60.0

    harmonics = []
    f = fundamental
    while f < fs / 2:
        harmonics.append(f)
        f += fundamental
    return harmonics
```

**Q factor selection rationale:**
- Q = 30: 3dB bandwidth = 2 Hz at 60 Hz. Rejects powerline interference robustly even with ±1 Hz frequency drift. Narrow enough to preserve ECG content.
- Q < 10: Too wide — attenuates ECG content near 50/60 Hz (affects the high end of QRS spectrum).
- Q > 50: Very narrow — works only if the power line frequency is very stable. May miss interference if the mains frequency drifts (common in developing-country power grids).

**IIR vs FIR for notch:**

| Property | IIR Notch | FIR Notch |
|----------|-----------|-----------|
| Filter order (for same notch depth) | 2 (biquad) | 100–1000 |
| Computational cost | Very low | High |
| Phase distortion | Yes (but corrected by filtfilt) | No (linear phase) |
| Stability | Must check pole locations | Always stable |
| Group delay | Variable (corrected by filtfilt) | Constant |

**Decision: IIR with filtfilt.** The zero-phase property of filtfilt eliminates the phase distortion concern. The computational cost advantage of IIR is significant for long recordings. FIR is preferred for real-time/streaming applications where filtfilt is not applicable — not our use case (offline analysis of complete recordings).

**Note on harmonics:** Apply notch filters at each harmonic separately (50, 100, 150 Hz or 60, 120, 180 Hz) rather than a single wideband notch. Each harmonic has its own amplitude and phase; independent filters give better rejection with less passband distortion.

## 4. Baseline Wander Removal

### 4.1 Highpass Filter Approach

A highpass filter with cutoff below the ECG's lowest-frequency component (P wave at ~0.5 Hz) and above the baseline wander frequencies:

```python
from scipy.signal import butter, filtfilt
import math

def apply_highpass_filter(signal_1d: np.ndarray,
                          fs: float,
                          cutoff_hz: float = 0.05) -> np.ndarray:
    """
    Removes baseline wander using a zero-phase Butterworth highpass filter.

    Cutoff 0.05 Hz: AHA recommendation for diagnostic-mode ECG.
    This preserves U waves and the ST segment while removing slow wander.

    WARNING: Do NOT use cutoff >= 0.5 Hz for diagnostic ECG.
    At 0.5 Hz, the filter attenuates ST segment information and can
    create pseudo-ST changes of clinical significance (up to 0.1 mV error).

    Order 4: sufficient roll-off below 0.05 Hz with minimal passband ripple.
    filtfilt doubles the effective order to 8 — adequate for wander suppression.

    EDGE TRANSIENT WARNING (filtfilt):
    filtfilt pads the signal at both ends using reflection. For a 0.05 Hz
    highpass filter, the filter time constant is ~3.2 seconds. Edge transients
    from the reflected padding decay over approximately 3× the time constant
    (~10 seconds). For 10-second PTB-XL recordings, this means the entire
    recording lies within the potential transient zone.

    MITIGATION: After calling this function, the caller must exclude the first
    and last `safe_margin_samples = ceil(3 / cutoff_hz * fs)` samples from
    all analysis (fiducial detection, feature extraction). These samples are
    unreliable. The exclusion zone is stored in PreprocessedECGRecord as
    `safe_analysis_start_sample` and `safe_analysis_end_sample`.

    For PTB-XL (500 Hz, 0.05 Hz cutoff): safe_margin = ceil(3/0.05 × 500/500)
    = ceil(60) = 60 samples = 0.12 seconds. This is acceptable.
    The concern is most severe for recordings shorter than 10 seconds —
    those require a proportionally larger fraction excluded.
    """
    nyq = fs / 2.0
    normalized_cutoff = cutoff_hz / nyq
    b, a = butter(4, normalized_cutoff, btype="high")
    return filtfilt(b, a, signal_1d)


def compute_safe_analysis_window(n_samples: int,
                                  fs: float,
                                  lowcut_hz: float = 0.05) -> tuple[int, int]:
    """
    Returns (safe_start_sample, safe_end_sample) for the reliable analysis region
    after filtfilt preprocessing.

    The edge margin is 3 × filter_time_constant in samples, where
    filter_time_constant = 1 / (2π × lowcut_hz).

    For 0.05 Hz at 500 Hz: margin = 3 × (1/(2π×0.05)) × 500 ≈ 4775 samples ≈ 9.5 s.

    NOTE: This is conservative. In practice, filtfilt edge transients for a
    Butterworth order-4 highpass at 0.05 Hz typically decay within 2–3 seconds
    (1000–1500 samples at 500 Hz). Using the theoretical time-constant margin
    is safe but may exclude too much of short recordings. VT-1.2-02 must
    empirically determine the actual transient duration and tighten this margin.

    Until VT-1.2-02 is complete, use 1500 samples (3 seconds at 500 Hz)
    as the safe margin — empirically adequate for Butterworth order 4.
    """
    # Conservative empirical margin pending VT-1.2-02 calibration
    margin_samples = int(math.ceil(3.0 * fs))  # 3 seconds
    safe_start = min(margin_samples, n_samples // 4)
    safe_end = max(n_samples - margin_samples, 3 * n_samples // 4)
    return safe_start, safe_end
```

**Cutoff frequency selection — clinical safety analysis:**

The AHA/ACC/HRS (2007 standardization document) specifies:
- Diagnostic mode: 0.05 Hz highpass cutoff
- Monitor mode: 0.5–1.0 Hz highpass cutoff (trades ST accuracy for motion tolerance)

The difference matters:
- At 0.05 Hz cutoff (Butterworth order 4): ST segment attenuation at 0.1 Hz ≈ −0.5 dB (negligible)
- At 0.5 Hz cutoff (Butterworth order 4): ST segment attenuation at 0.1 Hz ≈ −20 dB (severe — ST elevation of 1 mm appears as 0.1 mm)

**YouOwnECG uses 0.05 Hz exclusively.** The pipeline rejects recordings flagged as monitoring-mode (Node 1.3 quality gate).

### 4.2 Wavelet-Based Baseline Estimation (Alternative)

For recordings with severe wander that contains frequencies up to 0.5 Hz (which overlaps early P-wave content), a linear highpass filter will distort P waves. Wavelet-based baseline estimation is more surgical:

**Algorithm:**
1. Decompose signal using discrete wavelet transform (DWT) with db4 or sym8 wavelet
2. Zero out the lowest-frequency approximation coefficients (levels ≥ 7 at 500 Hz — these capture periods ≥ 500/2⁷ ≈ 4 s → frequencies ≤ 0.25 Hz)
3. Reconstruct signal from modified coefficients

```python
import pywt

def wavelet_baseline_removal(signal_1d: np.ndarray,
                              wavelet: str = "db4",
                              level: int = 9) -> np.ndarray:
    """
    Removes baseline wander by zeroing low-frequency DWT coefficients.

    Level 9 at 500 Hz captures frequencies <= 500/2^9 ≈ 1 Hz.
    Level 7 captures <= 500/2^7 ≈ 4 Hz (too aggressive).

    The approximation coefficients at `level` represent the slowly-varying
    baseline. Setting them to zero removes the baseline estimate while
    preserving all detail coefficients (ECG signal content).
    """
    coeffs = pywt.wavedec(signal_1d, wavelet, level=level)
    # Zero the approximation (lowest-frequency component)
    coeffs[0] = np.zeros_like(coeffs[0])
    return pywt.waverec(coeffs, wavelet)[:len(signal_1d)]
```

**Highpass filter vs wavelet — decision matrix:**

| Property | Highpass (0.05 Hz) | Wavelet (db4, level 9) |
|----------|--------------------|------------------------|
| ST segment preservation | Excellent (0.05 Hz cutoff) | Good |
| P-wave preservation | Good | Slightly better for severe wander |
| Implementation complexity | Low | Medium |
| Computational cost | Low | Medium |
| Interpretability | High (well-understood) | Lower |
| AHA guideline compliance | Yes (0.05 Hz = AHA standard) | Not directly specified |

**Decision: Highpass filter (0.05 Hz, Butterworth, filtfilt) as the default.** Wavelet-based removal is available as a fallback for recordings flagged as having severe wander (amplitude >2 mV) at Node 1.3. The fallback does not change the primary pipeline — it is triggered by a quality flag.

## 5. Bandpass Filter Design

### 5.1 Upper Cutoff Selection

The ECG's clinically useful content extends to approximately 150 Hz for standard diagnostic purposes. Above 150 Hz, the spectrum is dominated by noise with no clinical signal. EMG artifacts above 40 Hz are flagged separately (Node 1.3) rather than removed by the bandpass filter.

**Upper cutoff: 150 Hz.** This is the AHA-specified upper limit for diagnostic-mode recording. It preserves QRS high-frequency content (needed for late potential analysis, if used) while removing broadband noise above 150 Hz.

**Lower cutoff: 0.05 Hz** (same as baseline wander removal — the highpass step serves as the low-frequency cutoff of the bandpass).

### 5.2 Bandpass Filter Implementation

```python
def apply_bandpass_filter(signal_1d: np.ndarray,
                          fs: float,
                          lowcut_hz: float = 0.05,
                          highcut_hz: float = 150.0,
                          order: int = 4) -> np.ndarray:
    """
    Applies a zero-phase Butterworth bandpass filter.

    In practice, the lowcut (0.05 Hz) step is already applied by the
    highpass filter in the baseline removal stage. This function is
    used for the highcut (150 Hz) step specifically.

    At 500 Hz sampling rate, 150 Hz is exactly the Nyquist limit (0.6 × fs/2).
    Butterworth order 4 gives ≥40 dB attenuation at 2× the cutoff (300 Hz).
    """
    nyq = fs / 2.0
    low = lowcut_hz / nyq
    high = highcut_hz / nyq
    if high >= 1.0:
        # 150 Hz at 250 Hz sampling rate would exceed Nyquist — clip to safe value
        high = 0.95
    b, a = butter(order, [low, high], btype="band")
    return filtfilt(b, a, signal_1d)
```

**Filter type selection rationale:**

| Type | Passband ripple | Stopband roll-off | Phase (before filtfilt) | Use case |
|------|----------------|------------------|------------------------|----------|
| Butterworth | Maximally flat (0 dB) | Moderate | Non-linear | ECG — phase corrected by filtfilt |
| Chebyshev I | Equiripple passband | Better than Butterworth | Non-linear | Not recommended for ECG |
| Chebyshev II | Flat passband | Equiripple stopband | Non-linear | Acceptable |
| Elliptic | Equiripple both | Best | Non-linear | Overly complex for this use case |
| FIR (linear phase) | Depends on design | Depends | Zero phase natively | Streaming ECG; not needed here |

**Decision: Butterworth with filtfilt.** Zero passband ripple means no amplitude distortion within the 0.05–150 Hz band. filtfilt corrects phase distortion. Order 4 is sufficient.

## 6. Pipeline Ordering

### 6.1 Correct Order: Notch → Baseline → Bandpass

The preprocessing steps must be applied in this specific order:

**Step 1: Notch filter (powerline removal)**
**Step 2: Highpass filter (baseline wander removal, 0.05 Hz)**
**Step 3: Lowpass filter (upper cutoff, 150 Hz)**

**Justification for notch-first:**

If baseline wander removal is applied before the notch filter, the baseline signal may have already been distorted by powerline interference that then gets folded into the reconstructed signal during baseline estimation. In severe cases, a 60 Hz artifact riding on baseline wander causes the baseline estimator to produce an artifact-contaminated baseline estimate, leading to residual oscillation after subtraction.

Additionally, for wavelet-based baseline removal, the presence of large 60 Hz components increases the energy in high-level wavelet coefficients, potentially causing the threshold-based zeroing to aggressively remove ECG content. Notch-first prevents this.

**Justification for baseline-before-lowpass:**

The highpass baseline removal filter has a very low cutoff (0.05 Hz). At this cutoff, a Butterworth filter requires significant filter length to achieve adequate roll-off. If the signal already contains large-amplitude noise at high frequencies (e.g., EMG artifacts at 100–300 Hz), those artifacts increase the numerical range of the signal being fed into the high-order filtfilt computation, potentially causing floating-point instability in the denominator polynomial evaluation. Removing the large-amplitude high-frequency noise before baseline removal avoids this.

However, the notch filter (Step 1) does not remove broadband high-frequency noise — only the powerline line frequency. For recordings with extreme EMG artifacts, an optional pre-conditioning lowpass at 500 Hz (or the Nyquist limit) may be applied as step 0. This is handled by the quality gate at Node 1.3.

**Verified ordering:**

```
Notch (50/60 Hz + harmonics) → Highpass (0.05 Hz) → Lowpass (150 Hz)
```

This is consistent with the order recommended in:
- Pan & Tompkins (1985): notch before highpass for QRS detection preprocessing
- Sörnmo & Laguna (2005) "Bioelectrical Signal Processing": baseline removal before QRS detection, notch recommended first
- AHA (2007): specifies filter parameters but not explicit ordering; notch-first is standard practice

## 7. Artifact Detection (Flag, Do Not Remove)

### 7.1 EMG Artifact Detection

```python
def detect_emg_segments(signal_1d: np.ndarray,
                         fs: float,
                         window_s: float = 0.2,
                         emg_band: tuple = (40.0, 150.0),
                         threshold_snr_db: float = 10.0) -> list[tuple[int, int]]:
    """
    Detects windows dominated by EMG artifact.

    Method: sliding window band-power ratio.
    - Compute power in EMG band (40–150 Hz) and ECG band (5–40 Hz) per window
    - If EMG_power / ECG_power > threshold: flag window as artifact
    Returns list of (start_sample, end_sample) artifact windows.

    This does NOT remove signal. Flagged windows are passed to Node 1.3
    for beat-level exclusion decisions.
    """
    window_n = int(window_s * fs)
    artifact_windows = []
    for start in range(0, len(signal_1d) - window_n, window_n // 2):
        window = signal_1d[start:start + window_n]
        fft = np.abs(np.fft.rfft(window))
        freqs = np.fft.rfftfreq(window_n, d=1/fs)

        ecg_mask = (freqs >= 5) & (freqs <= 40)
        emg_mask = (freqs >= emg_band[0]) & (freqs <= emg_band[1])

        ecg_power = np.sum(fft[ecg_mask] ** 2)
        emg_power = np.sum(fft[emg_mask] ** 2)

        if ecg_power > 0 and 10 * np.log10(emg_power / ecg_power) > threshold_snr_db:
            artifact_windows.append((start, start + window_n))
    return artifact_windows
```

## 8. Resampling (Non-500 Hz Inputs)

ECGdeli's default wavelet decomposition is calibrated for 500 Hz. If the ingested signal is at a different sampling rate, resampling to 500 Hz is required:

```python
from scipy.signal import resample_poly
from math import gcd

def resample_to_500hz(signal: np.ndarray,
                      source_fs: float,
                      target_fs: float = 500.0) -> np.ndarray:
    """
    Resamples from source_fs to target_fs using polyphase filtering.

    resample_poly is preferred over scipy.signal.resample for ECG because:
    - resample_poly uses an anti-aliasing FIR filter before decimation
    - scipy.signal.resample uses FFT-based resampling which assumes
      periodicity and can introduce edge artifacts
    - Edge artifacts in ECG resampling can appear as spurious P or T waves
      at the beginning/end of the recording

    Handles common clinical rates:
    - 250 Hz → 500 Hz: up=2, down=1
    - 1000 Hz → 500 Hz: up=1, down=2
    - 360 Hz → 500 Hz: up=25, down=18 (MIT-BIH database rate)
    """
    source_fs_int = int(source_fs)
    target_fs_int = int(target_fs)
    g = gcd(source_fs_int, target_fs_int)
    up = target_fs_int // g
    down = source_fs_int // g

    resampled = np.zeros((int(len(signal) * up / down), signal.shape[1]),
                          dtype=signal.dtype)
    for lead_idx in range(signal.shape[1]):
        resampled[:, lead_idx] = resample_poly(signal[:, lead_idx], up, down)
    return resampled
```

Resampling is applied to `ECGRecord.signal` before preprocessing if `ecg_record.sampling_rate_hz != 500.0`. The result is stored in `PreprocessedECGRecord.preprocessed_signal` at 500 Hz. The original `ECGRecord.signal` at its native sampling rate is preserved.

## 9. Preprocessing Pipeline — Complete Sequence

```python
def preprocess_ecg(ecg_record: "ECGRecord") -> "PreprocessedECGRecord":
    """
    Full preprocessing pipeline. Returns PreprocessedECGRecord.

    The raw ECGRecord is never modified. The preprocessed_signal is a
    new (n_samples_500hz, 12) array.

    Steps (per lead):
    1. Resample to 500 Hz (if needed)
    2. Notch filter: powerline fundamental + harmonics
    3. Highpass: 0.05 Hz (baseline wander removal)
    4. Lowpass: 150 Hz (upper cutoff)
    5. Artifact flagging (EMG, electrode pop) — no removal

    Skipped for leads in ecg_record.clipping_detected_leads:
    Clipped leads are preserved as-is for quality assessment (Node 1.3)
    but not preprocessed — preprocessing a clipped signal may produce
    fictitious oscillations from the filter ringing at the flat peaks.
    """
    signal = ecg_record.signal.copy()
    fs = ecg_record.sampling_rate_hz

    if fs != 500.0:
        signal = resample_to_500hz(signal, fs, 500.0)
        fs = 500.0

    # Capture the resampled-but-unfiltered signal for morphology use (Issue 3 fix).
    # This is the correct amplitude reference for Node 1.5 morphology features
    # when the source was not at 500 Hz — raw amplitude, 500 Hz sample grid.
    signal_before_filtering = signal.copy()

    preprocessed = np.zeros_like(signal)
    filters_applied = []
    powerline_freq = None
    severe_wander_leads = []

    for lead_idx, lead_name in enumerate(ecg_record.lead_names):
        lead_signal = signal[:, lead_idx]

        # Skip clipped leads — preserve for quality gate
        if lead_name in ecg_record.clipping_detected_leads:
            preprocessed[:, lead_idx] = lead_signal
            continue

        # Step 1: Notch
        pf = _detect_powerline_frequencies(lead_signal, fs)
        if powerline_freq is None and pf:
            powerline_freq = pf[0]
        lead_signal = apply_notch_filter(lead_signal, fs, pf)

        # Step 2: Baseline wander removal
        # Measure wander energy in 0.05–0.5 Hz band (where wander lives).
        # This is more reliable than amplitude-based heuristics that conflate
        # large QRS amplitude with large wander amplitude.
        fft_mag = np.abs(np.fft.rfft(lead_signal))
        fft_freqs = np.fft.rfftfreq(len(lead_signal), d=1/fs)
        wander_mask = (fft_freqs >= 0.05) & (fft_freqs <= 0.5)
        total_mask = fft_freqs > 0
        wander_power_fraction = (np.sum(fft_mag[wander_mask] ** 2) /
                                  (np.sum(fft_mag[total_mask] ** 2) + 1e-12))
        # Threshold: empirical — to be calibrated by VT-1.2-02 on PTB-XL noisy subset.
        # Initial value 0.15: if >15% of total signal power is in the wander band,
        # the wander is severe enough to use the wavelet fallback.
        if wander_power_fraction > 0.15:
            lead_signal = wavelet_baseline_removal(lead_signal)
            severe_wander_leads.append(lead_name)
        else:
            lead_signal = apply_highpass_filter(lead_signal, fs, cutoff_hz=0.05)

        # Step 3: Lowpass
        lead_signal = apply_bandpass_filter(lead_signal, fs,
                                             lowcut_hz=0.05, highcut_hz=150.0)

        preprocessed[:, lead_idx] = lead_signal

    filters_applied = ["notch", "highpass_0.05hz", "lowpass_150hz"]

    # morphology_signal: raw amplitude at 500 Hz sample grid (unfiltered).
    # If the source was already 500 Hz: use the original raw signal directly.
    # If resampling was performed: use the resampled-but-unfiltered signal
    # (the `signal` variable before any filters were applied — captured here).
    # This ensures fiducial indices from ECGdeli (500 Hz grid) align correctly
    # with morphology amplitude measurements in Node 1.5.
    if ecg_record.sampling_rate_hz != 500.0:
        morphology_signal = signal_before_filtering  # resampled, unfiltered
    else:
        morphology_signal = ecg_record.signal        # original, unmodified

    safe_start, safe_end = compute_safe_analysis_window(
        preprocessed.shape[0], fs=500.0, lowcut_hz=0.05
    )

    return PreprocessedECGRecord(
        raw=ecg_record,
        preprocessed_signal=preprocessed,
        morphology_signal=morphology_signal,
        filters_applied=filters_applied,
        powerline_freq_hz=powerline_freq or 60.0,
        severe_wander_leads=severe_wander_leads,
        safe_analysis_start_sample=safe_start,
        safe_analysis_end_sample=safe_end,
    )
```
