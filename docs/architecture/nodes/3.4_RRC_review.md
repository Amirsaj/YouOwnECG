## Issues Found

### REQUIRED-1: Cardiologist annotation override is not bound to a user identity

**Location:** Node 3.4, Section 6.3 (cardiologist annotation)

The spec states cardiologist overrides are logged to the audit trail, but does not specify what identity is attached to the override log entry. If the override is logged as "clinician override" without a user ID, the audit trail is useless for medical-legal purposes — it shows that a clinician overrode a finding but not WHICH clinician.

The spec also does not address whether the cardiologist must authenticate before overriding. An unauthenticated user (e.g., a nurse with read-only dashboard access) could override a HIGH-confidence STEMI finding without any authentication gate.

**Required fix:**
- Override action must require an authenticated session with a role of `CARDIOLOGIST` or `PHYSICIAN` (not `NURSE` or `TECHNICIAN`).
- Audit log entry must include `user_id`, `user_role`, and `timestamp` alongside the override text.
- If the system has no authentication (MVP), the override UI must be explicitly disabled with a note: "Override requires clinician login — not available in demo mode."

### REQUIRED-2: PDF watermark implementation relies on "PDF canvas" but rendering library not specified

**Location:** Node 3.4, Section 6.2 (export)

The spec states the AI-GENERATED watermark "is part of the PDF canvas, not a DOM element." However, the PDF rendering library is not specified. If the implementation uses a browser-side library (e.g., jsPDF or react-pdf), the watermark is rendered as text on the page and can be removed with PDF editing tools. If it uses a server-side library (reportlab, WeasyPrint), the watermark can be embedded as a vector element that is harder to remove but still editable.

True non-removable watermarks require image-flattening the PDF (rasterizing each page), which prevents text selection but also makes the PDF inaccessible.

**Required fix:** Specify the PDF rendering library and the watermark approach explicitly. For MVP: use `reportlab` (Python, server-side). Watermark is rendered as a semi-transparent diagonal text stamp on each page at the PDF canvas level. Document that this is not cryptographically protected — it is a visible deterrent, not an enforcement mechanism. Note this limitation in the spec.
