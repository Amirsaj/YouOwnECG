## Executive Summary

Node 1.3 defines the signal quality assessment framework: computing per-lead quality scores, per-beat quality flags, and making the gating decision (proceed / proceed-with-warnings / reject).

Two principles drive every decision:

1. **Fail-loud, never silent.** If the recording quality is insufficient for clinical analysis, the pipeline rejects it with an explicit explanation. The orchestrator returns `PIPELINE_FAILED("QUALITY_REJECTED: <reason>")` to the caller. A silent degradation — producing a low-confidence report from a poor-quality recording — is clinically worse than a rejection, because the ER clinician does not know the analysis is unreliable.

2. **Lead-level granularity.** A single bad lead (electrode disconnection, EMG artifact on V2) should not invalidate the entire recording. The quality assessment operates per-lead. If V2 is flatlined but the remaining 11 leads are clean, the pipeline proceeds with V2 excluded and the agent notified. Only if the minimum required leads are unavailable does the pipeline reject.

## 1. Signal Quality Metrics

### 1.1 Signal-to-Noise Ratio (SNR) Estimation

SNR for ECG cannot use the classical definition (signal power / noise power) directly because signal and noise overlap in frequency. The ECG-specific SNR estimate separates signal from noise using the QRS as a template:

**Method: Template-based SNR (after Friesen et al., 1990)**

1. Detect R-peaks in the preprocessed signal (using a simple threshold on the smoothed signal — just enough for SNR estimation, not clinical-grade detection)
2. Extract a window around each R-peak: [−150 ms, +300 ms] (captures QRS and initial T wave)
3. Compute the ensemble average (template) across all detected beats
4. For each beat window: noise = beat_window − template; signal power = mean(template²); noise power = mean(noise²)
5. SNR (dB) = 10 × log₁₀(signal_power / noise_power)

```python
def _bootstrap_r_peaks(signal_1d: np.ndarray, fs: float) -> np.ndarray:
    """
    Internal bootstrap R-peak detector for SNR estimation only.
    NOT the clinical FPT. NOT fed to ECGdeli. ECGdeli performs its own
    full detection from scratch in Node 1.4.

    Method: threshold on squared first-derivative (simplified Pan-Tompkins).
    Robust on most recordings with SNR > 5 dB. Returns empty array on failure.

    Steps:
    1. Differentiate: emphasizes high-slope QRS upstroke
    2. Square: make all values positive, amplify large slopes
    3. Smooth with a 150 ms moving average: integrates QRS energy
    4. Threshold at 60% of max: peaks above threshold are R-peak candidates
    5. Enforce minimum RR interval of 250 ms (240 bpm max)
    """
    if len(signal_1d) < int(fs):
        return np.array([], dtype=int)

    diff = np.diff(signal_1d.astype(float), prepend=signal_1d[0])
    squared = diff ** 2
    window = int(0.150 * fs)
    integrated = np.convolve(squared, np.ones(window) / window, mode="same")

    threshold = 0.60 * np.max(integrated)
    above = integrated > threshold
    # Find transitions from below to above threshold
    crossings = np.where(np.diff(above.astype(int)) > 0)[0]

    if len(crossings) == 0:
        return np.array([], dtype=int)

    # Within each crossing region, find the actual peak
    min_rr = int(0.250 * fs)
    r_peaks = []
    for c in crossings:
        region_end = min(c + int(0.200 * fs), len(integrated))
        r_peaks.append(c + np.argmax(integrated[c:region_end]))

    # Enforce minimum RR
    filtered = [r_peaks[0]]
    for rp in r_peaks[1:]:
        if rp - filtered[-1] >= min_rr:
            filtered.append(rp)

    return np.array(filtered, dtype=int)


def estimate_snr_db(signal_1d: np.ndarray, fs: float) -> float:
    """
    Estimates SNR using ensemble averaging of beats around R-peaks.

    R-peaks are detected internally using `_bootstrap_r_peaks()` —
    a simplified derivative-threshold detector that does NOT require
    prior quality assessment and does NOT produce clinical fiducials.
    Its sole purpose is SNR estimation bootstrapping.

    Returns SNR in dB. Values below 10 dB indicate poor quality.
    Returns 0.0 if fewer than 3 beats are detectable (recording too noisy
    or too short for reliable SNR estimation — treated as POOR quality).
    """
    r_peaks = _bootstrap_r_peaks(signal_1d, fs)
    pre_samples = int(0.15 * fs)
    post_samples = int(0.30 * fs)
    window_len = pre_samples + post_samples

    beats = []
    for rp in r_peaks:
        start = rp - pre_samples
        end = rp + post_samples
        if start >= 0 and end <= len(signal_1d):
            beats.append(signal_1d[start:end])

    if len(beats) < 3:
        return 0.0  # insufficient beats for reliable estimate

    beats = np.array(beats)
    template = beats.mean(axis=0)
    noise = beats - template[np.newaxis, :]

    signal_power = np.mean(template ** 2)
    noise_power = np.mean(noise ** 2)

    if noise_power < 1e-12 or signal_power < 1e-12:
        return 0.0
    return float(10 * np.log10(signal_power / noise_power))
```

**SNR thresholds (from literature + AHA):**

| SNR (dB) | Quality label | Clinical interpretation |
|-----------|--------------|------------------------|
| ≥ 20 | GOOD | High confidence in all measurements |
| 15–20 | ACCEPTABLE | Minor quality concerns; proceed with note |
| 10–15 | MARGINAL | Increased measurement uncertainty; flag affected leads |
| < 10 | POOR | Unreliable; reject lead or recording |

These thresholds are consistent with Friesen et al. (1990) and the PTB-XL quality scoring literature (Strodthoff et al., 2020 report median SNR >25 dB for clean PTB-XL records).

### 1.2 Per-Lead Signal Quality Index (SQI)

The SQI is a composite score combining multiple quality indicators:

```python
def compute_sqi(lead_signal: np.ndarray,
                fs: float,
                snr_db: float,
                clipping_detected: bool,
                flatline_detected: bool,
                emg_artifact_fraction: float,
                baseline_wander_severe: bool) -> float:
    """
    Computes Signal Quality Index in [0.0, 1.0].

    Components:
    - SNR contribution (0.0–0.5): scaled from SNR
    - Clipping penalty: −0.5 if clipping detected (hard penalty)
    - Flatline penalty: −1.0 if flatline (lead is unusable — overrides all)
    - EMG artifact penalty: proportional to fraction of signal with EMG artifact
    - Wander penalty: −0.1 if severe wander (wavelet fallback was used)
    """
    if flatline_detected:
        return 0.0  # Lead is completely unusable

    if clipping_detected:
        return 0.1  # Severely degraded; only rhythm analysis may be possible

    # SNR contribution: map [0, 30] dB → [0.0, 0.5]
    snr_score = min(max(snr_db / 30.0, 0.0), 1.0) * 0.5

    # EMG penalty: emg_artifact_fraction is fraction [0,1] of signal windows flagged
    emg_penalty = emg_artifact_fraction * 0.3

    # Wander penalty
    wander_penalty = 0.1 if baseline_wander_severe else 0.0

    sqi = snr_score - emg_penalty - wander_penalty
    return float(max(0.0, min(1.0, sqi)))
```

### 1.3 Clipping Detection (Refined from Node 1.1)

Node 1.1 performed a coarse clipping check on raw ADU values. Node 1.3 refines this on the calibrated mV signal:

```python
def detect_clipping_mv(signal_1d: np.ndarray,
                        amplitude_ceiling_mv: float = 5.0,
                        flat_fraction_threshold: float = 0.001) -> bool:
    """
    Detects clipping in calibrated mV signal.

    Two criteria:
    1. Any sample exceeds ±amplitude_ceiling_mv (physical ADC limit)
    2. More than 0.1% of samples share the same max or min value (flat peaks)

    amplitude_ceiling_mv: typical ADC full-scale for diagnostic ECG recorders.
    Some devices use ±3.2 mV (16-bit at 100 μV/LSB) — this threshold should
    match the device spec if known.
    """
    if np.any(np.abs(signal_1d) > amplitude_ceiling_mv):
        return True
    for extreme in [np.nanmin(signal_1d), np.nanmax(signal_1d)]:
        if np.sum(signal_1d == extreme) / len(signal_1d) > flat_fraction_threshold:
            return True
    return False
```

### 1.4 Flatline Detection (Refined from Node 1.1)

```python
def detect_flatline(signal_1d: np.ndarray,
                     variance_threshold_mv2: float = 1e-4,
                     window_fraction: float = 0.5) -> bool:
    """
    Detects flatline (electrode disconnection or amplifier failure).

    Two criteria:
    1. Variance of the entire signal < threshold (whole-recording flatline)
    2. Variance in the central 50% of the signal < threshold (partial flatline)
       — catches recordings that start and end with signal but have a long
       disconnection in the middle

    Threshold 1e-4 mV²: P-wave amplitude ≈ 0.1 mV → variance ≈ 0.005 mV²
    which is 50× the flatline threshold. Any real cardiac signal exceeds this.
    """
    if np.nanvar(signal_1d) < variance_threshold_mv2:
        return True
    n = len(signal_1d)
    start = int(n * (1 - window_fraction) / 2)
    end = int(n * (1 + window_fraction) / 2)
    if np.nanvar(signal_1d[start:end]) < variance_threshold_mv2:
        return True
    return False
```

## 2. Quality Gating Logic

### 2.1 Lead Classification

Each lead receives one of five quality labels:

| Label | SQI range | Meaning | Downstream use |
|-------|-----------|---------|----------------|
| `GOOD` | ≥ 0.7 | Clean signal | All analysis |
| `ACCEPTABLE` | 0.5–0.7 | Minor noise | All analysis with note |
| `MARGINAL` | 0.3–0.5 | Significant noise | Rhythm/rate only; morphology excluded |
| `POOR` | 0.1–0.3 | Severely degraded | Excluded from all analysis |
| `UNUSABLE` | < 0.1 | Flatline or clipped | Excluded; downstream agents notified |

### 2.2 Minimum Required Leads

The pipeline defines which leads are required for each agent domain:

```python
# Minimum lead requirements per analysis domain
REQUIRED_LEADS = {
    "rhythm_rate":    ["II"],
    "conduction":     ["I", "V1"],
    # Ischemia requires all 12 leads: every territory has a designated lead set.
    # Inferior STEMI: II, III, aVF. Lateral STEMI: I, aVL. Anterior/posterior: V1–V6.
    # Omitting any of these means an entire STEMI territory cannot be assessed.
    "ischemia":       ["I", "II", "III", "aVL", "aVF",
                       "V1", "V2", "V3", "V4", "V5", "V6"],
    "morphology":     ["I", "II", "V1", "V5"],
    "axis":           ["I", "II"],
    "full_analysis":  ["I", "II", "III", "aVR", "aVL", "aVF",
                       "V1", "V2", "V3", "V4", "V5", "V6"],
}
```

A lead is considered "available" for an analysis domain if its quality label is `GOOD`, `ACCEPTABLE`, or `MARGINAL`. `POOR` and `UNUSABLE` leads are excluded.

### 2.3 Recording-Level Gate Decision

```python
from enum import Enum

class QualityGateDecision(Enum):
    PASS = "PASS"                    # All required leads clean
    PASS_WITH_WARNINGS = "PASS_WITH_WARNINGS"  # Some leads degraded; note in report
    REJECT = "REJECT"                # Insufficient leads for minimum analysis

def quality_gate(lead_sqis: dict[str, float],
                 lead_labels: dict[str, str]) -> tuple[QualityGateDecision, str]:
    """
    Makes the recording-level quality gate decision.

    Returns (decision, explanation).
    """
    available_leads = {lead for lead, label in lead_labels.items()
                       if label in ("GOOD", "ACCEPTABLE", "MARGINAL")}
    good_leads = {lead for lead, label in lead_labels.items()
                  if label in ("GOOD", "ACCEPTABLE")}

    # Hard rejection: rhythm analysis impossible
    if "II" not in available_leads:
        return QualityGateDecision.REJECT, (
            "Lead II is UNUSABLE or absent. Rhythm analysis requires Lead II. "
            f"Lead II SQI: {lead_sqis.get('II', 0):.2f}"
        )

    # Hard rejection: fewer than 6 analyzable leads
    if len(available_leads) < 6:
        return QualityGateDecision.REJECT, (
            f"Only {len(available_leads)}/12 leads are analyzable. "
            "Minimum 6 leads required for any clinical analysis."
        )

    # Conditional pass: fewer than 8 good/acceptable leads
    if len(good_leads) < 8:
        degraded = [l for l, lab in lead_labels.items()
                    if lab in ("MARGINAL", "POOR", "UNUSABLE")]
        return QualityGateDecision.PASS_WITH_WARNINGS, (
            f"Degraded leads: {degraded}. Analysis restricted to available leads. "
            "Morphology findings in degraded leads are not reported."
        )

    return QualityGateDecision.PASS, "All required leads within acceptable quality range."
```

### 2.4 Fail-Loud Protocol

Rejections must be explicit and actionable. The pipeline never silently degrades:

```python
def format_rejection_message(decision: QualityGateDecision,
                              explanation: str,
                              lead_labels: dict[str, str]) -> str:
    """
    Produces a clinician-readable rejection message.
    """
    if decision == QualityGateDecision.REJECT:
        unusable = [l for l, lab in lead_labels.items()
                    if lab in ("POOR", "UNUSABLE")]
        return (
            f"ECG QUALITY REJECTED: {explanation}\n"
            f"Unusable leads: {unusable}\n"
            f"Suggested action: Check electrode placement and reacquire ECG."
        )
    return explanation
```

The orchestrator converts `REJECT` into `PIPELINE_FAILED("QUALITY_REJECTED: <message>")` per Node 2.3 contract. The ER interface (SDA-3) displays this as a clearly labeled failure with the explanation, not as a blank result.

## 3. Per-Beat Quality Assessment

Recording-level quality gates are insufficient for clinical analysis. Individual beats within a recording may be contaminated by transient artifacts while neighboring beats are clean.

### 3.1 Beat-Level Quality Scoring

For each detected beat (using rough R-peak locations — refined fiducials come from Node 1.4):

```python
def score_beat_quality(beat_signal: np.ndarray,
                        template: np.ndarray,
                        fs: float) -> dict:
    """
    Scores a single beat against the ensemble template.

    beat_signal: signal window around the beat [-150ms, +400ms]
    template: ensemble average of all beats in the recording

    Returns dict with:
    - correlation_to_template: float [0,1] — how similar to normal beats
    - has_emg_burst: bool — high-frequency energy spike in this beat window
    - has_amplitude_spike: bool — sudden >3× amplitude relative to adjacent beats
    - quality_label: "CLEAN" | "NOISY" | "EXCLUDED"
    """
    if len(beat_signal) != len(template):
        return {"quality_label": "EXCLUDED", "correlation_to_template": 0.0,
                "has_emg_burst": False, "has_amplitude_spike": False}

    # Correlation to ensemble template
    corr = float(np.corrcoef(beat_signal, template)[0, 1])
    if np.isnan(corr):
        corr = 0.0

    # EMG burst: high power in 40–150 Hz band relative to 5–40 Hz band
    fft_mag = np.abs(np.fft.rfft(beat_signal))
    fft_freqs = np.fft.rfftfreq(len(beat_signal), d=1/fs)
    hf_power = np.sum(fft_mag[(fft_freqs >= 40) & (fft_freqs <= 150)] ** 2)
    lf_power = np.sum(fft_mag[(fft_freqs >= 5) & (fft_freqs <= 40)] ** 2)
    has_emg_burst = (hf_power > 3 * lf_power) if lf_power > 1e-12 else False

    # Amplitude spike: peak amplitude > 3× template peak amplitude
    has_amplitude_spike = (np.max(np.abs(beat_signal)) > 3 * np.max(np.abs(template)))

    # Label
    if corr < 0.5 or has_emg_burst or has_amplitude_spike:
        label = "EXCLUDED"
    elif corr < 0.8 or has_emg_burst:
        label = "NOISY"
    else:
        label = "CLEAN"

    return {
        "quality_label": label,
        "correlation_to_template": corr,
        "has_emg_burst": has_emg_burst,
        "has_amplitude_spike": has_amplitude_spike,
    }
```

### 3.2 Minimum Analyzable Beats

```python
MIN_CLEAN_BEATS = 3       # absolute minimum for any rhythm analysis
MIN_BEATS_FOR_STATS = 5   # minimum for meaningful statistical aggregation
```

If fewer than `MIN_CLEAN_BEATS` beats pass per-beat quality assessment, the pipeline rejects with `PIPELINE_FAILED("INSUFFICIENT_CLEAN_BEATS: only N beats passed quality assessment")`.

### 3.3 Beat Exclusion Criteria Summary

A beat is excluded from feature computation (but NOT from rhythm analysis) if:
- `quality_label == "EXCLUDED"` on ≥ 8 of 12 leads simultaneously
- The beat falls outside `safe_analysis_start_sample` / `safe_analysis_end_sample` from Node 1.2
- The beat is classified as `beat_class = 4` (unanalyzable) by ECGdeli (Node 1.4)

Excluded beats are counted and reported. If >50% of beats are excluded, the pipeline issues `PIPELINE_FAILED("MAJORITY_BEATS_EXCLUDED")`.

## 4. Monitoring-Mode Detection

ECGs recorded in monitoring mode (highpass ≥ 0.5 Hz) have distorted ST segments. The pipeline must detect and reject them:

```python
def detect_monitoring_mode(signal_1d: np.ndarray, fs: float) -> bool:
    """
    Heuristic: monitoring-mode recordings show characteristic ST distortion.

    Monitoring-mode highpass (0.5 Hz) attenuates the PR-to-ST energy transition,
    causing the ST segment to appear to slope downward relative to the PR baseline.
    Specifically, the power in 0.05–0.5 Hz (the band that monitoring mode removes)
    is very low relative to total ECG power in a monitoring-mode recording.

    After Node 1.2's 0.05 Hz highpass preprocessing, both monitoring and diagnostic
    mode recordings will have the baseline wander removed. However, a monitoring-mode
    ECG that was already highpass-filtered at 0.5 Hz before ADC will show
    characteristically low energy in 0.05–0.5 Hz even after our preprocessing
    (there is nothing there to preserve). A diagnostic-mode ECG will retain
    slow ST slope and T-wave baseline in this band.

    This detection is approximate — a definitive determination requires metadata
    from the recording device. If the device metadata is available (e.g., in the
    EDF header or WFDB comments), use that directly.
    """
    fft_mag = np.abs(np.fft.rfft(signal_1d))
    fft_freqs = np.fft.rfftfreq(len(signal_1d), d=1/fs)

    low_band = (fft_freqs >= 0.05) & (fft_freqs < 0.5)
    mid_band = (fft_freqs >= 0.5) & (fft_freqs < 40.0)

    low_power = np.sum(fft_mag[low_band] ** 2)
    mid_power = np.sum(fft_mag[mid_band] ** 2)

    if mid_power < 1e-12:
        return False
    # If the low-band power is less than 0.1% of mid-band power, suspect monitoring mode
    return (low_power / mid_power) < 0.001
```

If monitoring mode is detected: `PIPELINE_FAILED("MONITORING_MODE_DETECTED: ST analysis unreliable. Reacquire in diagnostic mode.")`. This failure mode is permanent — no fallback analysis is offered, because partial ST analysis on a monitoring-mode recording would be clinically dangerous.

## 6. Validation Against PTB-XL

PTB-XL includes per-record quality metadata in `ptbxl_database.csv`:
- `noise`: noise level annotation (0–3, where 0 = clean)
- `burst_noise`: binary flag for burst noise
- `electrodes_problems`: binary flag for electrode issues
- `extra_beats`: binary flag for extra beats (PVC/PAC)
- `pacemaker`: binary flag

**VT-1.3-01 validation plan:**
1. Run the quality pipeline on all 42,708 PTB-XL records
2. For records with `noise = 0` and `burst_noise = 0`: expect SQI ≥ 0.5 on all 12 leads in >95% of cases
3. For records with `electrodes_problems = 1`: expect ≥1 lead with `UNUSABLE` in >80% of cases
4. For records with `noise ≥ 2`: expect PASS_WITH_WARNINGS or REJECT in >90% of cases
5. False rejection rate (records with `noise = 0` that are REJECTED): target <0.5%
