## Preamble

The RRC adversarial review of Node 2.2 identified 15 issues (R1–R15): 7 mandatory (R1–R7), 6 required-before-pilot (R8–R12 + R13 as significant), and 2 recommended (R14–R15). This document adjudicates each issue, documents the disposition, and records the gate decision.

## Adjudication by Issue

### R1 — Token Budget Empirical Measurement

**RRC finding:** Feature object is estimated at 3,500–5,000 tokens — potentially 1.5–2× over the Phase 1 input budget stated in Node 2.1 Section 6.1.

**EPM ruling: PRE-IMPLEMENTATION VERIFICATION TASK — does not block design gate.**

Rationale: This is a measurement problem, not a design flaw. The design's full-feature routing decision (Section 1 and Executive Summary) is architecturally sound regardless of the exact token count. If the token count exceeds the Phase 1 budget, the response is well-defined: either expand the Phase 1 token budget (revising cost projections proportionally) or implement conditional feature inclusion for rarely-needed fields. Neither path requires redesigning the agent architecture. The EPM assigns this as VT-2.2-01: serialize a representative 12-beat 12-lead feature object using the Section 1.3 schema through the DeepSeek R1 tokenizer before Node 2.2 implementation begins.

**Status:** Recorded as VT-2.2-01. Design gate not blocked.

### R2 — T-Wave Symmetry Formula

**RRC finding:** `(Tpeak−Ton)/(Toff−Tpeak)` produces values outside 0–1; formula is mathematically incorrect.

**EPM ruling: FIXED IN DOCUMENT.**

The formula in Section 1.1 and Section 9.3 item 4 has been corrected to `min(Tpeak−Ton, Toff−Tpeak) / max(Tpeak−Ton, Toff−Tpeak)` with division-by-zero handling and null assignment for BBB beats. The fix is verified.

**Status:** Closed.

---

### R3 — TWA Detection on Short Strips

**RRC finding:** The 1.9μV threshold for T-wave alternans detection is not validated for 10-second ECG strips. On 12-beat strips the noise floor likely exceeds this threshold, generating high false-positive STAT rates.

**EPM ruling: FIXED IN DOCUMENT.**

Section 9.3 item 6 now specifies minimum recording duration of ≥60 seconds for reliable TWA detection. For recordings shorter than 60 seconds, `t_wave_alternans_present` must be set to `null` with `t_wave_alternans_reliability: "low"`. The practical consequence: TWA-based STAT flags will not fire on standard 10-second 12-lead ECGs. This is the correct clinical behavior — MTWA assessment requires sustained recording. The design document correctly acknowledges this limitation without removing the TWA field (which remains valid for longer Holter-style inputs).

**Status:** Closed.

---

### R4 — `p_wave_duration_ms` Missing from Schema

**RRC finding:** `p_wave_duration_ms` is referenced in RRC reasoning Step 2b and MR Step 6c but was absent from the Section 1.1 per-beat feature schema and from Section 9.3 SDA-1 requirements.

**EPM ruling: FIXED IN DOCUMENT.**

`p_wave_duration_ms[]` has been added to Section 1.1 per-beat fields (computed as `Poff − Pon` from ECGdeli FPT, null when P-wave absent). It has been added to Section 9.3 as item 6, explicitly requiring SDA-1 Feature Extraction to compute it. The inconsistency between reasoning chains and feature schema is resolved.

**Status:** Closed.

---

### R5 — `q_waves.type = "new"` Uncomputable

**RRC finding:** The IT output schema field `q_waves.type = "new|old|unknown"` requires comparison with a prior ECG. This system has no prior ECG access — the field will be hallucinated.

**EPM ruling: FIXED IN DOCUMENT.**

`q_waves.type` has been removed and replaced with `q_waves.present_in_territory: boolean`. The new field is computable from the current ECG alone. The distinction between new and old Q-waves is acknowledged in the design as outside scope — consistent with the system's stated single-ECG-per-analysis constraint. No CDS interaction depended on the new/old distinction (all CDS Q-wave interactions trigger on Q-wave presence, not age).

**Status:** Closed.

### R6 — Wellens Dual-Ownership

**RRC finding:** Both IT (Step 5d, `wellens_pattern` schema object) and MR (Step 2d, `wellens_type` schema field) classified Wellens syndrome with no defined CDS conflict resolution protocol.

**EPM ruling: FIXED IN DOCUMENT — ASSIGNED EXCLUSIVELY TO IT.**

Resolution path chosen: Wellens is assigned exclusively to IT. Rationale: Wellens syndrome is a territorial ischemic pattern (critical LAD stenosis presenting as V2-V3 T-wave inversions after angina). Its significance is ischemic, not morphologic. IT is the correct owner. MR's role is to report raw T-wave morphology observations (`symmetric_inversion_v2v3: boolean`, T-wave symmetry scores, T-wave per-lead morphology enum). CDS receives both the IT Wellens classification and MR's raw T-wave data and may use them for confidence calibration, but the diagnostic call is IT's alone.

Specific fixes applied:
- MR Step 1c: removed "→ Wellens Type B" label; changed to "note symmetry score"
- MR Step 2d: changed from "Wellens pattern → emit hint" to "flag `symmetric_inversion_v2v3 = true`; do NOT classify as Wellens — IT's exclusive domain"
- MR output schema: removed `wellens_type: "A|B|null"` field from `t_wave_morphology`

**Status:** Closed.

### R7 — `PACED_RHYTHM_DETECTED` Hint Missing

**RRC finding:** Paced rhythm has no defined hint type and no ST suppression rule. This is a common real-world presentation with no analysis pathway.

**EPM ruling: FIXED IN DOCUMENT.**

`PACED_RHYTHM_DETECTED` has been added to the cross_domain_hint vocabulary in Section 6.3. RRC emits this when paced rhythm is detected. IT responds by flagging all ST findings as unreliable in paced rhythm and requiring Sgarbossa-equivalent evaluation. The CDS priority queue in Section 5.4 Step 2 includes this hint type.

**Status:** Closed.

### R8 — CDS Priority Queue Incomplete

**RRC finding:** CDS Step 2 priority list omitted 6 of 14 hint types.

**EPM ruling: FIXED IN DOCUMENT.**

The CDS Step 2 priority queue in Section 5.4 has been expanded to all 17 hint types (including the three added during the fix pass: `PACED_RHYTHM_DETECTED`, `ARVC_SUSPECT`, `ELECTROLYTE_PATTERN_QTC`) with explicit ordering and required response for each.

**Status:** Closed.

### R9 — V4R Reference Without Schema Support

**RRC finding:** CDS cross-domain interaction 4 referenced `beats[*].leads.V4R` — a right-sided lead not in the standard 12-lead schema — with no fallback defined.

**EPM ruling: FIXED IN DOCUMENT.**

V4R reference removed from interaction 4. The RV STEMI criterion now uses V1 ST elevation ≥1mm as the primary criterion, which is both clinically valid and computable from the standard 12-lead feature set. V4R is not specified as a supported lead by SDA-1. If future expansion adds right-sided leads, the interaction can be upgraded.

**Status:** Closed.

---

### R10 — `LEFT_MAIN_PATTERN: boolean` Schema Conflict

**RRC finding:** `left_main_pattern: boolean` in IT output schema made IT a unilateral decision-maker for left main stenosis, bypassing CDS synthesis. This conflicts with the cross_domain_hint architecture's principle that cross-domain diagnoses are CDS outputs.

**EPM ruling: FIXED IN DOCUMENT.**

`left_main_pattern: boolean` removed from IT output schema. Replaced with `avr_st_status.elevation_mv_raw: float` (raw measurement). The `LEFT_MAIN_PATTERN` cross_domain_hint hint description has been updated to reflect that CDS detects the pattern from IT's `avr_st_status.elevation_mv_raw` + `st_depression.distribution = "diffuse"`. Left main stenosis diagnosis is a CDS synthesis output, not an IT output.

**Status:** Closed.

### R11 — RS Interval Definition Mismatch

**RRC finding:** Section 9.3 item 5 described `rs_interval_ms` as "onset of R to peak of S" — which does not match the Brugada Step 2 criterion (R onset to S nadir = `S_fiducial − QRSon_fiducial`).

**EPM ruling: FIXED IN DOCUMENT.**

Section 9.3 item 5 updated to specify `rs_interval_ms = S_fiducial − QRSon_fiducial`, using ECGdeli FPT columns directly. The Section 1.1 per-beat-per-lead schema field description has been updated to match. The Brugada algorithm in MR Step 5 and the RS interval definition are now consistent.

**Status:** Closed.

---

### R12 — AFib + LBBB WCT Not a Named Interaction

**RRC finding:** AFib + LBBB wide complex tachycardia combination had no cross-domain interaction in Section 5.5 and no RAG trigger.

**EPM ruling: FIXED IN DOCUMENT.**

AFib + LBBB added as cross-domain interaction 13 in Section 5.5 with RAG trigger rule. This covers the clinically important scenario where rate-controlled AFib with LBBB is misidentified as VT.

**Status:** Closed.

---

### R13 — Feature Schema Version Field Missing

**RRC finding:** No `feature_schema_version` field in the Section 1.3 feature object JSON. Schema evolution will be silent and incompatible.

**EPM ruling: FIXED IN DOCUMENT.**

`feature_schema_version: "1.0"` added to the top-level feature object schema in Section 1.3. All agents are expected to validate this field on input and fail with a structured error if the version is incompatible with their expected schema.

**Status:** Closed.

---

### R14 — Medical Safety Language

**RRC finding:** Section 7.2 prompt language used the word "survivable" to characterize procedural risk — imprecise and potentially trivializing.

**EPM ruling: FIXED IN DOCUMENT.**

Language revised. Section 7.2 now characterizes the asymmetry accurately: over-treatment from a false positive carries procedural risk; under-treatment from a false negative carries mortality risk. The framing is retained (asymmetric error costs) without minimizing procedural harm.

**Status:** Closed.

---

### R15 — Negative Control Unit Tests

**RRC finding (recommended):** Validation plan in Section 8 lacks negative control test cases (normal ECG → expect no STAT flags).

**EPM ruling: ACCEPTED AS IMPROVEMENT — NOT GATE-BLOCKING.**

Three negative control scenarios to be added in the Node 2.2 implementation phase when the full test suite is constructed:
- TC-RRC-04: Normal sinus rhythm — expect `rhythm.classification = "sinus_normal"`, no STAT flags
- TC-IT-04: Normal 12-lead ECG — expect no ST elevation, no STAT, no territorial findings
- TC-MR-04: Normal repolarization — expect `qtc_class = "normal"`, no morphology STAT

These are implementation-phase test cases, not design-phase requirements. The validation plan in Section 8.1 is complete for design gate purposes.

**Status:** Noted. To be added to Section 8 during implementation phase.
