### REQUIRED-2: PDF rendering library specified; watermark approach documented

**Fix applied to Node 3.4:**

PDF generation uses `reportlab` (Python, server-side). Watermark is a semi-transparent diagonal text stamp rendered at the PDF canvas level on every page:

```python
from reportlab.lib.colors import Color
from reportlab.pdfgen import canvas as pdf_canvas

WATERMARK_TEXT = "AI-GENERATED — NOT A CLINICAL DIAGNOSIS"
WATERMARK_COLOR = Color(0.6, 0.6, 0.6, alpha=0.25)  # semi-transparent gray

def add_watermark(c: pdf_canvas.Canvas, width: float, height: float):
    c.saveState()
    c.setFillColor(WATERMARK_COLOR)
    c.setFont("Helvetica-Bold", 28)
    c.translate(width / 2, height / 2)
    c.rotate(45)
    c.drawCentredString(0, 0, WATERMARK_TEXT)
    c.restoreState()
```

**Limitation documented:** The watermark is a visible deterrent. It is not cryptographically enforced. A user with a PDF editor can remove it. This is noted in the report footer: "This document contains an AI-generated watermark. Removal of the watermark does not change the nature of this document as AI-generated output."
