## Executive Summary

Node 1.1 defines the ECG ingestion layer: everything from reading a raw file off disk to producing a standardized in-memory signal representation that downstream nodes can trust.

Three constraints drive every decision:

1. **PTB-XL is the validation dataset, not the only input format.** The system must ingest WFDB (.mat/.hea/.dat), CSV, and EDF. PTB-XL uses WFDB at 500 Hz. Real-world ER inputs may be CSV or EDF at 250 or 1000 Hz. The internal representation must be format-agnostic.

2. **The sacred pipeline depends on correct signal units.** Node 1.2 (preprocessing), Node 1.4 (fiducial detection via ECGdeli), and Node 1.5 (feature extraction) all assume millivolts and seconds. Any calibration error at ingestion propagates through the entire pipeline. There is no later correction opportunity — fiducials detected at wrong amplitude scale produce wrong ST measurements.

3. **Lead naming must be canonical from the first moment.** ECGdeli, the feature extraction layer, and all agents use the standard 12-lead set {I, II, III, aVR, aVL, aVF, V1, V2, V3, V4, V5, V6}. Any alias (e.g., "AVFL", "lead2", "ch2") must be resolved at ingestion.

What this document does NOT specify: signal processing applied to the ingested signal (Node 1.2), quality scoring (Node 1.3), or feature extraction (Node 1.5).

## 1. First Principles: ECG Recording from Ion Channels to Digital Samples

### 1.1 Cardiac Electrophysiology — Why the ECG Has the Shape It Does

The surface ECG is a recording of time-varying potential differences between body surface electrodes. Understanding what generates these potentials is prerequisite to understanding what can go wrong with a digitized recording.

**Ion channel basis of cardiac action potential:**

Cardiomyocytes maintain a resting transmembrane potential of approximately −85 to −90 mV (inside relative to outside), sustained by the inward rectifier potassium current (IK1) keeping K⁺ near equilibrium. Depolarization is triggered by:
- Sinoatrial (SA) node spontaneous phase 4 depolarization (funny current If plus ICa,T/ICa,L)
- Propagated stimulus from neighboring cell gap junctions

The ventricular action potential phases:
- **Phase 0**: Rapid upstroke — voltage-gated Na⁺ channels (INa) open; membrane potential rises from −85 mV to +30 mV in ~1 ms. This is the fastest electrical event in the heart.
- **Phase 1**: Rapid repolarization — INa inactivates; transient outward K⁺ current (Ito) activates.
- **Phase 2**: Plateau — L-type Ca²⁺ current (ICa,L) balances K⁺ outflow; membrane potential stays near 0 mV for ~150–200 ms. This sustains contraction.
- **Phase 3**: Rapid repolarization — IKr (HERG channel) and IKs activate; ICa,L inactivates.
- **Phase 4**: Resting potential restored by IK1.

**From cellular action potential to body surface potential:**

A single depolarizing cell acts as a current dipole: positive charges flow toward the activated region on the outside, negative toward the resting region. The extracellular current flows through the body as a volume conductor. The body surface potential at any point is the vector sum of all active dipoles weighted by their distance and geometry.

The **mean cardiac vector** (MCV) at any instant is the net electrical axis of all simultaneously active dipoles. During QRS, the MCV sweeps through space as depolarization propagates from endocardium to epicardium, apex to base. The ECG lead measures the projection of the MCV onto that lead's axis.

**Clinical implication for ingestion:** The ECG signal is a projection, not a direct measurement of cellular activity. A 12-lead ECG samples 12 projections of the same 3D electrical vector. This is why leads are not independent — and why unit conversion errors (e.g., mV × 1000 confused for μV) affect all 12 leads predictably.

### 1.2 Lead Systems

**Einhoven limb leads (1903):**

Three electrodes: RA (right arm), LA (left arm), LL (left leg). The limb leads form an equilateral triangle (Einthoven's triangle) with the heart at the center.

```
Lead I   = LA − RA      (0° axis)
Lead II  = LL − RA      (+60° axis)
Lead III = LL − LA      (+120° axis)
```

Kirchhoff's voltage law: Lead I + Lead III = Lead II (this is a consistency check — a recording where this fails has a calibration error or electrode swap).

**Wilson central terminal (WCT):**

Wilson (1934) defined a reference by averaging the three limb electrodes:

```
WCT = (RA + LA + LL) / 3
```

WCT is approximately 0 mV relative to the body — it is an artificial ground that allows unipolar lead measurement.

**Goldberger augmented limb leads:**

Goldberger (1942) modified Wilson's approach to increase signal amplitude by ~50%:

```
aVR = RA − (LA + LL) / 2
aVL = LA − (RA + LL) / 2
aVF = LL − (RA + LA) / 2
```

Augmented leads are fully redundant with leads I, II, III (any three of the six can reconstruct the other three). Again this is a consistency check at ingestion.

**Precordial (chest) leads:**

V1–V6 measure voltage between the precordial electrode and WCT. Electrode positions are standardized (2017 AHA):
- V1: 4th intercostal space, right sternal border
- V2: 4th intercostal space, left sternal border
- V3: Between V2 and V4
- V4: 5th intercostal space, midclavicular line
- V5: Anterior axillary line
- V6: Midaxillary line

Placement errors are common in clinical practice and produce systematic distortions (e.g., V1/V2 placed too high → pseudo-rSR' pattern mimicking RBBB, or Brugada type 1). The ingestion layer cannot detect placement errors — they must be flagged at the quality assessment stage if deviations are suspected.

### 1.3 Analog-to-Digital Conversion

The surface ECG signal is analog. Before reaching the recording device, it undergoes:

1. **Electrode impedance**: Skin-electrode interface acts as a circuit with resistive and capacitive components. Poorly prepared skin (high impedance) increases baseline wander and common-mode noise.

2. **Instrumentation amplifier**: Differential amplifier with very high common-mode rejection ratio (CMRR, typically >100 dB) to reject the 50/60 Hz common-mode interference present on all body-surface electrodes simultaneously.

3. **Analog bandpass filter**: Pre-ADC filter typically 0.05–150 Hz (diagnostic mode) or 0.5–40 Hz (monitoring mode). The monitoring filter aggressively removes baseline wander at the cost of ST segment distortion — this is clinically important: monitoring-mode ECGs should not be used for STEMI diagnosis.

4. **ADC sampling**: The filtered signal is sampled at the Nyquist rate or above.

**Sampling rate and clinical significance:**

| Rate | Use case | Notes |
|------|----------|-------|
| 250 Hz | Most bedside monitors | Sufficient for rate/rhythm; QRS duration may be underestimated |
| 500 Hz | Standard diagnostic ECG | PTB-XL standard; acceptable for all measurements |
| 1000 Hz | High-resolution ECG | Required for HRV, late potentials, high-frequency QRS analysis |
| 2000+ Hz | Research | Rarely used clinically |

**ADC resolution:**

Modern ECG ADCs are 12–16 bit. At 12 bits: 2¹² = 4096 levels. If the full-scale range is ±5 mV, the least significant bit (LSB) is 10 mV / 4096 ≈ 2.4 μV — well below clinical significance (smallest clinically relevant ST change is 0.1 mV = 100 μV, ~41× the LSB).

**Quantization noise:** At 12-bit, the signal-to-quantization-noise ratio (SQNR) is approximately 6.02 × 12 + 1.76 ≈ 74 dB — negligible for clinical purposes.

**Gain/calibration factor:** Each ADC unit (integer sample value) represents a physical voltage. The calibration factor (gain) is specified in the recording header (WFDB: `gain` field in the `.hea` header). Correct unit conversion is:

```
V(mV) = (raw_sample − baseline_offset) / gain_adu_per_mV
```

Where:
- `raw_sample`: integer from ADC
- `baseline_offset`: the integer value corresponding to 0 mV (stored in `.hea`)
- `gain_adu_per_mV`: ADC units per millivolt (e.g., 200 adu/mV in PTB-XL)

**Critical ingestion invariant**: All downstream nodes assume signal values in millivolts. The calibration step is the first and only place where ADU → mV conversion occurs. If a recording uses μV (some older devices) and the ingestion layer treats the values as mV, all amplitude-based measurements (ST elevation, voltage criteria, QRS amplitude) will be 1000× too small.

## 2. WFDB Format Specification

### 2.1 Format Overview

WFDB (WaveForm DataBase) is the format used by PhysioNet and PTB-XL. A recording consists of paired files:

- `.hea`: ASCII header — contains metadata (sampling rate, lead names, gain, baseline, number of samples)
- `.dat` / `.mat`: Binary signal data

PTB-XL uses MATLAB `.mat` files (WFDB "format 16" stored in `.mat` wrapper) rather than raw `.dat` files. Both formats are handled by the `wfdb` Python package.

### 2.2 Header File Structure

The `.hea` header is a plain-text ASCII file. Example (PTB-XL record):

```
00001 12 500 5000
00001.mat 16+24 1000/mV 16 0 -12 16384 0 I
00001.mat 16+24 1000/mV 16 0 -7 16384 0 II
00001.mat 16+24 1000/mV 16 0 -3 16384 0 III
00001.mat 16+24 1000/mV 16 0 4 16384 0 aVR
00001.mat 16+24 1000/mV 16 0 -1 16384 0 aVL
00001.mat 16+24 1000/mV 16 0 -5 16384 0 aVF
00001.mat 16+24 1000/mV 16 0 -24 16384 0 V1
00001.mat 16+24 1000/mV 16 0 -7 16384 0 V2
00001.mat 16+24 1000/mV 16 0 16 16384 0 V3
00001.mat 16+24 1000/mV 16 0 9 16384 0 V4
00001.mat 16+24 1000/mV 16 0 25 16384 0 V5
00001.mat 16+24 1000/mV 16 0 18 16384 0 V6
```

**Line 1 — Record line:**
```
<record_name> <num_signals> <sampling_rate> <num_samples>
```
- `00001`: record name
- `12`: 12 leads
- `500`: sampling rate in Hz
- `5000`: number of samples (= 10 seconds at 500 Hz)

**Lines 2–13 — Signal lines (one per lead):**

Each field in order:
1. `filename`: the .mat or .dat file
2. `format+byte_offset`: `16+24` means format 16 (16-bit signed integer) with 24-byte header offset
3. `gain/units`: `1000/mV` means 1000 ADU per mV (gain = 1000)
4. `bit_resolution`: 16 (bits per sample)
5. `zero_value`: 0 (the ADU value representing the zero signal — baseline offset)
6. `first_sample_value`: the first sample's ADU value (used for checksum validation)
7. `checksum`: 16384 (sum of all samples mod 65536)
8. `block_size`: 0 (standard)
9. `description`: lead name string

**Calibration from header:**

For PTB-XL: `gain = 1000 adu/mV`, `baseline = 0`

```python
signal_mv = raw_adu.astype(float) / 1000.0
```

This is straightforward for PTB-XL. Other devices may use different gain values (200, 400, 500 adu/mV are common).

### 2.3 WFDB Loading with Python

The `wfdb` Python library (v4.x) handles all header parsing and binary decoding:

```python
import wfdb
import numpy as np

def load_wfdb_record(record_path: str) -> tuple[np.ndarray, dict]:
    """
    Loads a WFDB record and returns (signal, metadata).

    signal: float64 ndarray, shape (n_samples, n_leads), units: mV
    metadata: dict with sampling_rate, lead_names, n_samples, record_name
    """
    record = wfdb.rdrecord(record_path)
    # record.p_signal: physical signal in mV (wfdb applies gain/baseline automatically)
    # record.d_signal: digital (ADU) signal
    # record.sig_name: list of lead name strings
    # record.fs: sampling rate (Hz)
    # record.sig_len: number of samples

    signal = record.p_signal.astype(np.float64)  # (n_samples, n_leads), mV
    metadata = {
        "record_name": record.record_name,
        "sampling_rate_hz": record.fs,
        "n_samples": record.sig_len,
        "n_leads": record.n_sig,
        "lead_names_raw": record.sig_name,
        "units": record.units,
    }
    return signal, metadata
```

`wfdb.rdrecord()` with `physical=True` (default) applies the gain and baseline correction automatically, returning values in the units specified in the header (mV for PTB-XL). Do not re-apply gain manually — this would double-correct.

### 2.4 PTB-XL Specific Considerations

**Two sampling rates**: PTB-XL provides both 100 Hz and 500 Hz versions of every record. The 500 Hz version is stored in `records500/` subdirectories; 100 Hz in `records100/`. YouOwnECG uses 500 Hz exclusively — ECGdeli performs better at 500 Hz for P-wave and T-wave detection.

**Metadata join**: `ptbxl_database.csv` contains per-record metadata (ecg_id, patient_id, age, sex, scp_codes, quality metrics). The record filename (without extension) is the join key:

```python
import pandas as pd

db = pd.read_csv("ptbxl_database.csv", index_col="ecg_id")
# db.index is ecg_id (integer)
# db["filename_hr"] = path to 500Hz .hea file (relative to PTBXL root)
```

For YouOwnECG validation, `ecg_id` from `ptbxl_database.csv` becomes the `ecg_id` UUID after normalization. In production, `ecg_id` is a new UUID generated at ingestion — not derived from the PTB-XL record number (which would be PHI-linking risk if records were ever linked to patients).

**Diagnostic labels**: `scp_codes` column contains a JSON-like dict of SCP-ECG condition codes with confidence scores (e.g., `{'NORM': 100.0}` or `{'AMI': 80.0, 'STEMI': 100.0}`). Used for validation matching, not for analysis.

## 3. CSV and EDF Format Handling

### 3.1 CSV Ingestion

Clinical systems (e.g., GE MUSE, Philips PageWriter, Mindray) can export ECGs as CSV with varying structures. There is no single CSV ECG standard — auto-detection is required.

**Common CSV patterns:**

```
# Pattern A: time column + 12 lead columns (with header row)
time,I,II,III,aVR,aVL,aVF,V1,V2,V3,V4,V5,V6
0.000,-0.012,-0.007,-0.003,...
0.002,...

# Pattern B: 12 lead columns, no time, no header
-12,-7,-3,4,-1,-5,-24,-7,16,9,25,18
-11,-6,-3,...

# Pattern C: leads as rows (transposed)
I,-12,-11,-10,...
II,-7,-6,-5,...
```

**Auto-detection algorithm:**

```python
        "transposed": bool,
        "inferred_units": "mV" | "uV" | "ADU" | "unknown",
        "inferred_sampling_rate_hz": float | None,
    }
    """
    ...
```

**Unit detection heuristic**: ECG amplitudes in mV are typically in the range −2.0 to +2.0. If all values exceed ±1000, units are likely μV (divide by 1000). If values are integers in the range −16384 to +16383, units are likely ADU (gain unknown — flag for manual specification). If metadata in a companion `.txt` file specifies units, use that.

**Sampling rate detection**: If a time column exists: `fs = 1 / (time[1] - time[0])`. Otherwise, if file metadata is absent: assume 500 Hz and flag as assumed.

### 3.2 EDF/EDF+ Ingestion

EDF (European Data Format, 1992) is a binary format used by many hospital systems and some ECG recorders. EDF+ (2003) adds annotations.

**EDF header structure (fixed 256-byte header + per-signal headers):**

```
Bytes 0–7:   Version ("0       ")
Bytes 8–87:  Local patient ID (80 chars) — contains potential PHI
Bytes 88–167: Local recording ID (80 chars)
Bytes 168–175: Start date (DD.MM.YY)
Bytes 176–183: Start time (HH.MM.SS)
Bytes 184–191: Number of bytes in header
Bytes 192–235: Reserved
Bytes 236–243: Number of data records (-1 if unknown)
Bytes 244–251: Duration of each data record (seconds)
Bytes 252–255: Number of signals (ns)

Per-signal headers (ns × 256 bytes total):
- Signal label (16 chars): "EEG Fpz-Cz" or "ECG" — parsing needed
- Transducer type (80 chars)
- Physical dimension (8 chars): "mV", "uV", "V"
- Physical minimum (8 chars): e.g., "-3.2768"
- Physical maximum (8 chars): e.g., "3.2767"
- Digital minimum (8 chars): e.g., "-32768"
- Digital maximum (8 chars): e.g., "32767"
- Prefiltering (80 chars)
- Number of samples per data record (8 chars)
- Reserved (32 chars)
```

**EDF calibration**: Physical value = (digital_value - digital_min) × gain + physical_min, where:

```
gain = (physical_max - physical_min) / (digital_max - digital_min)
```

The `mpy.edfio` or `pyedflib` library handles this automatically.

**PHI in EDF headers**: The EDF header `local_patient_id` (80 chars) and `local_recording_id` (80 chars) fields may contain patient name, date of birth, and hospital ID in plaintext. These fields must be stripped at ingestion before any further processing.

**PHI stripping procedure (EDF):**

1. Read the EDF file headers using `pyedflib.EdfReader`.
2. Extract only the per-signal data arrays (the actual ECG samples) and the signal labels, sample frequencies, and physical dimension.
3. Do NOT store `local_patient_id`, `local_recording_id`, `startdate`, or `starttime` anywhere in the internal representation.
4. The `ECGRecord` receives no EDF header text — only the de-identified fields: `patient_sex` and `patient_age_years`, which must come from a companion metadata source or be left as "unknown"/0 if absent.

**Source filepath logging:** The source filepath is stored in `ECGRecord.source_filepath` for provenance. If the filepath contains a patient name (e.g., `/data/SMITH_JOHN_2026-03-29.edf`), the filepath itself is PHI. The ingestion layer logs the filepath only to the local audit database (not to any external aggregator). Application log output at INFO level must log only `ecg_id`, format, and file size — not the raw filepath.

**Unit test requirement:** A test must verify that a constructed `ECGRecord` from a synthetic EDF file with a `local_patient_id = "SMITH JOHN 1970-01-01 M"` does NOT contain any of those tokens in any string field of the `ECGRecord` dataclass (checked via `repr(ecg_record)`).

**Lead identification from EDF signal labels**: EDF signal labels for ECG are not standardized. Common patterns: "ECG I", "EKG I", "ECG_I", "Lead I", "ch1", "channel 1". The ingestion layer must apply a normalization map to identify lead names.

```python
import pyedflib

def load_edf_record(filepath: str) -> tuple[np.ndarray, dict]:
    with pyedflib.EdfReader(filepath) as f:
        n_signals = f.signals_in_file
        signal_labels = f.getSignalLabels()
        sample_frequencies = f.getSampleFrequencies()
        physical_dimensions = f.getPhysicalDimension()

        signals = np.array([f.readSignal(i) for i in range(n_signals)])
        # signals: (n_signals, n_samples) — note: transposed vs WFDB convention

    signal = signals.T
```

## 4. Internal Signal Representation

### 4.1 Design Requirements

The internal representation must:

1. **Store signal as float64 ndarray in mV** — downstream nodes assume mV
2. **Store canonical lead names** — in the fixed order {I, II, III, aVR, aVL, aVF, V1–V6}
3. **Store sampling rate** — required by preprocessing, ECGdeli, and feature extraction
4. **Be format-agnostic** — same structure regardless of WFDB/CSV/EDF source
5. **Record provenance** — which file was loaded, when, what format
6. **Carry quality pre-flags** — clipping detected at ingestion should be flagged here
7. **Never contain PHI** — patient name, MRN, DOB must be stripped; only `patient_sex` and `patient_age_years` are permitted

### 4.2 Representation Schema

```python
from dataclasses import dataclass
import numpy as np

# Canonical lead order — used throughout the pipeline
CANONICAL_LEADS = ["I", "II", "III", "aVR", "aVL", "aVF",
                   "V1", "V2", "V3", "V4", "V5", "V6"]

@dataclass
class ECGRecord:
    """
    Standardized in-memory ECG representation.

    All downstream nodes (preprocessing, quality, fiducials, features, agents)
    consume this structure. The format-specific loaders (WFDB, CSV, EDF)
    all produce this structure.
    """

    # Core signal data
    signal: np.ndarray          # shape: (n_samples, 12), dtype: float64, units: mV
    sampling_rate_hz: float     # samples per second (e.g., 500.0)
    lead_names: list[str]       # always CANONICAL_LEADS order after normalization

    # Identity (de-identified)
    ecg_id: str                 # UUID assigned at ingestion
    patient_sex: str            # "M" | "F" | "unknown"
    patient_age_years: int      # integer years; 0 if unknown

    # Provenance
    source_format: str          # "wfdb" | "csv" | "edf"
    source_filepath: str        # absolute path on local filesystem
    ingested_at_utc: str        # ISO 8601 timestamp

    # Pre-flags set at ingestion (before preprocessing)
    clipping_detected_leads: list[str]   # leads where ADC saturation detected
    flatline_detected_leads: list[str]   # leads with near-zero variance
    missing_leads: list[str]             # leads absent from source file
    computed_leads: list[str]            # leads derived algebraically (e.g., III, aVR, aVL, aVF from I+II)
    assumed_sampling_rate: bool          # True if rate was inferred, not read from header

    @property
    def n_samples(self) -> int:
        return self.signal.shape[0]

    @property
    def duration_s(self) -> float:
        return self.n_samples / self.sampling_rate_hz

    @property
    def lead_index(self) -> dict[str, int]:
        return {name: i for i, name in enumerate(self.lead_names)}
```

**Why a dataclass, not a dict:** The downstream nodes (ECGdeli wrapper, feature extraction) need to access `signal[..., lead_index["V1"]]` repeatedly. A typed dataclass with a `lead_index` property is more robust against key typos than a raw dict. The `@dataclass` is a slim, zero-dependency structure — not a heavy ORM or Pydantic model.

**Signal array layout**: `(n_samples, 12)` — time on the first axis, leads on the second. This matches numpy broadcasting conventions (operations over time on a single lead are `signal[:, i]`; operations over all leads at one time are `signal[t, :]`).

**ECGdeli interface shape (verified from source):** ECGdeli processes one lead at a time. `mastermind_delineate(signal, fs, fpt)` accepts a 1-D array for a single lead; `qrs_detection(signal, samplerate)` likewise takes 1-D input. The Node 1.4 caller iterates over leads:

```python
for lead_idx, lead_name in enumerate(CANONICAL_LEADS):
    lead_signal = ecg_record.signal[:, lead_idx]  # 1-D, (n_samples,)
    fpt = mastermind_delineate(lead_signal, fs, fpt, lead_name=lead_name)
```

No transposition is required. `ECGRecord.signal` shape `(n_samples, 12)` is correct and compatible with ECGdeli's per-lead 1-D interface. Verified: [ecgdeli/mastermind.py:581–614](ecgdeli/mastermind.py#L581) and [ecgdeli/qrs_detection.py:226](ecgdeli/qrs_detection.py#L226).

### 4.3 Lead Name Normalization

Sources use varied lead name strings. The normalization table maps all known variants to canonical names:

```python
LEAD_NAME_MAP = {
    # Standard variations
    "I": "I", "II": "II", "III": "III",
    "AVR": "aVR", "AVL": "aVL", "AVF": "aVF",
    "aVR": "aVR", "aVL": "aVL", "aVF": "aVF",
    "avr": "aVR", "avl": "aVL", "avf": "aVF",
    "AVFL": "aVL", "AVFR": "aVR",
    # Precordial leads
    "V1": "V1", "V2": "V2", "V3": "V3",
    "V4": "V4", "V5": "V5", "V6": "V6",
    # GE MUSE style
    "CH1": "I", "CH2": "II", "CH3": "III",
    "CH4": "aVR", "CH5": "aVL", "CH6": "aVF",
    "CH7": "V1", "CH8": "V2", "CH9": "V3",
    "CH10": "V4", "CH11": "V5", "CH12": "V6",
    # Mindray style
    "LEAD_I": "I", "LEAD_II": "II", "LEAD_III": "III",
    # ECG prefix variants
    "ECG I": "I", "ECG II": "II", "ECG III": "III",
    "EKG I": "I", "EKG II": "II", "EKG III": "III",
}

def normalize_lead_name(raw_name: str) -> str | None:
    """Returns canonical name or None if not recognized."""
    key = raw_name.strip().upper().replace(" ", "_").replace("-", "_")
    # Try direct lookup after case normalization
    for k, v in LEAD_NAME_MAP.items():
        if k.upper().replace(" ", "_") == key:
            return v
    return None
```

**Unrecognized lead names:** If a source file has 12 channels but some names are unrecognized, the ingestion layer:
1. Logs a warning with the unrecognized names
2. Sets those leads to `np.nan` in the signal array
3. Adds the unrecognized names to `missing_leads` in `ECGRecord`
4. Does NOT fail — downstream quality assessment will flag the missing leads

- If I and II are present: compute III, aVR, aVL, aVF. Mark computed leads in `ECGRecord` (e.g., add `"augmented_leads": ["III", "aVR", "aVL", "aVF"]` to provenance).

### 4.5 Signal Integrity Pre-Checks at Ingestion

Before the record is passed downstream, ingestion performs lightweight pre-checks:

**Clipping detection:**

ADC saturation produces flat peaks/troughs at the maximum integer value. Detection:

```python
def detect_clipping(signal_mV: np.ndarray,
                    threshold_fraction: float = 0.001) -> list[int]:
    """
    Returns indices of leads with suspected ADC clipping.
    Heuristic: if >0.1% of samples are at the min or max value of the lead,
    clipping is likely.
    """
    clipped = []
    for lead_idx in range(signal_mV.shape[1]):
        col = signal_mV[:, lead_idx]
        if np.isnan(col).all():
            continue
        min_val, max_val = np.nanmin(col), np.nanmax(col)
        at_min = np.sum(col == min_val) / len(col)
        at_max = np.sum(col == max_val) / len(col)
        if at_min > threshold_fraction or at_max > threshold_fraction:
            clipped.append(lead_idx)
    return clipped
```

**Flatline detection:**

Zero-variance segments indicate electrode disconnection or a failed amplifier channel:

```python
def detect_flatlines(signal_mV: np.ndarray,
                     variance_threshold_mV2: float = 1e-6) -> list[int]:
    """Returns indices of leads with near-zero variance (flatline)."""
    flat = []
    for lead_idx in range(signal_mV.shape[1]):
        col = signal_mV[:, lead_idx]
        if np.isnan(col).all():
            continue
        if np.nanvar(col) < variance_threshold_mV2:
            flat.append(lead_idx)
    return flat
```

These pre-checks are diagnostic flags only — they do not reject the recording. Rejection decisions are made at Node 1.3 (Quality Assessment).

## 6. Format Decision Matrix

| Format | When to use | Parser | Key challenge |
|--------|------------|--------|---------------|
| WFDB (.mat/.hea) | PTB-XL, PhysioNet datasets | `wfdb` library | None — well-specified |
| EDF/EDF+ | Hospital monitors, Holter devices | `pyedflib` | PHI in header; signal label non-standardization |
| CSV | GE MUSE, Philips, custom exports | Custom parser | No standard — structure varies per vendor |
| Raw binary | Research devices | Vendor-specific | Requires calibration spec from device documentation |

**Format detection heuristic** (for files without explicit format specification):

```python
def detect_format(filepath: str) -> str:
    """Detects ECG file format from extension and magic bytes."""
    if filepath.endswith(".hea"):
        return "wfdb"
    if filepath.endswith(".mat"):
        # Check if companion .hea exists (WFDB) or standalone (other MATLAB)
        if os.path.exists(filepath.replace(".mat", ".hea")):
            return "wfdb"
    if filepath.endswith((".edf", ".bdf")):
        return "edf"
    if filepath.endswith(".csv"):
        return "csv"
    # Magic byte check for EDF: first 8 bytes are "0       "
    with open(filepath, "rb") as f:
        magic = f.read(8)
    if magic == b"0       ":
        return "edf"
    raise ValueError(f"Cannot detect format of {filepath}")
```

## 7. Downstream Interface Contract

Node 1.1 produces `ECGRecord` objects consumed by:

| Node | What it uses from ECGRecord |
|------|-----------------------------|
| 1.2 (Preprocessing) | `signal` (mV), `sampling_rate_hz`, `clipping_detected_leads` |
| 1.3 (Quality) | `signal`, `clipping_detected_leads`, `flatline_detected_leads` |
| 1.4 (Fiducials — ECGdeli) | `signal` (mV), `sampling_rate_hz` (must be 500 Hz for ECGdeli default config) |
| 1.5 (Features) | `signal` (mV), `sampling_rate_hz`, `lead_names`, `patient_sex`, `patient_age_years` |
| All agents | `ecg_id`, `patient_sex`, `patient_age_years` (via feature object) |

**Sampling rate constraint for ECGdeli (Node 1.4):** ECGdeli's wavelet decomposition uses fixed-scale filters calibrated for 500 Hz. If the ingested record is at 250 Hz or 1000 Hz, resampling to 500 Hz is required before fiducial detection. This resampling is performed at the preprocessing stage (Node 1.2), not at ingestion — ingestion preserves the original sampling rate.
