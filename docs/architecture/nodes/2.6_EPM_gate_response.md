## EPM Adjudication

### Issue 1 — Timeout Race Condition: `asyncio.wait_for()` vs httpx Internal Timeout
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

Runtime assertion added at `call_agent()` entry:
```python
assert client.timeout == httpx.Timeout(None), (
    f"call_agent() requires client.timeout=httpx.Timeout(None); "
    f"got {client.timeout}. httpx internal timeouts race with asyncio.wait_for()."
)
```
Section 2.2 docstring documents the precondition and the error consequence (httpx timeout raises `AgentAPIError` instead of `AgentTimeoutError`, breaking orchestrator retry branching). Connection pool state after `asyncio.wait_for()` cancellation documented: httpx ≥0.23 closes the connection cleanly; pool is not degraded. Persistent client creation enforced at application startup (Section 9.4), where `timeout=httpx.Timeout(None)` is set in one place. Issue resolved.

### Issue 2 — `dispatch_phase1()` Collection in Insertion Order Breaks STAT Fast-Path
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

`dispatch_phase1()` removed. Replaced with `iter_phase1_results()` async generator (Section 2.4) using `asyncio.as_completed()`:
```python
for completed_task in asyncio.as_completed(task_to_agent.keys()):
    agent_id = task_to_agent[completed_task]
    ...
    yield agent_id, result
```
Yields `(agent_id, output_or_exception)` in completion order — not insertion order. Orchestrator usage pattern documented: STAT fast-path check fires on each yielded result immediately. Issue resolved.

### Issue 3 — `_extract_json()` Strategy 3 Fragile Under Reasoning Bleed
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

Strategy 3 greedy regex (`r"\{[\s\S]+\}"`) replaced with `json.JSONDecoder().raw_decode()` scanning at each `{` position in `content`. Handles both reasoning-bleed (prose containing `{` characters before the JSON) and trailing text after the closing brace. Docstring documents the reasoning-bleed scenario as a known DeepSeek R1 behavior at temperature=0. Issue resolved.

### Issue 4 — PHI Validation Uses Denylist; `ALLOWED_PATIENT_FIELDS` Wired but Unused
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

`validate_no_phi()` rewritten from denylist to allowlist logic:
```python
ALLOWED_PATIENT_FIELDS = {"patient_sex", "patient_age_years"}
_PHI_PREFIXES = ("patient_", "subject_", "visit_", "encounter_", "mrn", "dob", "ssn", "name")

def validate_no_phi(feature_obj: dict) -> None:
    blocked = [k for k in feature_obj.keys()
               if any(k.lower().startswith(p) for p in _PHI_PREFIXES)
               and k not in ALLOWED_PATIENT_FIELDS]
    if blocked:
        raise ValueError(...)
```
`ALLOWED_PATIENT_FIELDS` constant is now the primary gate. Any new quasi-identifier field added by SDA-1 that matches a PHI prefix is blocked without needing to update a denylist. Issue resolved.

### Issue 5 — Cost Estimates 3–5× Too Low for Complex Cases
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 5.2 now presents two separate cost tables: typical case (~$0.046/ECG) and complex case (~$0.086/ECG typical complex, up to ~$0.15/ECG extreme). Operational cost basis stated as $0.046–$0.15 range. The $0.046 figure is explicitly noted as not a safe planning number for complex STAT cases.

`_log_usage()` updated with:
- `MAX_COST_PER_ECG_USD = 0.20` circuit breaker constant
- `_estimate_call_cost()` helper computing per-call cost from usage dict
- `max_cost_per_ecg_usd` parameter on `_log_usage()` to accumulate per-ECG total
- Alert behavior: operational warning only (not a hard abort — suppressing a STEMI finding to save cost is not acceptable)

VT-2.1-03 expanded to require: (a) reasoning token billing verification, (b) actual reasoning token counts on at least 3 PTB-XL ECGs with complex diagnoses (STEMI, polymorphic VT or Brugada, LBBB masking ischemia), (c) per-ECG cost recorded; if >$0.15 flagged for cost model revision. Issue resolved.

### Issue 6 — `_round_floats` as `default=` Handler Never Invoked for Native Floats
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

`_round_floats` as `default=` handler replaced with `_RoundedFloatEncoder(json.JSONEncoder)` subclass overriding `iterencode`. Python `float` is natively JSON-serializable — `json.dumps default=` is never called for it. The JSONEncoder subclass intercepts float encoding at the correct level. Docstring documents the root cause explicitly. Issue resolved.

### Issue 7 — `_current_model_version` Global Resets on Restart; No Cross-Session Persistence
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 6.2 redesigned:
- `_current_model_version` module global removed as primary tracking mechanism
- `load_model_version_baseline(db)` loads stored version from `api_config` table at startup
- `_check_and_persist_model_version(response_model, db)` compares against DB-stored baseline; updates DB and logs if changed; establishes baseline on first run
- Startup sequence specified: single-threaded init path loads baseline before accepting requests; no async-safety issue for the init-time load
- Docstring explains why the module global is insufficient (resets on restart = cannot detect between-shift API updates) Issue resolved.

### Issue 8 — Audit Record Destination and Governance Unspecified
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT (Section 6.4 added)

Section 6.4 specifies:

| Dimension | Decision |
|-----------|----------|
| Database | SQLite (`youownecg.db`), table `agent_audit_log` |
| Schema | Full DDL specified: audit_id, ecg_id, pipeline_run_id, agent_id, model_version, prompt/completion/reasoning/cache tokens, estimated_cost_usd, latency_s, finish_reason, hallucination_flags, timestamp_utc |
| Write failure | **Fail-open**: pipeline continues; write failure logged to app error log; 3+ consecutive failures trigger operational alert |
| Retention | 7 years minimum from analysis date (consistent with US HIPAA 6-year minimum; aligned to typical 7–10 year hospital retention) |
| Access control | Write: application service account only. Read: authorized clinical staff via application audit interface; sysadmin with access log. No direct DB access for front-end users. |
| Write mode | **Synchronous** within `call_agent()` before return. <2ms SQLite insert latency is within pipeline budget. Async-deferred write rejected: silent failure creates false audit trail impression. |

Issue resolved.

### Issue 9 — No Connection Pooling; TLS Overhead Per ECG
**RRC Severity:** RECOMMENDED
**EPM Decision:** FIXED IN DOCUMENT (Section 9.4 added)

Section 9.4 specifies:
- Persistent `httpx.AsyncClient` created at application startup with `timeout=httpx.Timeout(None)` and `limits=httpx.Limits(max_connections=20, max_keepalive_connections=10)`
- Injected from application state into all `call_agent()` call sites
- Closed on graceful shutdown
- Note: centralizing client creation at startup also enforces the timeout precondition (Issue 1) in one place — the assertion at `call_agent()` entry is defense-in-depth, not the primary enforcement mechanism

This eliminates per-ECG TLS handshake overhead (100–300ms) and prevents burst connection patterns. Issue resolved.

### Issue 10 — No Local Replay Proxy for Integration Testing
**RRC Severity:** RECOMMENDED
**EPM Decision:** FIXED IN DOCUMENT (Section 9.5 added)

Section 9.5 specifies:
- Replay proxy exposes same interface as DeepSeek API at `localhost:8765`; activated via `DEEPSEEK_API_URL` override
- Fixture format: real API responses stored as JSON files under `tests/fixtures/replay/<ecg_id>_<agent_id>.json`
- Fixture capture: `CAPTURE_FIXTURES=true` env var; fixtures committed to test suite
- Lookup: hash of `messages[1].content` (user content containing ECG features); 404 on miss — explicit failure, not silent wrong-fixture substitution
- Initial fixtures: captured during VT-2.6-01 and VT-2.6-02; regression fixture set (10 PTB-XL ECGs from Section 9.2) captured and committed before first deployment

Project no-mock rule is respected: fixtures are real API responses, not fabricated. Issue resolved.
