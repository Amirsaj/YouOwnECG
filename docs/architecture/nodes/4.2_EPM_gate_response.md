## Resolution of RRC Issues

### REQUIRED-1: Tail merge size guard

**Fix applied to Node 4.2:**

Tail chunk merge now checks that the merged result does not exceed `MAX_CHUNK_TOKENS`:

```python
if current_tokens < MIN_CHUNK_TOKENS and chunks:
    last_chunk_tokens = len(tokenizer.encode(chunks[-1]["text"]))
    if last_chunk_tokens + current_tokens <= MAX_CHUNK_TOKENS:
        chunks[-1]["text"] += "\n\n" + current_text.strip()
    else:
        chunks.append({**metadata, "text": current_text.strip()})
else:
    chunks.append({**metadata, "text": current_text.strip()})
```

VT-4.2-01 (0 chunks above 800 tokens) is now enforced even after tail merges.

### REQUIRED-2: chunk_id uses section_title_hash instead of section_idx

**Fix applied to Node 4.2:**

```python
import hashlib

def make_chunk_id(
    book_short: str,
    chapter_number: int,
    section_title: str,
    page_number: int,
    chunk_idx: int,
) -> str:
    section_hash = hashlib.md5(section_title.encode()).hexdigest()[:8]
    return f"{book_short}_ch{chapter_number:02d}_{section_hash}_p{page_number:04d}_c{chunk_idx}"
```

Section titles are stable across re-ingestions (they come from the book text). Hash collisions across different section titles within the same chapter are astronomically unlikely for 8-char MD5. The chunk_id format is updated in the metadata schema in Node 4.2 Section 4.
