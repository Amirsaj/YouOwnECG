## EPM Adjudication

### Issue 1 — `ECGRecord.signal` Shape vs ECGdeli Input Contract (Unverified)
**RRC Severity:** CRITICAL
**EPM Decision:** FIXED IN DOCUMENT

ECGdeli source code verified directly:

- `mastermind_delineate(signal, fs, fpt, lead_name)` — [ecgdeli/mastermind.py:581](ecgdeli/mastermind.py#L581): `signal = np.asarray(signal, dtype=float).ravel()` — expects 1-D input for a single lead.
- `qrs_detection(signal, samplerate)` — [ecgdeli/qrs_detection.py:226](ecgdeli/qrs_detection.py#L226): same 1-D per-lead interface.

ECGdeli processes one lead at a time. There is no multi-lead array input. The Node 1.4 caller iterates over CANONICAL_LEADS and passes `ecg_record.signal[:, lead_idx]` (1-D slice) to ECGdeli per lead. `ECGRecord.signal` shape `(n_samples, 12)` is correct and requires no transposition.

Section 4.2 updated with the verified ECGdeli interface and a concrete Node 1.4 usage pattern. Issue resolved.

---

### Issue 2 — EDF PHI Stripping Requirement Without Mechanism
**RRC Severity:** REQUIRED
**EPM Decision:** FIXED IN DOCUMENT

Section 3.2 now specifies:
1. Exact EDF fields stripped: `local_patient_id`, `local_recording_id`, `startdate`, `starttime`
2. Mechanism: read signal arrays and labels only; PHI fields never stored in `ECGRecord`
3. `patient_sex` / `patient_age_years` must come from a companion metadata source or default to "unknown"/0
4. Filepath logging: stored in `ECGRecord.source_filepath` for local provenance only; INFO-level logs emit only `ecg_id`, format, file size
5. Unit test requirement: `repr(ecg_record)` must not contain any tokens from a test EDF's `local_patient_id`

Issue resolved.

---

### Issue 3 — Computed Leads Not Typed as Distinct Field
**RRC Severity:** RECOMMENDED
**EPM Decision:** FIXED IN DOCUMENT

`computed_leads: list[str]` added as a typed field on `ECGRecord` alongside `clipping_detected_leads`, `flatline_detected_leads`, and `missing_leads`. Downstream consumers (including agents) can check `"aVL" in ecg_record.computed_leads` to weight confidence accordingly for derived limb leads.

Issue resolved.
