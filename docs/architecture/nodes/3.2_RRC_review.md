## Issues Found

### REQUIRED-1: EXPLANATION_TEMPLATES use f-string substitution with unvalidated feature values

**Location:** Node 3.2, Section 3 (explanation templates)

The deterministic explanation templates substitute feature values directly:

```python
f"ST elevation of {st_elevation_mv:.2f} mV detected in lead {lead}"
```

If `st_elevation_mv` is `None` (e.g., POOR quality lead with no measurement), the f-string raises `TypeError: unsupported format character`. This will crash the disease dashboard render for any POOR-quality lead.

**Required fix:** All feature values substituted into EXPLANATION_TEMPLATES must be guarded:

```python
def fmt_mv(v):
    return f"{v:.2f}" if v is not None else "N/A"
```

The template must degrade gracefully: "ST elevation of N/A mV (measurement unavailable — signal quality insufficient)".

### REQUIRED-2: VL2 unavailability fallback not specified for figure panel

**Location:** Node 3.2, Section 4 (VL2 visual confirmation component)

The spec states that when VL2 is unavailable, the visual confirmation panel is replaced with a fallback. But the fallback content is not specified. If the component renders an empty panel or a generic "VL2 unavailable" string, the cardiologist loses context about WHY the VL2 result is absent (timeout vs unsupported lead configuration vs VL2 API error).

**Required fix:** The fallback must distinguish three states:
- `VL2_TIMEOUT`: "Visual confirmation timed out (> 15 s). Signal features used only."
- `VL2_NOT_APPLICABLE`: "Visual confirmation not applicable for this finding type."
- `VL2_ERROR`: "Visual confirmation failed (error). Signal features used only."

These states must be part of `VisionVerificationResult` (or its absence record) from Node 1.6.
