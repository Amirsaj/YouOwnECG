## 1. Human Visual Perception for Data Comprehension

### 1.1 Gestalt Principles Applied to Medical Dashboards

- **Proximity:** Group related information spatially. HR/rhythm/intervals belong together; findings list belongs together. Do NOT scatter related information across the screen.
- **Similarity:** Use consistent visual encoding. If orange = ST elevation on the ECG, orange must mean ST elevation everywhere in the interface (not used for any other meaning).
- **Continuity:** The eye follows lines and curves. Use the ECG trace as the natural focus path — annotations should point TO the trace, not away from it.
- **Figure-Ground:** Critical findings must stand out from background. STAT badge must have maximum contrast against the surrounding interface.

### 1.2 Limits of Working Memory

Miller's Law: 7 ± 2 items in working memory. Clinical implementation: never present more than 5–7 findings simultaneously in the initial view. Additional findings are accessible via "show more" — they are not suppressed.

## 2. Pre-Attentive Processing in Medical Interfaces

Pre-attentive features (processed in < 200 ms, before focal attention):
- **Color** — use for urgency coding (red = STAT, orange = critical, green = normal)
- **Size** — STAT badge is the largest text element
- **Motion** — pulse animation for STAT only; never use animation for non-critical information (alert fatigue)
- **Shape** — symbol encoding (✓, ▲, ⬛) supplements color for colorblind accessibility

**Anti-pattern:** "Visual noise" — too many colors, animations, or badges simultaneously defeats pre-attentive processing. The STAT badge must be unique in the interface — if everything is red and blinking, nothing is.

## 3. Medical Informatics: CDS Interface Design (2024–2026)

**Key literature findings:**

1. **Alert fatigue (van der Sijs et al., 2006):** Clinicians override 49–96% of clinical decision support alerts in EHR systems. Primary cause: too many alerts for non-critical conditions. Solution: reserve the highest alert tier strictly for conditions requiring immediate action (8 STAT conditions in YouOwnECG, not 50).

2. **Cognitive load in emergency medicine (Croskerry, 2002):** Dual-process theory — System 1 (fast, intuitive) and System 2 (slow, analytical). ER nurses under stress default to System 1. The UI must support correct System 1 responses: STAT = red = act now. This encoding must be consistent and never violated.

3. **Information foraging theory (Pirolli & Card, 1999):** Clinicians scan for "information scent" — cues that indicate where valuable information is. Strong scent: clear visual hierarchy, consistent labels, prominent abnormal findings. Weak scent: everything looks equally important.

4. **Explainability in clinical AI (Tonekaboni et al., 2019):** Clinicians trust AI recommendations more when reasoning is visible but do NOT want to read it during triage. Correct pattern: show conclusion first, reasoning on demand.

## 4. Design Principles Derived from Research

**P1 — One dominant visual element per urgency level.** The STAT badge is unique. Nothing else is red and animated.

**P2 — Never use color as the sole encoding for critical information.** Always add shape + text.

**P3 — Show conclusions before reasoning.** Finding name and confidence first; full reasoning chain behind a click.

**P4 — Reserve highest alert tier strictly.** Maximum 8–10 conditions trigger STAT. More conditions in the STAT tier = more overrides = missed real STEMIs.

**P5 — Preserve the ECG as the ground truth.**  The interface exists to annotate and explain the ECG, not to replace it. The ECG trace is always visible above the fold.

**P6 — Deterministic data is not generated by LLM.** Measurement values (PR = 162 ms, ST = +0.28 mV) are computed and displayed as-is. The LLM is invoked only for interpretation and explanation, not measurement.

**P7 — "AI-GENERATED" label is non-removable and persistent.** Not a fine-print disclaimer. Visible on the dashboard, on every report page, in every exported document.

**P8 — Clinical language for ER display; technical language for cardiologist display.** "Possible Anterior STEMI" for the nurse; "ST elevation 2.8 mm V2–V4 with hyperacute T, consistent with anterior STEMI involving LAD territory, Sgarbossa score 0" for the cardiologist.
