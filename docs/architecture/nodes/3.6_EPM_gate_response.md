## Resolution of RRC Issues

### REQUIRED-1: P8 language switch — schema-level specification

**Fix applied to Node 3.6 (and Node 2.3 schema amendment):**

The `DiagnosticFinding` schema carries both language registers, generated in a single agent pass:

```python
@dataclass
class DiagnosticFinding:
    finding_type: str
    confidence: str
    clinical_summary: str      # ≤ 15 words, plain English: "Possible Anterior STEMI"
    technical_detail: str      # Full clinical terminology with values: "ST elevation 2.8 mm V2–V4, hyperacute T, Sgarbossa score 0, LAD territory"
    # ... other fields ...
```

The ER dashboard renders `clinical_summary`. The cardiologist extension renders `technical_detail`. No second LLM call is made. The agent generates both fields in a single structured output response (enforced by the agent prompt JSON schema).

### REQUIRED-2: Finding count limit applies to distinct conditions, not beat events

**Fix applied to Node 3.6:**

The finding count threshold for Miller's Law (5–7) applies to `DiagnosticResult.findings`, which contains one entry per condition after BeatSummary aggregation (Node 2.5). Beat-level events (PVCs, PACs) are pre-aggregated:

```python
# Correct: one entry per condition
findings = [
    DiagnosticFinding(finding_type="possible_anterior_stemi", ...),
    DiagnosticFinding(finding_type="sinus_tachycardia", ...),
    DiagnosticFinding(finding_type="occasional_pvcs", clinical_summary="10 PVCs (16.7% of beats)", ...),
    # ...
]
# len(findings) = 3–8 typically. "Show more" triggers at > 7.
```

The "show more" button is rendered when `findings.length > 7`. The initial view shows the top 7 by confidence (HIGH first, then MEDIUM, then LOW). Remaining findings are accessible via "show more" — not suppressed.
