## Issues Found

### REQUIRED-1: Zustand store initialized from Server Component but hydration mismatch is not addressed

**Location:** Node 3.5, Section 4 (state management)

The spec states "ECG data is fetched once and never changes — Zustand with no re-render triggers." The data is fetched in a Server Component and the Zustand store is populated on the client. However, Next.js App Router Server Components serialize data to the client via React's serialization protocol. If the FeatureObject contains non-serializable fields (e.g., `Float32Array`, Python-specific types from the FastAPI response), Next.js will throw a serialization error at build time or silently coerce the values.

The spec uses `Float32Array` for ECG signal data (Node 3.5 Section 5: 240 KB raw). `Float32Array` is not serializable through React Server Component boundaries.

**Required fix:** The ECG signal data must be transferred as a plain JSON array (or base64-encoded ArrayBuffer) through the Server Component boundary, then converted to `Float32Array` client-side before rendering. Document the conversion in the architecture spec:

```typescript
// In Server Component: return signal as base64 string
const signalBase64 = Buffer.from(signalFloat32Array.buffer).toString('base64');

// In Client Component: decode to Float32Array
const buffer = Buffer.from(signalBase64, 'base64');
const signal = new Float32Array(buffer.buffer);
```

### REQUIRED-2: Performance budget for Q&A response (< 3 s first token) has no fallback for network timeout

**Location:** Node 3.5, Section 5 (performance budget)

The spec targets < 3 s first token for the Q&A streaming response. If the DeepSeek API is slow or the network is degraded, the user sees a spinner with no feedback for potentially 10–30 seconds. In an ER environment, a clinician waiting with no feedback may click away or assume the system hung.

**Required fix:** The Q&A interface must show a loading indicator with a timeout message. After 5 seconds without a first token:
- Show: "Response is taking longer than expected. Still working..."
After 15 seconds without a first token:
- Show: "The AI response timed out. The signal analysis is still available above." — and abort the streaming request.

The abort mechanism must use `AbortController` with a 15-second timeout on the fetch call.

## Not Issues

- Next.js App Router structure with Server/Client Component split is correct.
- Canvas for signal trace + D3 SVG for annotations is the correct performance approach.
- React Context for UI state (zoom/pan) with `useMemo` is correct to avoid unnecessary re-renders.
- Performance budget values (< 200 ms render, < 100 ms zoom) are achievable with the Canvas approach.
