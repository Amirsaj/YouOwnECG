## 1. Dashboard Template System

Every per-disease dashboard is an instance of a five-component template:

```
┌─────────────────────────────────────────────────────────┐
│ [1] HEADER: Disease name · Confidence badge · STAT flag │
├──────────────────────────────┬──────────────────────────┤
│ [2] ECG PANEL                │ [3] EXPLANATION PANEL    │
│  Only relevant leads         │  Why this is abnormal    │
│  Color highlights on         │  In clinician language   │
│  abnormal segments           │  (not agent reasoning)   │
│  Arrows on key findings      │                          │
├──────────────────────────────┼──────────────────────────┤
│ [4] EVIDENCE PANEL                                      │
│  RAG-retrieved textbook excerpt · Book/chapter/page     │
│  Citation: "Chou's, Ch. 5, p. 142, Fig. 5.3"          │
├─────────────────────────────────────────────────────────┤
│ [5] ACTION PANEL             (STAT conditions only)     │
│  "Consider immediate cath lab activation"               │
│  Acknowledge button                                     │
└─────────────────────────────────────────────────────────┘
```

The action panel (component 5) is only rendered for STAT findings. For non-STAT findings it is absent.

### 1.1 ECG Panel — Highlight Logic

The ECG panel shows only the leads relevant to the finding (not all 12). Relevance is determined by the finding's territory map:

```python
DASHBOARD_LEADS = {
    "anterior_stemi":   ["V1", "V2", "V3", "V4"],
    "inferior_stemi":   ["II", "III", "aVF", "I", "aVL"],  # + lateral for reciprocal
    "lateral_stemi":    ["I", "aVL", "V5", "V6"],
    "afib":             ["II"],                             # rhythm strip only
    "lbbb":             ["I", "V1", "V5", "V6"],
    "rbbb":             ["V1", "V2", "I", "V6"],
    "long_qt":          ["II", "V5"],                       # best QT measurement leads
    "brugada":          ["V1", "V2", "V3"],
    "wellens":          ["V2", "V3"],
    "de_winter":        ["V2", "V3", "V4", "V5", "aVR"],
    "lvh":              ["I", "aVL", "V1", "V5", "V6"],
    "complete_avb":     ["II"],                             # rhythm strip showing AV dissociation
    "hyperkalemia":     ["II", "V1", "V5"],
    "pericarditis":     ["II", "aVR", "V5"],
}
```

**Highlights:**
- Abnormal segment color: orange for elevation, blue for depression, purple for inversion
- Normal comparison overlay: thin gray trace showing the normal range
- Arrow annotations: SVG arrows pointing to specific features (J-point, T nadir, epsilon wave location)

### 1.2 Explanation Panel — Language Rules

The explanation panel uses pre-defined templates filled with computed values. It does NOT use agent free-text reasoning — that is in the reasoning chain viewer (Node 3.4). The explanation is factual and concise:

```python
EXPLANATION_TEMPLATES = {
    "anterior_stemi": (
        "ST elevation is present in {leads} (max {max_st:.1f} mm), "
        "consistent with anterior ST-elevation myocardial infarction "
        "involving the left anterior descending artery territory. "
        "Reciprocal depression {reciprocal_str}."
    ),
    "afib": (
        "The RR intervals are irregular (variability {rr_cv:.0%}). "
        "P waves are absent — replaced by irregular fibrillatory baseline. "
        "Ventricular rate: {hr:.0f} bpm."
    ),
    "long_qt": (
        "QTc is {qtc_bazett:.0f} ms (Bazett) / {qtc_frid:.0f} ms (Fridericia), "
        "above the upper limit of normal ({threshold:.0f} ms for this patient). "
        "Risk of Torsades de Pointes is elevated."
    ),
    # ... one template per condition
}
```

Values are drawn directly from the Node 1.5 feature object. No LLM involvement in the explanation panel — it is fully deterministic.

### 1.3 VL2 Unavailability Fallback

When `vision.vl2_call_succeeded = False`, the ECG panel uses the rendered image from the signal pipeline without the VL2 visual overlay. No fallback is needed for the explanation panel (it is signal-based). The evidence panel falls back to signal-only citations. A subtle notice is shown: "Visual analysis unavailable."

## 2. STEMI Dashboards (by Territory)

Five STEMI territory dashboards, each pre-configured:

| Territory | Primary leads | Reciprocal leads | Artery |
|-----------|--------------|-----------------|--------|
| Anterior | V1–V4 | II, III, aVF | LAD |
| Inferior | II, III, aVF | I, aVL, V1–V4 | RCA / LCx |
| Lateral | I, aVL, V5, V6 | II, III, aVF | LCx |
| Posterior | V1–V2 (depression) | V7–V9 (if available) | LCx / RCA |
| Right Ventricular | V1, V4R (V4 if V4R absent) | — | RCA proximal |

STEMI dashboards include:
- Numeric ST values per lead in a table
- Reciprocal change table (present/absent per expected territory)
- Sgarbossa score if LBBB is also present
- STAT action panel: "Consider immediate cath lab activation — clinical correlation required"

## 3. Arrhythmia Dashboards

**AFib:** rhythm strip (lead II, full 10 s), RR interval scatter plot (irregular pattern visualization), absent P-wave annotation, fibrillatory baseline highlight

**AV Blocks:**
- 1° AVB: PR interval display with normal range bar, per-beat PR values
- Wenckebach: PR sequence display (progressive lengthening → dropped beat, annotated)
- Mobitz II: fixed PR bars then dropped beat marker
- Complete AVB: P-P and R-R interval ladders (classic AV block ladder diagram)

**VT:** wide complex tachycardia annotation, STAT red banner, per-beat QRS duration shown

**BBB:** QRS morphology display in V1 and I/V6, rSR' vs broad notched R annotation

---

## 4. Special Pattern Dashboards

**Wellens:** V2 and V3 strips only. T-wave highlighted. Type A (biphasic) vs Type B (deep symmetric) classification from IT agent. Warning: "Represents critical LAD stenosis — patient may be pain-free at time of ECG."

**de Winter:** V2–V5 and aVR. Upsloping ST depression highlighted in blue. Tall symmetric T highlighted in orange. aVR elevation noted.

**Brugada:** V1–V3 only. Type 1 (coved): convex ST highlighted with descending T highlighted. Type 2 (saddle-back): labeled separately. Note: "Requires EP consultation."

**Long QT:** QT interval bracket on best-measurement lead (II or V5). QTc values for all 4 formulas in table. Normal range bar. TdP risk message if QTc > 500 ms.

**Hyperkalemia:** Progressive severity display — peaked T → widened QRS → loss of P → sine wave. Current stage highlighted based on features.
