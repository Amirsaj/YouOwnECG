## 1. Clinical ECG Display Standards

The standard clinical 12-lead layout is the **3×4+rhythm** format — identical to what cardiologists read on paper ECGs. Any deviation from this format requires a cardiologist to mentally re-orient, introducing error risk. This layout is mandatory for the general dashboard.

```
Row 1: I   | aVR | V1 | V4    (each 2.5 seconds)
Row 2: II  | aVL | V2 | V5
Row 3: III | aVF | V3 | V6
Row 4: II  (full-width 10-second rhythm strip)
```

**Display scale:**
- Paper speed: 25 mm/s — one small square (1 mm) = 40 ms
- Amplitude: 10 mm/mV — one small square = 0.1 mV
- Small squares: 1 mm; large squares: 5 mm (5 small squares)
- Calibration pulse: 1 mV (10 mm) square wave, 200 ms wide, start of each strip

**Color convention:**
- ECG trace: black on white background
- Grid small: #FFCCCC (light red); grid large: #FF9999 (red)
- Lead labels: dark gray, upper-left of each strip

## 2. D3.js Rendering Architecture

### 2.1 SVG vs Canvas Decision

At 500 Hz × 10 seconds × 12 leads = 60,000 sample points. SVG renders each point as a DOM element — 60,000 SVG elements will cause severe performance degradation (> 1 second render time). **Canvas must be used for ECG trace rendering.** SVG is used only for annotations (fiducial markers, interval brackets) which are sparse (< 500 elements).

```
Architecture:
  <div class="ecg-viewer">
    <canvas id="ecg-trace-layer" />     ← ECG signal + grid (Canvas 2D API)
    <svg id="ecg-annotation-layer" />   ← Fiducials, labels, finding highlights (SVG)
    <div id="ecg-interaction-layer" />  ← Hit-test overlay for click/zoom/pan
  </div>
```

Canvas layer renders the ECG signal at 60fps using `requestAnimationFrame`. SVG annotation layer sits on top, CSS `pointer-events: none` so it does not intercept mouse events. The interaction layer (transparent div) handles all mouse events and forwards to either the canvas (for zoom/pan) or the annotation logic (for beat selection, Q&A).

### 2.2 Downsampling for Display

500 Hz signal has 1 sample every 2 ms. At 6 px/mm × 25 mm/s = 3 px per 40 ms = 0.075 px/sample. Multiple samples map to the same pixel column — render all of them as the `min-max envelope` at each pixel column (prevents aliasing artifacts that look like noise):

```javascript
function renderMinMaxEnvelope(ctx, signal, pixelWidth, sampleStart, sampleEnd) {
    const samplesPerPixel = (sampleEnd - sampleStart) / pixelWidth;
    for (let px = 0; px < pixelWidth; px++) {
        const s0 = Math.floor(px * samplesPerPixel) + sampleStart;
        const s1 = Math.floor((px + 1) * samplesPerPixel) + sampleStart;
        const slice = signal.slice(s0, s1);
        const yMin = mvToPixel(Math.min(...slice));
        const yMax = mvToPixel(Math.max(...slice));
        ctx.beginPath();
        ctx.moveTo(px, yMin);
        ctx.lineTo(px, yMax);
        ctx.stroke();
    }
}
```

### 2.3 Zoom and Pan

- **Zoom (time axis):** scales `sampleStart`/`sampleEnd` window, re-renders canvas
- **Zoom (amplitude):** scales mV-to-pixel conversion factor
- **Pan:** shifts `sampleStart` while keeping window width fixed
- Minimum time window: 1 second (clinical readability floor)
- Maximum time window: full recording
- Interaction: scroll wheel (zoom time), shift+scroll (zoom amplitude), drag (pan)
- Performance target: zoom/pan < 100 ms re-render

## 3. Annotation Overlays

Annotations are rendered in the SVG layer and toggled independently:

| Layer | What it shows | Default |
|-------|--------------|---------|
| `fiducials` | P/QRS/T onset, peak, offset markers | OFF |
| `intervals` | PR, QRS, QT brackets with ms values | OFF |
| `findings` | Color-coded segment highlights for abnormal regions | ON |
| `beat_labels` | Beat class label above each QRS (N/V/A) | OFF |
| `quality` | Lead quality badge (GOOD/POOR/UNUSABLE) | ON |

**Finding highlight encoding:**
- ST elevation: orange overlay on ST segment
- ST depression: blue overlay on ST segment
- T-wave inversion: purple overlay on T segment
- QRS widening: gray overlay on QRS
- STAT finding: red flashing border on affected lead strip

**Toggle UI:** layer visibility controlled by a compact checkbox panel in the top-right corner. All toggles accessible via keyboard (Tab + Space).

## 4. Interactive Controls

### 4.1 Beat Selection

Each beat is a hit-test target. The interaction layer tracks the time axis — a click at pixel `x` maps to sample index `s = sampleStart + x / pixelsPerSample`. The nearest R-peak index from the FiducialTable determines which beat was clicked.

On beat click:
- Highlight the beat in the annotation layer (blue border around RR window)
- Show a beat detail panel: beat class, all fiducial indices, all features for this beat
- The beat detail panel is the entry point for Q&A (see Section 5)

### 4.2 Lead Isolation

Clicking the lead label expands that lead to a full-width detail view (replaces the 3×4 grid). The expanded view shows:
- Full-width trace with fine grid
- All fiducial markers visible
- Per-lead features in a sidebar (ST elevation, T morphology, IDT, etc.)
- Click again to return to 3×4 grid

### 4.3 Measurement Calipers

Two-point caliper tool: user clicks point A, then point B. The system computes `(sampleB - sampleA) / 500 × 1000` and displays the interval in ms. Displayed as a bracket with ms value in the annotation SVG layer.

## 5. Natural-Language Q&A (Click-to-Ask)

### 5.1 Click Location Resolution

When the user asks a question after clicking a location, the system must resolve the click to a clinical context. The context assembly is:

```python
def assemble_qa_context(
    click_sample_idx: int,
    click_lead: str,
    feature_object: FeatureObject,
    vision_result: VisionVerificationResult,
    fiducial_table: FiducialTable,
) -> dict:
    """
    Resolves a click location to the nearest beat and lead,
    extracts relevant features, and assembles the context for the Q&A call.
    """
    # 1. Find nearest beat
    r_peaks = [b_idx for b_idx in fiducial_table.fpts[click_lead][:, 5]
               if b_idx != -1]  # column 5 = R peak
    beat_idx = min(range(len(r_peaks)), key=lambda i: abs(r_peaks[i] - click_sample_idx))

    # 2. Identify segment (P / QRS / ST / T) based on click position within beat
    beat = feature_object.beats[beat_idx]
    segment = classify_segment(click_sample_idx, beat.leads[click_lead], fiducial_table)

    # 3. Extract relevant features for this beat + lead
    relevant_features = {
        "beat_index": beat_idx,
        "beat_class": beat.beat_class,
        "lead": click_lead,
        "segment_clicked": segment,
        "pr_interval_ms": beat.pr_interval_ms,
        "qrs_duration_ms": beat.qrs_duration_ms,
        "qt_interval_ms": beat.qt_interval_ms,
        "st_elevation_mv": beat.leads[click_lead].st_elevation_mv,
        "st_morphology": beat.leads[click_lead].st_morphology,
        "t_morphology": beat.leads[click_lead].t_morphology,
        "r_amplitude_mv": beat.leads[click_lead].r_amplitude_mv,
    }

    # 4. Include aggregate for this lead
    relevant_features["lead_aggregate"] = {
        "st_elevation_mean": feature_object.aggregate.leads[click_lead].st_elevation_mean_mv,
        "t_morphology_dominant": feature_object.aggregate.leads[click_lead].t_morphology_dominant,
    }

    return {
        "question_context": relevant_features,
        "patient_sex": feature_object.patient_sex,
        "patient_age_years": feature_object.patient_age_years,
    }
```

### 5.2 Q&A Prompt Assembly

The Q&A uses DeepSeek reasoner (reasoning mode, budget 4,000 tokens — smaller than diagnostic agents since this is a targeted question, not a full analysis). The prompt:

```
System: You are a clinical cardiologist answering a specific question about an ECG.
The question is about a specific location on the ECG. Answer concisely and
reference the computed signal features provided. Do not invent measurements.
Always include a citation from the RAG context if available.

User: [User's natural language question]

ECG context:
  Location: Lead [V2], Beat [4], Segment [ST segment]
  ST elevation at this point: +0.28 mV (threshold for this patient: 2.0 mV)
  ST morphology: UPSLOPING
  T wave: UPRIGHT_PEAKED (T/QRS ratio: 0.82)
  [+ top-3 RAG chunks for this query]

Answer the user's question about this location.
```

### 5.3 Response Display

Responses are shown in a tooltip/panel near the click location, not in a separate chat window. This keeps the ECG in view while reading the answer. The panel shows:
- Answer text (≤ 150 words for conciseness)
- Citation: "Source: Chou's Electrocardiography, Ch. 5, p. 142"
- Confidence badge matching the agent confidence for the finding in question
- "AI-GENERATED — NOT A DIAGNOSIS" footer
