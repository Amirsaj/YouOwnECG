## 1. Retrieval Architecture

The retrieval pipeline is invoked by IT and MR agents during Phase 2 (Node 2.3). Each agent calls `retrieve(query, k=5)` with a sub-question derived from the current diagnostic finding. The pipeline returns ranked chunks with metadata.

---

## 2. Query Formulation

Agents do NOT pass the raw feature object to the RAG system. They formulate a focused natural-language query:

**IT agent example query for a suspected STEMI:**
```
"ST elevation anterior leads V1-V4 with hyperacute T waves differential diagnosis criteria"
```

**MR agent example query for QTc prolongation:**
```
"prolonged QTc Bazett 448ms borderline threshold drug-induced risk assessment"
```

Query formulation is done by the agent as part of its reasoning chain (the agent decides what to ask). The retrieval pipeline receives a plain string.

## 3. Retrieval Function

```python
import faiss
import numpy as np

TOP_K_INITIAL = 20          # initial candidate pool
TOP_K_RETURNED = 5          # final returned to agent
SIMILARITY_THRESHOLD = 0.35  # minimum cosine similarity; below this, chunk is not returned

def retrieve(
    query: str,
    index: faiss.IndexFlatIP,
    metadata: list[dict],
    embedder,  # async embed_query function
    top_k: int = TOP_K_RETURNED,
) -> list[dict]:
    """
    Returns up to top_k chunks most relevant to query.
    Each result includes chunk metadata + similarity score + adjacent chunk IDs.
    """
    query_embedding = embedder(query)  # L2-normalized, shape (1536,)
    query_vec = query_embedding.reshape(1, -1).astype(np.float32)

    scores, indices = index.search(query_vec, TOP_K_INITIAL)
    scores = scores[0]
    indices = indices[0]

    results = []
    for score, idx in zip(scores, indices):
        if idx == -1 or score < SIMILARITY_THRESHOLD:
            continue
        chunk = {**metadata[idx], "similarity_score": float(score)}
        results.append(chunk)

    # Deduplicate: if adjacent chunks both retrieved, keep only the higher-scoring one
    results = _deduplicate_adjacent(results)

    return results[:top_k]


def _deduplicate_adjacent(results: list[dict]) -> list[dict]:
    """
    If a chunk and its next_chunk_id (or prev_chunk_id) are both in results,
    keep only the higher-scoring one. Prevents returning near-duplicate text.
    """
    chunk_ids_in_results = {r["chunk_id"] for r in results}
    filtered = []
    suppressed = set()

    for result in sorted(results, key=lambda r: r["similarity_score"], reverse=True):
        if result["chunk_id"] in suppressed:
            continue
        filtered.append(result)
        # Suppress adjacent if present
        for adj_key in ("prev_chunk_id", "next_chunk_id"):
            adj_id = result.get(adj_key)
            if adj_id and adj_id in chunk_ids_in_results:
                suppressed.add(adj_id)

    return filtered
```

## 4. Context Expansion

When a chunk scores above `CONTEXT_EXPANSION_THRESHOLD = 0.70`, its adjacent chunk (prev or next, whichever is in the same section) is appended to the returned text. This recovers context split across a paragraph boundary without retrieving it as a separate ranked result.

```python
CONTEXT_EXPANSION_THRESHOLD = 0.70

def expand_context(result: dict, metadata: list[dict], metadata_index: dict) -> dict:
    """
    If similarity_score >= CONTEXT_EXPANSION_THRESHOLD, appends adjacent chunk text.
    metadata_index: dict mapping chunk_id → metadata row.
    """
    if result["similarity_score"] < CONTEXT_EXPANSION_THRESHOLD:
        return result

    expanded_text = result["text"]
    next_id = result.get("next_chunk_id")

    if next_id and next_id in metadata_index:
        next_chunk = metadata_index[next_id]
        # Only expand if same section
        if next_chunk["section"] == result["section"]:
            expanded_text = expanded_text + "\n\n" + next_chunk["text"]
            result = {**result, "text": expanded_text, "context_expanded": True}

    return result
```

## 5. Metadata Filtering

Before FAISS search, a pre-filter can restrict candidates to specific books or chapter types. This is NOT used in normal operation (all books are authoritative). It is exposed as an optional parameter for the Q&A interface (Node 3.1 Section 5) when the user selects "Search only Chou's."

```python
def filter_metadata_by_book(
    metadata: list[dict],
    allowed_books: list[str],
    index: faiss.IndexFlatIP,
) -> tuple[faiss.IndexFlatIP, list[dict]]:
    """
    Returns a subset index and metadata for allowed_books only.
    Used for cardiologist-directed searches.
    """
    subset_indices = [i for i, m in enumerate(metadata) if m["book_short"] in allowed_books]
    subset_metadata = [metadata[i] for i in subset_indices]

    # Rebuild subset index
    dim = index.d
    sub_index = faiss.IndexFlatIP(dim)
    vectors = np.stack([index.reconstruct(i) for i in subset_indices]).astype(np.float32)
    sub_index.add(vectors)

    return sub_index, subset_metadata
```

Retrieved chunks are formatted into a context block passed to the agent's system prompt:

```

"ST elevation in the anterior leads (V1–V4) exceeding 1 mm in men or 1.5 mm in women, particularly when upsloping and accompanied by hyperacute (tall, peaked, symmetric) T waves, is diagnostic of early STEMI involving the LAD territory. Progressive beat-to-beat increase in ST elevation over 30–60 minutes may precede diagnostic threshold elevation..."

[2] Marriott's Practical Electrocardiography, Chapter 8: Myocardial Infarction, Section 8.3 Hyperacute T Waves (p. 211) [similarity: 0.74]
"Hyperacute T waves, defined as T amplitude > QRS amplitude in the same lead, represent the earliest electrocardiographic manifestation of myocardial ischemia, preceding ST elevation by 15–30 minutes in many cases..."
```
